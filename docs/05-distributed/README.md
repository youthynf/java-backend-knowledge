# 分布式

分布式模块关注多节点环境下的核心问题：**一致性、可用性、并发控制、故障隔离和事务协调**。面试中不要只背名词，要能说明每个方案解决什么问题、牺牲了什么。

## 面试官想考什么

- 你是否理解 CAP、BASE、最终一致性等分布式基础思想；
- 你是否能说明 Redis/ZooKeeper/数据库实现分布式锁的差异；
- 你是否知道 2PC、TCC、Saga、本地消息表等事务方案的适用边界；
- 你是否能解释熔断、限流、降级如何防止雪崩。

## 标准回答框架

> 分布式问题通常不是追求绝对完美，而是在一致性、性能、可用性和复杂度之间做取舍。回答时先明确业务目标，再选择方案，最后说明异常场景和补偿机制。

## 易错点

- 把分布式锁当作万能并发方案；
- 忽略网络分区、时钟漂移、节点宕机等异常；
- 只谈强一致，不考虑性能和可用性；
- 只说方案，不说失败后的恢复与观测。
