# 结构型模式-代理模式

结构型模式-代理模式
目的是为其他对象提供一种代理以控制对这个对象的访问。关键要点：主题接口 + 真实主题 + 代理类（持有真实主题引用），代理类在调用真实方法前后可添加额外逻辑（如延迟加载、日志）。

// 主题接口
public interface Image {
    void display();
}

// 真实主题（重量级对象，如大图片）
public class RealImage implements Image {
    private String filename;

    public RealImage(String filename) {
        this.filename = filename;
        loadFromDisk();  // 加载图片（耗时操作）
    }

    private void loadFromDisk() {
        System.out.println("从磁盘加载图片：" + filename);
    }

    @Override
    public void display() {
        System.out.println("显示图片：" + filename);
    }
}

// 代理主题（控制对真实主题的访问）
public class ProxyImage implements Image {
    private RealImage realImage;
    private String filename;

    public ProxyImage(String filename) {
        this.filename = filename;
    }

    @Override
    public void display() {
        // 延迟加载：仅在需要显示时才创建真实对象
        if (realImage == null) {
            realImage = new RealImage(filename);
        }
        realImage.display();  // 调用真实对象的方法
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        Image image = new ProxyImage("photo.jpg");
        // 第一次调用：触发加载并显示
        image.display();  // 输出：从磁盘加载图片：photo.jpg → 显示图片：photo.jpg
        // 第二次调用：直接显示（已加载）
        image.display();  // 输出：显示图片：photo.jpg
    }
}
助记：代理模式是“控制访问”，在功能上更像是一种封装和增强

## 面试复习版

### 核心概念
- 结构型模式关注类/对象组合关系。
- 代理控制访问，装饰器动态增强，适配器转换接口，外观简化入口，桥接分离抽象与实现，组合表达树形结构，享元共享细粒度对象。

### 面试官想考什么
- 相似模式如何区分。
- JDK/Spring 中的实际应用。

### 标准回答
结构型模式-代理模式 的核心是通过组合降低耦合或增强能力。面试回答先说明意图，再给出典型场景，最后与相近模式对比。

### 深挖追问
- 代理和装饰器区别？
- 适配器和外观区别？
- 组合模式如何处理叶子和容器一致性？

### 实战场景/代码示例
```java
interface Service { void call(); }
class LoggingService implements Service {
  private final Service target;
  public void call(){ System.out.println("log"); target.call(); }
}
```

### 易错点/总结
- 结构型模式常优先使用组合而非继承。
- 不要只背 UML，要能落到框架例子。
