# 创建型模式-单例模式

创建型模式-单例模式
单例模式
使用 volatile 修饰保证可见性和防止重排序；
构造方法私有化；
使用 synchronized 同步代码块+双重判空。

public class Singleton {
    //  volatile保证多线程下instance的可见性
    private static volatile Singleton instance;

    // 私有构造方法，禁止外部实例化
    private Singleton() {}

    // 双重检查锁定（DCL），兼顾性能与线程安全
    public static Singleton getInstance() {
        if (instance == null) {  // 第一次检查（无锁，提高效率）
            synchronized (Singleton.class) {  // 加锁
                if (instance == null) {  // 第二次检查（避免多线程并发创建）
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
助记：volatile防止重排序，构造方法私有，synchronized同步代码块，双重判空

## 面试复习版

### 核心概念
- 单例保证一个类在 JVM 内通常只有一个实例，并提供全局访问点。
- 常见实现有饿汉、懒汉、双重检查锁、静态内部类、枚举。

### 面试官想考什么
- 线程安全、懒加载、反射/序列化破坏。
- volatile 在 DCL 中的作用。

### 标准回答
推荐回答静态内部类或枚举单例；DCL 必须配合 volatile，避免指令重排导致半初始化对象被读取。

### 深挖追问
- 枚举单例为什么更安全？
- Spring Bean 单例是否等同设计模式单例？
- 单例有哪些缺点？

### 实战场景/代码示例
```java
class Singleton{
 private Singleton(){}
 private static class Holder{ static final Singleton I=new Singleton(); }
 static Singleton get(){ return Holder.I; }
}
```

### 易错点/总结
- 单例是全局状态，可能增加测试和并发复杂度。
- 分布式环境下 JVM 单例不等于全局唯一。
