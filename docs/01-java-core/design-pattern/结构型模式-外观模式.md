# 结构型模式-外观模式

结构型模式-外观模式
为子系统中的一组接口提供一个统一的高层接口，简化子系统的使用（如电脑启动只需按电源键，无需手动启动 CPU、内存、硬盘）。

// 子系统1：CPU
public class CPU {
    public void start() {
        System.out.println("CPU启动");
    }
}

// 子系统2：内存
public class Memory {
    public void start() {
        System.out.println("内存加载");
    }
}

// 子系统3：硬盘
public class HardDrive {
    public void start() {
        System.out.println("硬盘读取");
    }
}

// 外观类：提供简化接口
public class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive hardDrive;

    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.hardDrive = new HardDrive();
    }

    // 高层接口：一键启动电脑
    public void start() {
        cpu.start();
        memory.start();
        hardDrive.start();
        System.out.println("电脑启动完成");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        ComputerFacade computer = new ComputerFacade();
        computer.start(); // 客户端无需关心子系统细节
        // 输出：
        // CPU启动
        // 内存加载
        // 硬盘读取
        // 电脑启动完成
    }
}

## 面试复习要点

面试中回答“结构型模式-外观模式”时，不要只给结论，建议按 **定位 → 原理 → 使用边界 → 排查方式** 展开：

1. **先讲定位**：说明它解决的核心问题，以及在整个技术栈中处于哪一层。
2. **再讲原理**：把关键流程、核心数据结构或关键组件讲清楚，避免只背名词。
3. **补充边界**：说明它适合什么场景，不适合什么场景，以及常见误用。
4. **结合排查**：如果线上出现性能、稳定性或一致性问题，要能说出观测指标、日志线索和排查顺序。

## 标准回答思路

可以这样组织答案：**结构型模式-外观模式 的核心价值是解决特定场景下的工程问题。实际使用时，需要理解它的工作机制、成本和限制，不能只看 API 或表面现象。在线上落地时，还要结合监控、日志和压测结果判断是否真的适合当前业务。**
