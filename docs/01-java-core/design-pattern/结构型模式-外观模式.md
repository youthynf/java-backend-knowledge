# 结构型模式-外观模式

结构型模式-外观模式
为子系统中的一组接口提供一个统一的高层接口，简化子系统的使用（如电脑启动只需按电源键，无需手动启动 CPU、内存、硬盘）。

// 子系统1：CPU
public class CPU {
    public void start() {
        System.out.println("CPU启动");
    }
}

// 子系统2：内存
public class Memory {
    public void start() {
        System.out.println("内存加载");
    }
}

// 子系统3：硬盘
public class HardDrive {
    public void start() {
        System.out.println("硬盘读取");
    }
}

// 外观类：提供简化接口
public class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive hardDrive;

    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.hardDrive = new HardDrive();
    }

    // 高层接口：一键启动电脑
    public void start() {
        cpu.start();
        memory.start();
        hardDrive.start();
        System.out.println("电脑启动完成");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        ComputerFacade computer = new ComputerFacade();
        computer.start(); // 客户端无需关心子系统细节
        // 输出：
        // CPU启动
        // 内存加载
        // 硬盘读取
        // 电脑启动完成
    }
}

## 面试复习版

### 核心概念
- 结构型模式关注类/对象组合关系。
- 代理控制访问，装饰器动态增强，适配器转换接口，外观简化入口，桥接分离抽象与实现，组合表达树形结构，享元共享细粒度对象。

### 面试官想考什么
- 相似模式如何区分。
- JDK/Spring 中的实际应用。

### 标准回答
结构型模式-外观模式 的核心是通过组合降低耦合或增强能力。面试回答先说明意图，再给出典型场景，最后与相近模式对比。

### 深挖追问
- 代理和装饰器区别？
- 适配器和外观区别？
- 组合模式如何处理叶子和容器一致性？

### 实战场景/代码示例
```java
interface Service { void call(); }
class LoggingService implements Service {
  private final Service target;
  public void call(){ System.out.println("log"); target.call(); }
}
```

### 易错点/总结
- 结构型模式常优先使用组合而非继承。
- 不要只背 UML，要能落到框架例子。
