# 行为型模式-备忘录模式

行为型模式-备忘录模式
在不破坏封装的前提下，捕获对象的内部状态并保存，以便后续恢复（如游戏存档、撤销操作）。

// 备忘录：存储状态
public class Memento {
    private String text;

    public Memento(String text) {
        this.text = text;
    }

    // 仅允许原发器访问状态
    String getText() {
        return text;
    }
}

// 原发器：需要保存状态的对象
public class TextEditor {
    private String text = "";

    public void type(String content) {
        text += content;
    }

    public String getText() {
        return text;
    }

    // 创建备忘录（保存当前状态）
    public Memento save() {
        return new Memento(text);
    }

    // 从备忘录恢复状态
    public void restore(Memento memento) {
        this.text = memento.getText();
    }
}

// 负责人：管理备忘录（可选）
public class Caretaker {
    private Memento memento;

    public void saveMemento(Memento m) {
        memento = m;
    }

    public Memento getMemento() {
        return memento;
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();
        Caretaker caretaker = new Caretaker();

        editor.type("Hello ");
        caretaker.saveMemento(editor.save()); // 存档1

        editor.type("World");
        System.out.println(editor.getText()); // Hello World

        editor.restore(caretaker.getMemento()); // 恢复到存档1
        System.out.println(editor.getText()); // Hello 
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-备忘录模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
