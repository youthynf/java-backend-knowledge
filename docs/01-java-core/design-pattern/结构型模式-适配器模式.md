# 结构型模式-适配器模式

结构型模式-适配器模式
目的是将一个类的接口转换成客户端期望的另一个接口，使不兼容的类可以一起工作。关键要点：目标接口 + 适配者 + 适配器（继承或组合适配者），实现接口转换。

// 目标接口（客户端期望的接口）
public interface TypeC {
    void chargeWithTypeC();
}

// 适配者（需要被适配的类，接口不兼容）
public class Usb {
    public void chargeWithUsb() {
        System.out.println("使用USB充电");
    }
}

// 类适配器（通过继承适配者，实现目标接口）
public class UsbToTypeCAdapter extends Usb implements TypeC {
    @Override
    public void chargeWithTypeC() {
        // 适配逻辑：将Type-C调用转换为USB调用
        super.chargeWithUsb();
        System.out.println("（通过适配器转换为Type-C）");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        TypeC adapter = new UsbToTypeCAdapter();
        adapter.chargeWithTypeC();  // 输出：使用USB充电（通过适配器转换为Type-C）
    }
}

- 结构型模式关注类/对象组合关系。
