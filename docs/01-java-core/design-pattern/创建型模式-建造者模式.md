# 创建型模式-建造者模式

创建型模式-建造者模式
目的是将复杂对象的构建与表示分离，使得同样的构建过程可以创建不同的表示。关键要点：产品类私有构造 + 静态 Builder 内部类 + 链式调用，解决 “telescoping constructor（重叠构造器）” 的冗余问题。

// 产品：电脑
public class Computer {
    private String cpu;    // 必选
    private String ram;    // 必选
    private String gpu;    // 可选
    private String storage;// 可选

    // 私有构造器，仅允许Builder访问
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ram = builder.ram;
        this.gpu = builder.gpu;
        this.storage = builder.storage;
    }

    // 静态内部类：Builder
    public static class Builder {
        private String cpu;    // 必选参数（无默认值）
        private String ram;    // 必选参数

        private String gpu = "集成显卡";  // 可选参数（默认值）
        private String storage = "512GB SSD";

        // 必选参数在构造器中传入
        public Builder(String cpu, String ram) {
            this.cpu = cpu;
            this.ram = ram;
        }

        // 可选参数通过链式方法设置
        public Builder gpu(String gpu) {
            this.gpu = gpu;
            return this;
        }

        public Builder storage(String storage) {
            this.storage = storage;
            return this;
        }

        // 构建产品
        public Computer build() {
            return new Computer(this);
        }
    }

    // 测试
    public static void main(String[] args) {
        Computer gamingPc = new Computer.Builder("i9-13900K", "32GB DDR5")
                .gpu("RTX 4090")
                .storage("2TB SSD")
                .build();
    }
}

## 面试复习版

### 核心概念
- 建造者模式分步骤构建复杂对象。
- 适合参数多、可选项多、需要校验构建过程的对象。

### 面试官想考什么
- 与构造器、JavaBean 的区别。
- 链式调用和不可变对象构建。

### 标准回答
Builder 将复杂构建过程封装起来，避免长参数列表，常与不可变对象配合。build 阶段集中校验必填项和组合约束。

### 深挖追问
- Lombok @Builder 有哪些坑？
- Builder 和工厂模式区别？
- 如何保证构建对象不可变？

### 实战场景/代码示例
```java
User u=User.builder().id(1L).name("Tom").build();
```

### 易错点/总结
- 不要为简单 DTO 过度使用。
- Builder 仍需做参数校验。
