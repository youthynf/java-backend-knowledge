## 面试复习版

### 核心概念
- 设计模式是可复用的设计经验，用于解决特定上下文中的常见问题。
- 按目的分为创建型、结构型、行为型。

### 面试官想考什么
- 能否说清意图、结构、场景、优缺点。
- 能否结合 JDK/Spring 实例。

### 标准回答
回答设计模式题不要只背定义，应说明它解决什么变化点、牺牲了什么复杂度、在项目中如何落地。

### 深挖追问
- 模式和原则是什么关系？
- 如何避免过度设计？
- 项目中用过哪些模式？

### 实战场景/代码示例
```java
// 设计模式通常围绕接口编程、组合复用、封装变化点展开。
```

### 易错点/总结
- 没有业务变化点时不要硬套模式。
- 能对比相似模式比背诵更重要。
