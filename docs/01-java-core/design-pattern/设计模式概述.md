# 设计模式概述

设计模式概述
设计模式（Design Patterns）是软件设计中常见问题的典型解决方案，它们是由经验丰富的开发人员在长期实践中总结出来的最佳实践。设计模式主要分为三大类：创建型模式、结构型模式和行为型模式。以下是完整的 23 种经典设计模式（基于 GoF《设计模式》一书）及其简要说明：
一、创建型模式（5种）
关注对象的创建机制，提供灵活的对象创建方式，降低系统耦合度。
单例模式（Singleton）
•  确保一个类只有一个实例，并提供全局访问点。
•  示例：数据库连接池、日志管理器。

工厂方法模式（Factory Method）
•  定义一个创建对象的接口，但由子类决定实例化哪个类。
•  示例：日志记录器（文件日志 vs 数据库日志）。

抽象工厂模式（Abstract Factory）
•  创建相关或依赖对象的家族，而无需指定具体类。
•  示例：跨平台 UI 组件库（Windows/Mac 按钮、文本框）。

建造者模式（Builder）
•  分步骤构建复杂对象，分离构造与表示。
•  示例：生成不同格式的文档（HTML、Markdown）。

原型模式（Prototype）
•  通过复制现有对象来创建新对象，避免重复初始化。
•  示例：游戏中的敌人克隆。

二、结构型模式（7种）
关注类和对象的组合，形成更大的结构。
适配器模式（Adapter）
•  将不兼容的接口转换为可兼容的接口。
•  示例：旧版支付接口适配新系统。

桥接模式（Bridge）
•  将抽象与实现分离，使它们可以独立变化。
•  示例：不同形状（圆形/方形）与颜色（红/蓝）的组合。

组合模式（Composite）
•  以树形结构组合对象，表示"部分-整体"层次。
•  示例：文件系统（文件+文件夹的统一操作）。

装饰器模式（Decorator）
•  动态地为对象添加额外职责，避免子类爆炸。
•  示例：Java I/O 流（BufferedInputStream 包装 FileInputStream）。

外观模式（Facade）
•  为复杂子系统提供简化接口。
•  示例：一键启动电脑（封装CPU、内存、硬盘等操作）。

享元模式（Flyweight）
•  共享大量细粒度对象，减少内存消耗。
•  示例：文本编辑器中的字符对象池。

代理模式（Proxy）
•  为其他对象提供代理以控制访问。
•  示例：远程代理（RPC）、虚拟代理（延迟加载图片）。

三、行为型模式（11种）
关注对象间的通信和职责分配。
责任链模式（Chain of Responsibility）
•  将请求沿处理链传递，直到有对象处理它。
•  示例：审批流程（经理→总监→CEO）。

命令模式（Command）
•  将请求封装为对象，支持撤销、队列等操作。
•  示例：GUI 按钮操作、事务管理。

解释器模式（Interpreter）
•  定义语言的语法表示，并解释执行。
•  示例：正则表达式解析器。

迭代器模式（Iterator）
•  提供一种方法顺序访问聚合对象的元素。
•  示例：Java 中的 Iterator 接口。

中介者模式（Mediator）
•  通过中介对象减少对象间直接耦合。
•  示例：聊天室（用户不直接通信，通过聊天室中转）。

备忘录模式（Memento）
•  捕获并外部化对象状态，以便后续恢复。
•  示例：游戏存档。

观察者模式（Observer）
•  定义对象间的一对多依赖，当一个对象状态改变时，所有依赖者自动通知。
•  示例：事件监听、消息订阅。

状态模式（State）
•  允许对象在内部状态改变时改变行为。
•  示例：订单状态（待支付→已发货→已完成）。

策略模式（Strategy）
•  定义一系列算法，封装每个算法并使它们可互换。
•  示例：排序算法（快速排序、冒泡排序的动态切换）。

模板方法模式（Template Method）
•  定义算法骨架，将某些步骤延迟到子类实现。
•  示例：咖啡与茶的制作流程（抽象共同的煮沸水步骤）。

访问者模式（Visitor）
•  将算法与对象结构分离，便于新增操作。
•  示例：编译器语法树分析。

四、其他重要模式
除了 GoF 的 23 种模式，还有一些常用的现代模式：
依赖注入（Dependency Injection）
•  通过外部注入依赖对象，降低耦合。
•  示例：Spring 框架的 @Autowired。

MVC模式（Model-View-Controller）
•  分离数据、界面和控制逻辑。
•  示例：Web 开发框架（如 Spring MVC）。

五、设计模式的选择原则
优先使用组合而非继承（如策略模式、装饰器模式）。
识别变化点：将易变的部分抽象出来（如工厂模式封装对象创建）。
遵循开闭原则：对扩展开放，对修改关闭（如观察者模式）。
