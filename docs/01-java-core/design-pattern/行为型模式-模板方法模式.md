# 行为型模式-模板方法模式

行为型模式-模板方法模式
定义一个操作中的算法骨架，将一些步骤延迟到子类中，使子类可以不改变算法结构而重定义特定步骤。关键要点：抽象类定义模板方法（固定流程）+ 抽象方法（子类实现可变步骤），模板方法用final修饰防止篡改流程。

// 抽象类（定义算法骨架）
public abstract class CaffeineBeverage {
    // 模板方法：固定流程，声明为final防止子类修改
    public final void prepareRecipe() {
        boilWater();       // 步骤1：煮水（固定）
        brew();            // 步骤2：冲泡（子类实现）
        pourInCup();       // 步骤3：倒入杯子（固定）
        addCondiments();   // 步骤4：加调料（子类实现）
    }

    // 抽象方法：由子类实现
    protected abstract void brew();
    protected abstract void addCondiments();

    // 具体方法：固定实现
    private void boilWater() {
        System.out.println("煮水");
    }

    private void pourInCup() {
        System.out.println("倒入杯子");
    }
}

// 具体子类：咖啡
public class Coffee extends CaffeineBeverage {
    @Override
    protected void brew() {
        System.out.println("用沸水冲泡咖啡");
    }

    @Override
    protected void addCondiments() {
        System.out.println("加糖和牛奶");
    }
}

// 具体子类：茶
public class Tea extends CaffeineBeverage {
    @Override
    protected void brew() {
        System.out.println("用沸水浸泡茶叶");
    }

    @Override
    protected void addCondiments() {
        System.out.println("加柠檬");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        CaffeineBeverage coffee = new Coffee();
        System.out.println("准备咖啡：");
        coffee.prepareRecipe();
        // 输出：
        // 准备咖啡：
        // 煮水
        // 用沸水冲泡咖啡
        // 倒入杯子
        // 加糖和牛奶

        CaffeineBeverage tea = new Tea();
        System.out.println("\n准备茶：");
        tea.prepareRecipe();
        // 输出：
        // 准备茶：
        // 煮水
        // 用沸水浸泡茶叶
        // 倒入杯子
        // 加柠檬
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-模板方法模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
