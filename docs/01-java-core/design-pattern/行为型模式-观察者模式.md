# 行为型模式-观察者模式

行为型模式-观察者模式
目的是定义对象间的一对多依赖，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。关键要点：主题接口（注册 / 移除 / 通知）+ 观察者接口（更新方法），实现松耦合的一对多通信。

import java.util.ArrayList;
import java.util.List;

// 主题接口（被观察者）
public interface Subject {
    void registerObserver(Observer observer);  // 注册观察者
    void removeObserver(Observer observer);   // 移除观察者
    void notifyObservers();                   // 通知所有观察者
}

// 观察者接口
public interface Observer {
    void update(float temperature, float humidity);  // 接收更新
}

// 具体主题：天气数据
public class WeatherData implements Subject {
    private List<Observer> observers;
    private float temperature;  // 温度
    private float humidity;     // 湿度

    public WeatherData() {
        observers = new ArrayList<>();
    }

    @Override
    public void registerObserver(Observer observer) {
        observers.add(observer);
    }

    @Override
    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }

    @Override
    public void notifyObservers() {
        // 通知所有观察者
        for (Observer observer : observers) {
            observer.update(temperature, humidity);
        }
    }

    // 当数据更新时，调用此方法触发通知
    public void setMeasurements(float temperature, float humidity) {
        this.temperature = temperature;
        this.humidity = humidity;
        notifyObservers();  // 数据变了，通知观察者
    }
}

// 具体观察者：手机APP
public class PhoneDisplay implements Observer {
    @Override
    public void update(float temperature, float humidity) {
        System.out.println("手机APP收到天气更新：温度=" + temperature + "℃，湿度=" + humidity + "%");
    }
}

// 具体观察者：仪表盘
public class DashboardDisplay implements Observer {
    @Override
    public void update(float temperature, float humidity) {
        System.out.println("仪表盘显示：温度=" + temperature + "℃，湿度=" + humidity + "%");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        WeatherData weatherData = new WeatherData();
        Observer phone = new PhoneDisplay();
        Observer dashboard = new DashboardDisplay();

        weatherData.registerObserver(phone);
        weatherData.registerObserver(dashboard);

        // 模拟数据更新
        weatherData.setMeasurements(25.5f, 60.0f);
        // 输出：
        // 手机APP收到天气更新：温度=25.5℃，湿度=60.0%
        // 仪表盘显示：温度=25.5℃，湿度=60.0%
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-观察者模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
