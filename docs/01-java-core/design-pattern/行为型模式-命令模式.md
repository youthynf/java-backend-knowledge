# 行为型模式-命令模式

行为型模式-命令模式
将请求封装为对象，使请求的发送者与接收者解耦，支持请求的参数化、队列化、撤销等。

// 命令接口
public interface Command {
    void execute();
    void undo(); // 可选：撤销操作
}

// 接收者：电灯
public class Light {
    public void on() {
        System.out.println("电灯打开");
    }

    public void off() {
        System.out.println("电灯关闭");
    }
}

// 具体命令：开灯
public class LightOnCommand implements Command {
    private Light light;

    public LightOnCommand(Light light) {
        this.light = light;
    }

    @Override
    public void execute() {
        light.on();
    }

    @Override
    public void undo() {
        light.off(); // 撤销=关灯
    }
}

// 具体命令：关灯
public class LightOffCommand implements Command {
    private Light light;

    public LightOffCommand(Light light) {
        this.light = light;
    }

    @Override
    public void execute() {
        light.off();
    }

    @Override
    public void undo() {
        light.on(); // 撤销=开灯
    }
}

// 调用者：遥控器
public class RemoteControl {
    private Command command;

    public void setCommand(Command command) {
        this.command = command;
    }

    public void pressButton() {
        command.execute();
    }

    public void pressUndoButton() {
        command.undo();
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        Light light = new Light();
        Command onCommand = new LightOnCommand(light);
        Command offCommand = new LightOffCommand(light);

        RemoteControl remote = new RemoteControl();

        remote.setCommand(onCommand);
        remote.pressButton(); // 电灯打开
        remote.pressUndoButton(); // 电灯关闭

        remote.setCommand(offCommand);
        remote.pressButton(); // 电灯关闭
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-命令模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
