# 行为型模式-迭代器模式

行为型模式-迭代器模式
提供一种方法顺序访问聚合对象（如集合）中的元素，而无需暴露其内部结构。

import java.util.NoSuchElementException;

// 迭代器接口
public interface Iterator<T> {
    boolean hasNext();
    T next();
}

// 聚合接口（可迭代对象）
public interface Aggregate<T> {
    Iterator<T> createIterator();
}

// 具体聚合：自定义列表
public class MyList<T> implements Aggregate<T> {
    private T[] elements;
    private int size;

    @SuppressWarnings("unchecked")
    public MyList(int capacity) {
        elements = (T[]) new Object[capacity];
    }

    public void add(T element) {
        elements[size++] = element;
    }

    public T get(int index) {
        return elements[index];
    }

    public int size() {
        return size;
    }

    @Override
    public Iterator<T> createIterator() {
        return new MyListIterator<>(this);
    }

    // 具体迭代器
    private static class MyListIterator<T> implements Iterator<T> {
        private MyList<T> list;
        private int index = 0;

        public MyListIterator(MyList<T> list) {
            this.list = list;
        }

        @Override
        public boolean hasNext() {
            return index < list.size();
        }

        @Override
        public T next() {
            if (!hasNext()) {
                throw new NoSuchElementException();
            }
            return list.get(index++);
        }
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        MyList<String> list = new MyList<>(3);
        list.add("A");
        list.add("B");
        list.add("C");

        Iterator<String> iterator = list.createIterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next()); // A → B → C
        }
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-迭代器模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
