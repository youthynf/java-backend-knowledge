# 结构型模式-装饰器模式

结构型模式-装饰器模式
目的是动态地给对象添加额外职责，不改变其原有结构。关键要点：组件接口 + 具体组件 + 装饰器抽象类（持有组件）+ 具体装饰器，支持多层嵌套装饰。

// 抽象组件（被装饰者的接口）
public abstract class Coffee {
    public abstract String getDescription();
    public abstract double cost();
}

// 具体组件（基础对象）
public class Espresso extends Coffee {
    @Override
    public String getDescription() {
        return "浓缩咖啡";
    }

    @Override
    public double cost() {
        return 20.0;
    }
}

// 装饰器抽象类（实现组件接口，持有组件引用）
public abstract class CoffeeDecorator extends Coffee {
    protected Coffee coffee;  // 被装饰的咖啡

    public CoffeeDecorator(Coffee coffee) {
        this.coffee = coffee;
    }
}

// 具体装饰器：加牛奶
public class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }

    @Override
    public String getDescription() {
        return coffee.getDescription() + " + 牛奶";
    }

    @Override
    public double cost() {
        return coffee.cost() + 5.0;  // 加牛奶加价5元
    }
}

// 具体装饰器：加糖
public class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) {
        super(coffee);
    }

    @Override
    public String getDescription() {
        return coffee.getDescription() + " + 糖";
    }

    @Override
    public double cost() {
        return coffee.cost() + 2.0;  // 加糖加价2元
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        Coffee coffee = new Espresso();
        coffee = new MilkDecorator(coffee);  // 加牛奶
        coffee = new SugarDecorator(coffee); // 再加糖
        System.out.println(coffee.getDescription() + "，总价：" + coffee.cost());
        // 输出：浓缩咖啡 + 牛奶 + 糖，总价：27.0
    }
}

## 面试复习版

### 核心概念
- 结构型模式关注类/对象组合关系。
- 代理控制访问，装饰器动态增强，适配器转换接口，外观简化入口，桥接分离抽象与实现，组合表达树形结构，享元共享细粒度对象。

### 面试官想考什么
- 相似模式如何区分。
- JDK/Spring 中的实际应用。

### 标准回答
结构型模式-装饰器模式 的核心是通过组合降低耦合或增强能力。面试回答先说明意图，再给出典型场景，最后与相近模式对比。
