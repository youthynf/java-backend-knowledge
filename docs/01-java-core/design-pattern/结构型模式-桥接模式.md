# 结构型模式-桥接模式

结构型模式-桥接模式
将抽象部分与实现部分分离，使它们可以独立变化（如分离 "形状" 与 "颜色"，各自扩展）。避免多维度变化导致的类爆炸（如圆形 + 红色、圆形 + 蓝色、方形 + 红色...）。

// 实现部分接口（颜色）
public interface Color {
    String getColor();
}

// 具体实现：红色
public class Red implements Color {
    @Override
    public String getColor() {
        return "红色";
    }
}

// 具体实现：蓝色
public class Blue implements Color {
    @Override
    public String getColor() {
        return "蓝色";
    }
}

// 抽象部分（形状）
public abstract class Shape {
    protected Color color; // 桥接：持有实现部分的引用

    public Shape(Color color) {
        this.color = color;
    }

    public abstract void draw();
}

// 具体抽象：圆形
public class Circle extends Shape {
    public Circle(Color color) {
        super(color);
    }

    @Override
    public void draw() {
        System.out.println("绘制" + color.getColor() + "的圆形");
    }
}

// 具体抽象：方形
public class Square extends Shape {
    public Square(Color color) {
        super(color);
    }

    @Override
    public void draw() {
        System.out.println("绘制" + color.getColor() + "的方形");
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        Shape redCircle = new Circle(new Red());
        redCircle.draw(); // 输出：绘制红色的圆形

        Shape blueSquare = new Square(new Blue());
        blueSquare.draw(); // 输出：绘制蓝色的方形
    }
}

## 面试复习版

### 核心概念
- 结构型模式关注类/对象组合关系。
- 代理控制访问，装饰器动态增强，适配器转换接口，外观简化入口，桥接分离抽象与实现，组合表达树形结构，享元共享细粒度对象。

### 面试官想考什么
- 相似模式如何区分。
- JDK/Spring 中的实际应用。

### 标准回答
结构型模式-桥接模式 的核心是通过组合降低耦合或增强能力。面试回答先说明意图，再给出典型场景，最后与相近模式对比。

### 深挖追问
- 代理和装饰器区别？
- 适配器和外观区别？
- 组合模式如何处理叶子和容器一致性？

### 实战场景/代码示例
```java
interface Service { void call(); }
class LoggingService implements Service {
  private final Service target;
  public void call(){ System.out.println("log"); target.call(); }
}
```

### 易错点/总结
- 结构型模式常优先使用组合而非继承。
- 不要只背 UML，要能落到框架例子。
