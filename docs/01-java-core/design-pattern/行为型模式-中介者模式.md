# 行为型模式-中介者模式

行为型模式-中介者模式
定义一个中介对象，封装对象间的交互，使对象无需直接通信，降低耦合。多个对象相互依赖（如聊天室、GUI 组件交互）。

import java.util.ArrayList;
import java.util.List;

// 中介者接口
public interface ChatMediator {
    void sendMessage(String message, User user);
    void addUser(User user);
}

// 具体中介者：聊天室
public class ChatRoom implements ChatMediator {
    private List<User> users = new ArrayList<>();

    @Override
    public void addUser(User user) {
        users.add(user);
    }

    @Override
    public void sendMessage(String message, User sender) {
        // 中介者转发消息给所有其他用户
        for (User user : users) {
            if (user != sender) {
                user.receive(message);
            }
        }
    }
}

// 同事类：用户
public abstract class User {
    protected ChatMediator mediator;
    protected String name;

    public User(ChatMediator mediator, String name) {
        this.mediator = mediator;
        this.name = name;
    }

    public abstract void send(String message);
    public abstract void receive(String message);
}

// 具体同事：普通用户
public class ConcreteUser extends User {
    public ConcreteUser(ChatMediator mediator, String name) {
        super(mediator, name);
    }

    @Override
    public void send(String message) {
        System.out.println(name + "发送：" + message);
        mediator.sendMessage(message, this);
    }

    @Override
    public void receive(String message) {
        System.out.println(name + "收到：" + message);
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        ChatMediator mediator = new ChatRoom();
        User user1 = new ConcreteUser(mediator, "张三");
        User user2 = new ConcreteUser(mediator, "李四");

        mediator.addUser(user1);
        mediator.addUser(user2);

        user1.send("大家好！"); 
        // 输出：
        // 张三发送：大家好！
        // 李四收到：大家好！
    }
}

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-中介者模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
