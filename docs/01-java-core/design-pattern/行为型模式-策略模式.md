# 行为型模式-策略模式

行为型模式-策略模式
目的是定义一系列算法，将每个算法封装起来并使它们可互换，让算法独立于使用它的客户端。关键要点：策略接口 + 具体策略 + 上下文（持有策略引用），客户端通过上下文切换策略，避免大量if-else。

// 策略接口
public interface PaymentStrategy {
    void pay(double amount);
}

// 具体策略：支付宝
public class AlipayStrategy implements PaymentStrategy {
    private String username;

    public AlipayStrategy(String username) {
        this.username = username;
    }

    @Override
    public void pay(double amount) {
        System.out.println(username + "使用支付宝支付了" + amount + "元");
    }
}

// 具体策略：微信支付
public class WechatPayStrategy implements PaymentStrategy {
    private String openId;

    public WechatPayStrategy(String openId) {
        this.openId = openId;
    }

    @Override
    public void pay(double amount) {
        System.out.println(openId + "使用微信支付了" + amount + "元");
    }
}

// 上下文（使用策略的客户端）
public class ShoppingCart {
    private PaymentStrategy paymentStrategy;

    // 设置支付策略（动态切换）
    public void setPaymentStrategy(PaymentStrategy strategy) {
        this.paymentStrategy = strategy;
    }

    // 结算
    public void checkout(double amount) {
        paymentStrategy.pay(amount);
    }
}

// 使用示例
public class Client {
    public static void main(String[] args) {
        ShoppingCart cart = new ShoppingCart();

        // 选择支付宝支付
        cart.setPaymentStrategy(new AlipayStrategy("user123"));
        cart.checkout(199.9);  // 输出：user123使用支付宝支付了199.9元

        // 切换为微信支付
        cart.setPaymentStrategy(new WechatPayStrategy("o6_bmjrPTlm6_2sgVt7hMZOPfL2M"));
        cart.checkout(299.5);  // 输出：o6_bmjrPTlm6_2sgVt7hMZOPfL2M使用微信支付了299.5元
    }
}
助记：适配器模式是“接口转换”，目的是让两个原本不兼容的类协同工作。

## 面试复习版

### 核心概念
- 行为型模式关注对象之间职责分配和交互流程。
- 策略封装算法，模板固定流程，观察者发布订阅，责任链逐级处理，状态把状态行为对象化，命令封装请求。

### 面试官想考什么
- 相似模式区分和适用边界。
- 在业务代码/框架中的落地。

### 标准回答
行为型模式-策略模式 应从“要解决的变化点”切入：是算法变化、流程骨架稳定、事件通知、请求流转，还是状态迁移。再说明参与角色和优缺点。

### 深挖追问
- 策略和状态有什么区别？
- 模板方法和策略能否组合？
- 责任链如何避免请求无人处理？

### 实战场景/代码示例
```java
interface DiscountStrategy { BigDecimal apply(BigDecimal amount); }
class VipDiscount implements DiscountStrategy { public BigDecimal apply(BigDecimal a){ return a.multiply(new BigDecimal("0.9")); } }
```

### 易错点/总结
- 模式是沟通语言，不是套壳模板。
- 链路过长、状态过多会增加排查成本。
