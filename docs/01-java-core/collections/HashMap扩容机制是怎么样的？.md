# HashMap扩容机制是怎么样的？

HashMap扩容机制是怎么样的？
在Java的HashMap中，扩容机制是其性能优化的一个重要部分。HashMap的扩容机制如下：
扩容的触发条件
HashMap扩容的触发是基于负载因子（Load Factor）和当前容量（Capacity）的关系：
负载因子：表示HashMap的装满程度，默认值为0.75。
当前容量：当存储的元素数量超过capacity * loadFactor时，就会触发扩容。
例如：默认初始容量是16，负载因子是0.75，则当元素达到12个时，刚好是达到阈值，但此时还不需要扩容，但元素达到13个时，超过这个阈值，则触发扩容机制。
为什么负载因子是0.75？
主要考虑了空间和时间复杂度的平衡，如果负载因子太低会导致大量的空桶，浪费空间；如果负载因子太高则会导致大量的碰撞，降低性能。

扩容的步骤
扩容通常会进行以下几步：
容量翻倍：HashMap的容量从当前值扩展为原来的两倍（如从16变为32）。扩容后，容量依然保持为2的n次方。
创建新数组：HashMap会为新的容量创建一个更大的桶数组（Node<K, V>[] table）。
数据重新分布（rehashing）：原有的数据需要重新分布到新的桶数组中。
复制数据：将旧数组中的数据按照新计算的索引分配到新数组中。
优化点解析：
HashMap利用扩容后容量是原来的两倍这一特性，通过检查哈希值的高位（新增的位是0还是1），快速将元素重新映射到新数组中，而无需完全重新计算哈希值。
假设原始容量为16，扩容为32，元素的位置有两种可能：
如果新增的高位是0，则位置不变，保留在原索引位置。
如果新增的高位是1，则位置变为i + oldCapacity，移动到新的索引位置（当前索引 + 原容量）。
