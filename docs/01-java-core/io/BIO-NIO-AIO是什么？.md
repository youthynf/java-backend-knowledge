# BIO/NIO/AIO是什么？

BIO/NIO/AIO是什么？
重要概念
阻塞IO和非阻塞IO
这两个概念是程序级别的. 主要描述的是程序请求操作系统IO操作后, 如果IO资源没有准备好, 那么程序如何处理的问题: 前者等待, 后者继续执行(并且使用线程一直轮询, 直到有IO资源准备好了).

同步IO和非同步IO
这两个概念是操作系统级别的. 主要描述的是操作系统在收到程序请求IO操作后, 如果IO资源没有准备好, 该如何响应程序的问题: 前者不响应, 直到IO资源响应准备好以后; 后者返回一个标记(好让程序和自己知道以后的数据往哪里通知), 当IO资源准备以后, 再佣时间机制返回给程序.

Java 相关IO
BIO（Blocking I/O）、NIO（Non - Blocking I/O）、AIO（Asynchronous I/O）是Java中三种不同的I/O处理方式，主要区别如下：

BIO
工作方式：BIO是一种阻塞式I/O模型。当一个线程发起一个I/O操作后，它会一直阻塞，直到这个I/O操作完成。比如，在进行网络通信读取数据时，线程调用read方法后就会等待数据，直到有足够的数据可读才会返回。
应用场景：适用于连接数少且连接时间长的场景，像数据库连接。因为它简单直接，每个连接对应一个线程，在连接数不多时可以高效处理。
性能特点：在高并发场景下效率较低，因为大量线程会被阻塞等待I/O操作，造成资源浪费。

NIO
工作方式：NIO是非阻塞式I/O模型。它基于通道（Channel）和缓冲区（Buffer）操作，线程在发起I/O操作后不会阻塞，可以去做其他事情。比如，通过轮询的方式检查通道是否可读或可写，有事件发生时再进行处理。
应用场景：适合处理高并发的网络连接场景，如网络服务器。它能够用一个或少量线程处理多个连接，有效利用系统资源。
性能特点：性能比BIO高，减少了线程阻塞等待的时间，但需要开发者自己处理轮询和事件分发等复杂操作。

AIO
工作方式：AIO是异步I/O模型。当线程发起I/O操作后，不用等待操作完成就可以继续执行其他任务，I/O完成后通过回调或其他机制通知线程。例如，在进行文件读取时，系统会在读取完成后通知线程。
应用场景：适用于需要高性能的大规模I/O操作场景，像高性能的网络服务器和文件读写服务。
性能特点：性能在理论上优于NIO，不过在实际应用中，需要考虑操作系统对AIO的支持程度等因素。

I/O多路复用
工作方式：使用select/poll/epoll等机制，实现单个线程监控多个I/O通道；
应用场景：高并发场景常用。

信号驱动I/O
工作方式：进程继续执行，I/O就绪时接收信号通知，从而减少轮询的开销。

注：Netty的I/O模型是基于NIO和事件驱动实现的，底层依赖的时NIO框架的多路复用Selector。采用epoll模式后，只需要一个线程负责Selector的轮询，当数据处于就绪状态后，需要一个事件分发器EventDispather负责将读写事件分发给对应的读写事件处理器EventHandler。而事件分发器有两种设计模式：Reactor和Proactor，前者使用同步I/O，后者使用异步I/O。
助记：区分阻塞和非阻塞，同步和非同步。是否阻塞是体现在线程是否需要阻塞地等待IO读写完成，是否同步体现在是有异步的事件回调机制，还是说当前线程通过轮询方式获取结果。
