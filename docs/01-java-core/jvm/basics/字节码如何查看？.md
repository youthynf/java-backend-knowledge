# 字节码如何查看

## 核心概念

`.class` 文件是 JVM 唯一识别的二进制字节流，由 `javac` 从 `.java` 编译而来。JVM 不关心源码语言是 Java、Kotlin 还是 Groovy，只要符合 Class 文件格式即可运行。理解字节码能帮助阅读反编译后的代码、排查框架兼容性问题、分析 JIT 行为、写字节码增强工具。

字节码查看的本质是反编译：把二进制 `.class` 转为人可读的文本格式，常用工具有 `javap`、IDE 插件（IDEA 的 jclasslib）、`ASM`、`Javassist`。

```text
源码 .java → javac → 字节码 .class → JVM 类加载 → 解释/JIT → 机器码
                                  ↑
                            反编译查看（javap / jclasslib）
```

## 标准回答

字节码是 `.class` 文件中的二进制指令，JVM 通过解析 Class 文件结构加载类。查看字节码常用 `javap -v -p ClassXxx`，能输出常量池、字段表、方法表、字节码指令。Class 文件采用伪结构存储，由无符号数和表组成，按固定顺序排列：魔数、版本号、常量池、访问标志、类索引、字段表、方法表、属性表。

要点：

1. **魔数**：`0xCAFEBABE`，标识 Class 文件。
2. **版本号**：major version，Java 8 是 52，Java 11 是 55，Java 17 是 61。
3. **常量池**：字面量和符号引用，是 Class 文件最大数据项。
4. **访问标志**：ACC_PUBLIC、ACC_FINAL、ACC_INTERFACE 等。
5. **字段表/方法表**：类的字段和方法描述。
6. **属性表**：Code 属性包含字节码指令、行号表、局部变量表。

## Class 文件结构

JVM 规范明确定义 Class 文件的二进制格式：

```text
ClassFile {
    u4             magic;                 // 魔数 0xCAFEBABE
    u2             minor_version;         // 次版本号
    u2             major_version;         // 主版本号
    u2             constant_pool_count;   // 常量池计数
    cp_info        constant_pool[];       // 常量池
    u2             access_flags;          // 访问标志
    u2             this_class;            // 当前类索引
    u2             super_class;           // 父类索引
    u2             interfaces_count;      // 接口数
    u2             interfaces[];          // 接口表
    u2             fields_count;          // 字段数
    field_info     fields[];              // 字段表
    u2             methods_count;         // 方法数
    method_info    methods[];             // 方法表
    u2             attributes_count;      // 属性数
    attribute_info attributes[];          // 属性表
}
```

| 数据类型 | 含义 |
|---------|------|
| u1 / u2 / u4 / u8 | 1/2/4/8 字节无符号数 |
| _info 结尾 | 表（如 `cp_info`、`field_info`），由若干无符号数和子表组成 |

## 反编译工具

### javap（JDK 自带）

最常用的反编译工具，输出 Class 文件的可读形式：

```bash
javap -v -p Main.class
# -v: verbose，输出常量池、字节码等完整信息
# -p: 显示 private 成员
```

### jclasslib（IDEA 插件）

IDEA 安装 jclasslib 插件，打开 `.java` 文件 → View → Show Bytecode With jclasslib，可视化查看 Class 文件结构。

### ASM / Javassist

用于字节码增强，详见 [字节码如何增强](/01-java-core/jvm/basics/字节码如何增强？.md)。

## 字段描述符

| 描述符 | 类型 |
|--------|------|
| B | byte |
| C | char |
| D | double |
| F | float |
| I | int |
| J | long |
| S | short |
| Z | boolean |
| V | void |
| L; | 对象，如 `Ljava/lang/String;` |
| [ | 数组，如 `[I` 表示 int[]，`[[Ljava/lang/String;` 表示 String[][] |

## Code 属性详解

每个非抽象方法都有 Code 属性，包含：

| 字段 | 含义 |
|------|------|
| `stack` | 最大操作数栈深度 |
| `locals` | 局部变量表大小（Slot 数） |
| `args_size` | 方法参数个数（含 this） |
| `code` | 字节码指令 |
| `LineNumberTable` | 源码行号 ↔ 字节码偏移量映射 |
| `LocalVariableTable` | 局部变量 ↔ 源码变量名映射（调试信息） |

## 常量池

常量池是 Class 文件的资源仓库，存放：

- **字面量 Literal**：文本字符串、final 常量值等。
- **符号引用 Symbolic Reference**：类/接口全限定名、字段名和描述符、方法名和描述符。

常量池项类型有 14 种（JDK 8）：

| 类型 | 含义 |
|------|------|
| Utf8 | UTF-8 字符串 |
| Integer / Float / Long / Double | 基本类型字面量 |
| Class | 类引用 |
| String | 字符串引用 |
| Fieldref | 字段引用 |
| Methodref | 方法引用 |
| InterfaceMethodref | 接口方法引用 |
| NameAndType | 名字和类型 |
| MethodHandle / MethodType / InvokeDynamic | JDK 7+ 动态语言支持 |

## 实战场景

| 场景 | 用法 | 注意点 |
|------|------|--------|
| 验证编译优化 | 看 `String.intern`、`+` 拼接的字节码 | JDK 9+ 字符串拼接用 invokedynamic |
| 排查类冲突 | 反编译两个 jar 中的同名类对比 | 同名类来自不同 jar 是常见问题 |
| 阅读框架源码 | 反编译第三方 jar 看实现 | 配合 CFR / Procyon 反编译器 |
| 学习 JVM 指令 | 写简单代码看字节码 | 推荐 IDEA 的 jclasslib 插件 |
| 兼容性检查 | 看 major version 判断 JDK 兼容性 | Java 8 是 52，Java 11 是 55，Java 17 是 61 |

## 深挖追问

### 如何判断一个 .class 文件由哪个 JDK 版本编译？

看 `major version` 字段：

| Java 版本 | major version |
|----------|---------------|
| Java 6 | 50 |
| Java 7 | 51 |
| Java 8 | 52 |
| Java 9 | 53 |
| Java 11 | 55 |
| Java 17 | 61 |
| Java 21 | 65 |

JVM 向后兼容：高版本 JVM 可以加载低版本 Class，但低版本 JVM 加载高版本 Class 会抛 `UnsupportedClassVersionError`。

### LineNumberTable 有什么用？

LineNumberTable 记录字节码偏移量到源码行号的映射，用于：

- 异常栈打印源码行号（`at com.x.Y.method(Y.java:42)`）。
- 调试器断点。
- `-g:none` 可以不生成，但异常栈会变成"unknown source"。

### 为什么 `String.intern()` 在不同 JDK 版本行为不同？

JDK 6：字符串常量池在永久代，`intern` 会把字符串复制到永久代。
JDK 7+：字符串常量池在堆中，`intern` 只存引用，不复制。所以 `new String("a").intern() == "a"` 在 JDK 7+ 可能为 true，JDK 6 一定是 false。

### invokedynamic 是什么？

JDK 7 引入的字节码指令，用于支持动态语言。`invokedynamic` 在运行时通过 bootstrap method 决定调用哪个方法。Java 8 的 Lambda、Java 9+ 的字符串拼接都用到它。详见 JVM 规范 §6.5。

## 易错点

- 把字节码当作汇编，字节码是 JVM 指令，不是 CPU 指令。
- 反编译时遗漏 `-v`，只看方法签名看不到字节码。
- 把"反编译"和"反汇编"混淆，反编译是从字节码还原 Java 源码（如 CFR），反汇编是把字节码转为人可读的指令（如 javap）。
- 以为 major version 表示小版本，实际是 JDK 大版本。
- 用 `javap` 反编译第三方 jar，需要先把 class 文件提取出来或用 `javap -classpath xxx.jar`。

## 总结

字节码查看是 JVM 调试和性能分析的基本功。`javap -v -p` 是最常用工具，能输出常量池、字段、方法、字节码指令。Class 文件由魔数、版本号、常量池、访问标志、类索引、字段表、方法表、属性表组成。理解字段描述符、Code 属性、常量池是阅读反编译输出的基础。借助 IDEA 的 jclasslib 插件可以更直观地查看。
