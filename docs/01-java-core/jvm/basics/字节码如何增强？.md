# 字节码如何增强

## 核心概念

字节码增强是修改或动态生成 `.class` 字节码的技术。它解决的问题是：在不修改源码、不重启应用的前提下，给类增加日志、监控、AOP、Mock 等能力。Spring AOP、MyBatis Mapper、Mockito、Arthas、SkyWalking 等都依赖字节码增强。

字节码增强的本质是：通过 ASM、Javassist、ByteBuddy 等库读写 `.class` 字节流，借助 Instrumentation API 在类加载时或运行时替换字节码，让 JVM 加载增强后的版本。

```text
源码 → javac → .class → 增强工具（ASM/Javassist/ByteBuddy）
                                ↓
                          Instrumentation API
                                ↓
              类加载时（premain）/ 运行时（agentmain）
                                ↓
                          JVM 加载增强后的字节码
```

## 标准回答

字节码增强是修改或动态生成 `.class` 字节码的技术，常用工具包括 ASM、Javassist、ByteBuddy。增强的入口有两个：JVM 启动时通过 `-javaagent` 挂载（premain），运行时通过 Attach API 挂载（agentmain）。实现机制是 `Instrumentation` API + `ClassFileTransformer`，在类加载或重定义时返回增强后的字节码。

要点：

1. **ASM**：底层指令级 API，性能最好，编程难度最高。CGLIB 基于 ASM。
2. **Javassist**：源码级 API，编程简单，性能略差。
3. **ByteBuddy**：现代 API，语法友好，性能接近 ASM。
4. **Instrumentation**：JVM 提供的字节码修改入口，支持启动时和运行时增强。
5. **应用场景**：AOP、监控、Mock、热部署、性能诊断（Arthas、bTrace）。

## 主流字节码增强工具对比

| 工具 | 抽象层级 | 性能 | 易用性 | 典型应用 |
|------|---------|------|--------|---------|
| ASM | 指令级 | 最高 | 难 | CGLIB、Spring AOP |
| Javassist | 源码级 | 中 | 易 | Hibernate、MyBatis |
| ByteBuddy | DSL | 高 | 友好 | Mockito、SkyWalking Agent |

## ASM

### 核心 API（流式，类似 SAX）

不需要把整个 Class 读取到内存，流式处理，节省内存，编程难度大。

关键类：

| 类 | 作用 |
|----|------|
| `ClassReader` | 读取已编译的 `.class` |
| `ClassWriter` | 生成或修改 Class 字节码 |
| `ClassVisitor` | 访问类结构 |
| `MethodVisitor` | 访问方法 |
| `FieldVisitor` | 访问字段 |

### 树形 API（DOM 风格）

把整个 Class 读到内存树状结构，编程简单，内存消耗大。

### ASM 实现方法前后打印日志

目标：在 `Base.process()` 方法前后打印 "start" 和 "end"。

```java
import org.objectweb.asm.*;

public class BaseClassVisitor extends ClassVisitor {
    public BaseClassVisitor(ClassVisitor cv) {
        super(Opcodes.ASM9, cv);
    }

    @Override
    public MethodVisitor visitMethod(int access, String name, String desc,
                                     String signature, String[] exceptions) {
        MethodVisitor mv = super.visitMethod(access, name, desc, signature, exceptions);
        if (!name.equals("<init>") && mv != null) {
            mv = new MethodTimerVisitor(mv);
        }
        return mv;
    }

    static class MethodTimerVisitor extends MethodVisitor {
        public MethodTimerVisitor(MethodVisitor mv) {
            super(Opcodes.ASM9, mv);
        }

        @Override
        public void visitCode() {
            // 方法开头插入：System.out.println("start");
            mv.visitFieldInsn(Opcodes.GETSTATIC, "java/lang/System", "out", "Ljava/io/PrintStream;");
            mv.visitLdcInsn("start");
            mv.visitMethodInsn(Opcodes.INVOKEVIRTUAL, "java/io/PrintStream", "println", "(Ljava/lang/String;)V", false);
            super.visitCode();
        }

        @Override
        public void visitInsn(int opcode) {
            // 在 return / athrow 之前插入：System.out.println("end");
            if ((opcode >= Opcodes.IRETURN && opcode <= Opcodes.RETURN) || opcode == Opcodes.ATHROW) {
                mv.visitFieldInsn(Opcodes.GETSTATIC, "java/lang/System", "out", "Ljava/io/PrintStream;");
                mv.visitLdcInsn("end");
                mv.visitMethodInsn(Opcodes.INVOKEVIRTUAL, "java/io/PrintStream", "println", "(Ljava/lang/String;)V", false);
            }
            mv.visitInsn(opcode);
        }
    }
}
```

使用：

```java
ClassReader cr = new ClassReader("com.example.Base");
ClassWriter cw = new ClassWriter(ClassWriter.COMPUTE_MAXS);
BaseClassVisitor cv = new BaseClassVisitor(cw);
cr.accept(cv, ClassReader.SKIP_DEBUG);
byte[] enhancedBytes = cw.toByteArray();
// 写回 .class 或通过 Instrumentation 加载
```

## Javassist

源码级 API，直接传 Java 代码字符串，无需了解字节码指令。

核心类：

| 类 | 作用 |
|----|------|
| `ClassPool` | 类容器，按类名获取 `CtClass` |
| `CtClass` | 编译时类信息 |
| `CtMethod` | 类的方法 |
| `CtField` | 类的字段 |

```java
import javassist.*;

public class JavassistDemo {
    public static void main(String[] args) throws Exception {
        ClassPool pool = ClassPool.getDefault();
        CtClass cc = pool.get("com.example.Base");
        CtMethod m = cc.getDeclaredMethod("process");
        m.insertBefore("{ System.out.println(\"start\"); }");
        m.insertAfter("{ System.out.println(\"end\"); }");

        Class<?> enhancedClass = cc.toClass();
        Object obj = enhancedClass.getDeclaredConstructor().newInstance();
        enhancedClass.getMethod("process").invoke(obj);
    }
}
```

## ByteBuddy

现代 DSL API，语法友好。

```java
import net.bytebuddy.ByteBuddy;
import net.bytebuddy.implementation.MethodDelegation;
import static net.bytebuddy.matcher.ElementMatchers.named;

Class<?> enhanced = new ByteBuddy()
    .subclass(Base.class)
    .method(named("process"))
    .intercept(MethodDelegation.to(new LoggingInterceptor()))
    .make()
    .load(JavassistDemo.class.getClassLoader())
    .getLoaded();

public class LoggingInterceptor {
    @RuntimeType
    public Object intercept(@Origin Method method,
                            @SuperCall Callable<?> callable) throws Exception {
        System.out.println("start " + method.getName());
        try {
            return callable.call();
        } finally {
            System.out.println("end " + method.getName());
        }
    }
}
```

## 运行时类重载：Instrumentation API

### 启动时增强（premain）

通过 `-javaagent:xxx.jar` 在 JVM 启动时挂载 Agent，在类加载时修改字节码。

```java
public class PreMainAgent {
    public static void premain(String args, Instrumentation inst) {
        inst.addTransformer(new MyTransformer());
    }
}

public class MyTransformer implements ClassFileTransformer {
    @Override
    public byte[] transform(ClassLoader loader, String className,
                            Class<?> classBeingRedefined,
                            ProtectionDomain protectionDomain,
                            byte[] classfileBuffer) {
        if (!className.equals("com/example/Base")) return null;
        // 用 ASM/Javassist 修改 classfileBuffer 并返回
        return enhancedBytes;
    }
}
```

`META-INF/MANIFEST.MF`：

```text
Manifest-Version: 1.0
Premain-Class: com.example.PreMainAgent
Can-Redefine-Classes: true
Can-Retransform-Classes: true
```

启动：

```bash
java -javaagent:agent.jar -jar app.jar
```

### 运行时增强（agentmain）

通过 Attach API 在运行时把 Agent 挂到目标 JVM：

```java
import com.sun.tools.attach.VirtualMachine;

public class Attacher {
    public static void main(String[] args) throws Exception {
        VirtualMachine vm = VirtualMachine.attach("12345"); // 目标 JVM pid
        vm.loadAgent("/path/to/agent.jar");
    }
}
```

Agent 类的 `agentmain` 方法：

```java
public class AgentMainAgent {
    public static void agentmain(String args, Instrumentation inst) {
        inst.addTransformer(new MyTransformer(), true);
        inst.retransformClasses(Base.class);  // 重定义已加载的类
    }
}
```

`MANIFEST.MF`：

```text
Manifest-Version: 1.0
Agent-Class: com.example.AgentMainAgent
Can-Redefine-Classes: true
Can-Retransform-Classes: true
```

## 实战场景

| 场景 | 工具 | 用法 |
|------|------|------|
| Spring AOP | CGLIB（基于 ASM） | 给无接口类生成代理子类，织入切面 |
| MyBatis Mapper | Javassist /动态代理 | 接口方法 → SQL 调用 |
| Mockito | ByteBuddy | mock 类的任意方法 |
| Arthas | ByteBuddy + Instrumentation | 在线诊断，反编译、watch、trace |
| SkyWalking Agent | ByteBuddy | 自动埋点，方法前后注入 trace 逻辑 |
| 热部署 | Instrumentation | 修改已加载类，立即生效 |
| Mock 测试 | ByteBuddy / Mockito | 替换方法实现 |

## 深挖追问

### ASM、Javassist、ByteBuddy 怎么选？

- 性能优先：ASM（CGLIB、Spring 都用它）。
- 简单优先：Javassist（源码级，无需了解字节码指令）。
- 现代项目：ByteBuddy（DSL 友好，文档好，Mockito、SkyWalking 都用它）。
- 学习目的：从 ASM 入手，理解字节码指令后用其他库都得心应手。

### premain 和 agentmain 的区别？

- premain：JVM 启动时通过 `-javaagent` 挂载，在类加载时增强。
- agentmain：运行时通过 Attach API 挂载，可以重定义已加载的类（`retransformClasses`）。
- Arthas 的 `attach` 用的是 agentmain，所以能诊断线上已启动的应用。

### 字节码增强有什么风险？

- 性能开销：每次类加载都要走 transformer。
- 兼容性：JDK 升级可能导致 ASM 不兼容（如 ASM 9 才支持 JDK 17 的 class file）。
- 安全：增强代码出错可能让应用崩溃。
- 调试：增强后的代码反编译可能看不到原始逻辑，排查难度大。
- 在生产环境慎用 `retransformClasses`，会触发 JIT 反优化。

### CGLIB 为什么不能代理 final 类和 final 方法？

CGLIB 通过生成子类实现代理，final 类不能被继承、final 方法不能被覆盖。JDK 动态代理只能代理接口，CGLIB 通过子类继承绕过接口限制，但仍受 final 约束。

## 易错点

- 以为 `Instrumentation` 只能用于字节码增强，实际还能查内存、查类加载等。
- 在 `transform` 方法中抛异常会导致类加载失败，应用启动不了。
- 多个 Agent 同时挂载时，transformer 顺序敏感。
- 忘记在 `MANIFEST.MF` 配置 `Can-Retransform-Classes: true`，运行时重定义会失败。
- 增强后的类与原类签名不一致，导致 `LinkageError`。

## 总结

字节码增强是 AOP、监控、Mock、热部署、性能诊断的底层技术。ASM 性能最好但难用，Javassist 源码级简单，ByteBuddy 现代 DSL 友好。Instrumentation API 是 JVM 提供的入口，支持 premain（启动时）和 agentmain（运行时）两种挂载方式。Arthas、SkyWalking、Mockito 等主流工具都依赖这套机制。理解字节码增强能帮助阅读框架源码、做线上诊断、写自定义 Agent。
