# 符号引用与直接引用怎么理解？

符号引用与直接引用怎么理解？
在Java虚拟机(JVM)中，符号引用(Symbolic References)和直接引用(Direct References)是类加载和动态连接过程中的重要概念，它们描述了方法调用和字段访问的不同阶段。

符号引用(Symbolic References)
符号引用是一种与虚拟机实现无关的引用形式，它使用一组符号来描述所引用的目标。符号引用可以包括：
•  类和接口的全限定名
•  字段的名称和描述符
•  方法的名称和描述符

特点：
•  符号引用与虚拟机的内存布局无关；
•  引用的目标不一定已经加载到内存中；
•  在编译阶段生成，存储在class文件的常量池中；

示例：
•  在Java代码中调用java.lang.Object.toString()方法时，编译后的class文件中会存储类似"java/lang/Object"、"toString"、"()Ljava/lang/String;"这样的符号引用。

直接引用(Direct References)
直接引用是指能够直接指向目标的指针、相对偏移量或能间接定位到目标的句柄。

特点：
•  直接引用与虚拟机的内存布局相关；
•  引用的目标必须已经存在于内存中；
•  在类加载过程(解析阶段)或第一次使用时转换得到

示例：
•  当一个类被加载后，其方法在方法区有确定的位置，直接引用可能就是这个方法的入口地址。

转换过程
符号引用到直接引用的转换发生在解析阶段，这是类加载过程中的一个步骤：
加载阶段：读取class文件，创建Class对象，此时各种引用仍以符号形式存在；
验证阶段：确保符号引用的正确性；
准备阶段：为类变量分配内存并设置初始值；
解析阶段：将常量池内的符号引用替换为直接引用；
初始化阶段：执行类构造器<clinit>()方法；

延迟解析
JVM不要求解析阶段一定在类加载时就完成，有些解析可以在符号引用第一次被使用时才进行(动态连接)。这体现了Java的"运行时绑定"特性。

具体应用场景
1. 方法调用：
•  静态方法、私有方法等在编译期可知，可以在类加载时解析(静态绑定)；
•  虚方法(实例方法)需要在运行时根据实际类型解析(动态绑定)；

字段访问：
•  静态字段在类加载时解析；
•  实例字段在对象创建后通过对象头中的类型指针解析；

异常情况
在解析阶段，如果找不到对应的类、字段或方法，JVM会抛出各种异常：
•  NoClassDefFoundError
•  NoSuchFieldError
•  NoSuchMethodError
