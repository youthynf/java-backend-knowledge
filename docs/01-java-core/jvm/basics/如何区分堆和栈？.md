# 如何区分堆和栈？

如何区分堆和栈？
用途：栈主要存储局部变量、方法调用的参数、方法返回地址以及一些临时数据。每当一个方法被调用，一个栈帧就会在栈中创建，用于存储该方法的信息，当方法执行完毕，栈帧也会被移除。堆用于存放对象的实例，包括类的实例和数组，当你使用new关键字创建一个对象时，对象的实例就会在对上分配空间；
生命周期：栈中的数据具有确定的生命周期，当一个方法调用结束，其对应的栈帧过就会被销毁，栈中存储的局部变量也会随之消失；堆中的对象生命周期不确定，对象会在垃圾回收机制检测到不再引用时才被回收；
存取速度：栈的存取速度会更快，遵循先进先出原则，操作简单；堆的存取速度相对较慢，因为对象在对上的分配和空间回收需要更多的时间，而且垃圾回收机制的运行也会影响性能；
存储空间：栈的空间相对较小，且固定，有操作系统管理，当栈溢出时，通常是因为递归过深或局部变量过大；堆的空间相对较大，动态扩展，由JVM管理，堆溢出通常是由于创建了太多的大对象或未能及时回收不再使用的对象；
可见性：栈中的数据对线程私有，每个线程有自己的占空间；对重的数据对线程共享，所有线程都能访问对上的对象；

# 面试复习补充

## 核心概念补充

这篇文章的主题是 **如何区分堆和栈？**。复习时应先给出定义，再说明它在 Java 并发或 JVM 体系中的位置，最后结合使用场景、限制条件和常见误区展开。

## 面试官想考什么

- 是否能用自己的话讲清楚概念，而不是只背术语。
- 是否理解底层机制、关键流程以及它和相邻知识点的区别。
- 是否能把知识点落到真实项目：如何使用、如何排查、如何调优、什么时候不该用。
- 是否知道常见坑点，例如线程安全、可见性、阻塞、内存泄漏、GC 停顿或参数误用。

## 标准回答

JVM 基础题要把运行时数据区、类加载、对象模型和执行引擎串起来。回答时要区分线程私有与线程共享区域、对象和引用、堆内与堆外、类元数据与实例数据，并结合 JDK 版本差异谨慎说明。

## 深挖追问

- 堆、栈、方法区/元空间、本地内存分别存放什么？
- 对象创建、初始化、使用、回收的完整链路是什么？
- 类加载的加载、验证、准备、解析、初始化分别做什么？
