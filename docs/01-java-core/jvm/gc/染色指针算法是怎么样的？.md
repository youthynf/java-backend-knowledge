# 染色指针算法是怎么样的？

染色指针算法是怎么样的？
概述
。在ZGC出现之前，GC信息被保存在对象头的Mark Word当中，64位的MarkWord中保存了GC和对象相关的信息，包括对象的哈希码、分代年龄、锁标志等。由于这些信息存在于对象头的MarkWord中，而对象实体一般存储在堆内存中，虚拟机栈帧中的局部变量表中存储的则是堆中对象的引用（指针）。因此，当访问对象的时候获取这些信息是很方便的，但是如果我们希望不直接访问对象，仅仅通过对象的指针就得到这些信息的话，这种实现方式就做不到。而染色指针（Colored Pointers）是ZGC（Z Garbage Collector）和Shenandoah等现代垃圾收集器使用的核心内存管理技术，它通过复用对象指针的冗余比特位来存储元数据（如标记状态、内存区域信息等），从而实现以下目标：
•  并发标记与整理：无需STW暂停即可完成对象标记和移动。
•  低延迟：将大部分GC操作从STW转为并发执行，实现亚毫秒级停顿。
•  内存效率：避免额外内存开销（如G1的卡表、SATB缓冲区）。

基本原理
ZGC将GC信息保存在了染色指针上，在64位JVM中，对象指针是64位，高16位都是0，暂时不用来寻址，剩下48位支持的内存可以达到256TB，不过ZGC并没有把48位都用来保存对象信息，而是用高4位保存了4个标志位，这样ZGC可以管理的最大内存可以达到16TB（2^44）。通过这四个标志位，JVM可以从指针上直接看到对象的三色标记状态（Marked0，Marked1）、是否进入了重分配集（Remapped）、是否需要通过finalize方法来方位（Finalizable）。无需进行对象方位就可以获得GC信息，大大提高了GC效率。
•  Marked0/Marked1：交替用于标记周期；
•  Remapped：表示指针已重映射；
•  Finalizable：用于终结器处理；

染色指针实质上是三色标记的物理实现：
•  白色：指针元数据高4位全0；
•  灰色：Marked0/1标志位置1，但对象在标记栈中；
•  黑色：Marked0/1标志位置1，且对象不在标记栈；

优势与挑战
优势：
•  极低停顿时间（亚毫秒级）
•  内存开销小（元数据内联在指针中）
•  并发执行效率高

挑战：
•  需要特定硬件支持（64位+地址空间充足）
•  指针操作变得复杂
•  调试难度增加

染色指针通过将GC状态编码到指针本身，实现了真正并发的垃圾回收，是新一代垃圾回收器的核心技术。ZGC借助该技术在TB级堆内存下仍能保持<1ms的停顿时间。
