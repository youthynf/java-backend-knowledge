# SATB算法是什么？

SATB算法是什么？
概述
SATB（Snapshot-At-The-Beginning）是G1垃圾收集器（Garbage-First）用于并发标记阶段的核心算法，旨在解决并发标记过程中对象引用变化导致的漏标问题。它通过维护“标记开始时的对象图快照”，确保垃圾回收的准确性。
核心问题
SATB算法主要用于解决并发标记的漏标问题，防止对象被错误回收，但是允许浮动垃圾的产生。在并发标记阶段，如果发生某个引用被删除，然后引用重新被赋值被忽略时，由于标记线程没有感知新增的引用，可能导致误判该引用的对象为垃圾，进行错误回收。
核心思想
SATB通过保留标记开始时的对象引用快照，确保所有在快照中存活的对象都会被标记为存活，即使引用后续被删除；
•  快照内容：记录并发标记开始时所有存活对象的引用关系；
•  处理新增引用：通过写屏障捕获所有新建立的引用。

实现机制
4.1 三色标记法
STATB基于三色标记法，对象分为白色（未标记，可能垃圾）、灰色（已标记，但子引用未扫描完）、黑色（已标记，且所有子引用全部扫描完成）。SATB保证所有在标记开始时能够通过引用链访问到的对象（即快照中的对象），最终都会被标记为黑色和灰色，不会被错误回收。
4.2 写屏障
在并发标记阶段，所有对象引用更新会触发写屏障，记录可能丢失的引用：
•  引用删除时保存旧值：原本引用的对象会被加入SATB缓冲区中，并强制标记为存活，并且会遍历其子引用关系；
•  处理新增引用：新引用通过常规标记流程处理，将对象标记为存活；

4.3 SATB缓冲区
每个线程有一个固定大小的SATB缓冲区（如256个条目），存储被删除的旧引用。当该缓冲区满时，线程会将其提交到全局队列，由标记线程处理。
SATB工作流程
•  初始标记：需要STW，标记GC Roots直接引用的对象（灰色），此时快速建立快照，即记录所有存活对象的初始状态。
•  并发标记：与应用线程并发执行，从灰色对象出发遍历引用链。同时使用写屏障拦截所有引用变更，保存被删除的引用存入SATB缓冲区中，新引用则通过常规标记为存活对象；
•  最终标记：需要STW，处理SATB缓冲区中剩余的对象，将对象标记为灰色，并扫描对象所有的引用字段，将其子对象加入标记栈，等待后续标记，更新对象标记为黑色，最终确保快照中所有对象被标记。
• 清理：回收未被标记的白色对象。
SATB优缺点
优点：通过快照机制，确保标记开始时存货对象不会被误回收，避免漏标；支撑并发标记实现，减少STW停顿时间，实现低延迟；
缺点：存在浮动垃圾，如标记后被断开的引用，需要等下次回收；同时写屏障开销较大，每个引用变更都需要额外操作，轻微影响吞吐量；
7、总结
SATB（Snapshot-At-The-Beginning）算法虽然通过快照机制避免了漏标（存活对象被误回收），但确实会产生浮动垃圾（Floating Garbage）。浮动垃圾是指在垃圾回收周期中，已经被标记为存活但实际上不再被引用的对象。这些对象会在下一次GC时被回收，导致本次GC的堆内存无法完全释放。写入SATB缓冲区的对象并不会被直接处理为“未存活”，而是会被强制标记为存活（即使它们实际上可能已经变成垃圾）。这是SATB算法的核心设计，目的是避免漏标（Missed Mark），确保所有在标记开始时存活的对象不会被错误回收。

### 核心回答

1. GC 的核心目标是在停顿时间、吞吐量和内存占用之间做权衡。
2. 判断对象是否可回收要理解可达性分析、引用类型、分代假设和具体收集器机制。
3. G1/ZGC 等现代收集器更强调可预测停顿，但仍需要根据对象分配速率和堆布局调优。
