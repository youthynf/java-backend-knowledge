# 浮动垃圾是指什么？

浮动垃圾是指什么？
CMS浮动垃圾产生：
CMS在第四阶段清除垃圾时时并发清除的，这时候，垃圾回收线程和用户线程并发执行，这个阶段产生的新的垃圾对象就是浮动垃圾。CMS垃圾收集器会预留一部分内存空间用于存放浮动垃圾，如空间达到70-80%就开始进行垃圾回收。
G1没有浮动垃圾：
G1的筛选回收是多个垃圾回收线程并行GC的，没有浮动垃圾的回收，G1 的标记阶段是并发的，和 CMS 类似。但是，G1 在标记阶段结束后，并不会立刻开始清理阶段。而是会在年轻代回收或者混合回收阶段，根据标记结果，同时清理在标记阶段新产生的垃圾。

Remembered Set（记忆集）的作用：
G1 为每个区域都维护了一个记忆集。记忆集记录了该区域中对象的引用情况，包括外部区域对本区域对象的引用。在标记阶段，通过记忆集可以快速地判断哪些对象是存活的，哪些是垃圾。在回收阶段，当对某个区域进行回收时，会根据记忆集再次确认对象的存活状态，这样就能把在标记阶段后新产生的垃圾一并清理掉。
