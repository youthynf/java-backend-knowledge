# 三色标记是什么？

三色标记是什么？
三色标记(Tri-color Marking)是垃圾回收中用于并发标记的一种经典算法，它通过颜色标记对象的状态，解决了并发标记过程中对象引用关系变化的问题。
一、基本概念
三种颜色含义
•  白色(White)：未被访问的对象（默认状态）
•  灰色(Gray)：已被标记但引用的对象还未完全扫描
•  黑色(Black)：已被完全标记（对象及其引用都已被标记）

算法核心思想
通过颜色变化模拟对象标记过程：白色 → 灰色 → 黑色

二、标记过程详解
初始阶段

// 伪代码表示
for (Object root : GC_ROOTS) {
   if (root.color == WHITE) {
       root.color = GRAY;  // 根对象变灰
   }
}

标记阶段

while 灰色对象不为空:
   取出一个灰色对象obj
   for field in obj的所有引用字段:
       if field.color == WHITE:
           field.color = GRAY  # 将引用对象标记为灰
   obj.color = BLACK  # 当前对象标记完成

完成条件
•  没有灰色对象剩余时，标记完成
•  剩余白色对象即为可回收垃圾

三、并发标记的挑战与解决方案
并发修改问题
场景：标记线程和用户线程并发运行时，可能出现：
•  已标记对象(黑)引用新创建的白对象
•  灰色对象引用的白对象被删除

解决方案：写屏障(Write Barrier)

// 写屏障伪代码
void write_barrier(Object src, Object field, Object new_value) {
   if (src.color == BLACK && new_value.color == WHITE) {
       new_value.color = GRAY;  // 破坏强三色不变式
   }
   src.field = new_value;  // 实际写操作
}

不变式保证
•  强三色不变式：黑对象不能直接引用白对象
•  弱三色不变式：白对象只能被灰对象引用

四、在三代GC中的具体实现
G1垃圾回收器
•  初始标记（STW）→ 并发标记 → 最终标记（STW）→存活对象统计
•  使用全局标记栈记录灰色对象
•  通过SATB(Snapshot-At-The-Beginning)算法处理并发修改

ZGC实现

// ZGC着色指针标记示例
void mark(Object obj) {
   if (!is_marked(obj)) {
       set_marked(obj);  // 使用指针元数据标记
       push_to_mark_stack(obj);  // 加入灰色集合
   }
}
•  将颜色信息编码到指针元数据中
•  每个线程有本地标记栈减少竞争

五、性能优化技术
并行标记

# 多线程标记示例
def worker(mark_stack):
   while not mark_stack.empty():
       obj = mark_stack.pop()
       for ref in obj.references():
           if ref.color == WHITE:
               ref.color = GRAY
               mark_stack.push(ref)
       obj.color = BLACK

增量标记
•  将标记过程分成多个小步骤
•  与用户线程交替执行

记忆集(Remembered Set)
•  记录跨代引用
•  减少全堆扫描范围

六、总结
三色标记算法通过颜色状态管理，实现了垃圾回收的并发执行，是现代GC算法的基石。不同垃圾回收器会根据具体需求对基础算法进行优化，如G1的SATB、ZGC的着色指针等，但核心思想都源于三色标记模型。
助记：三色标记方法需要构建一个在垃圾回收开始时的内存快照，后续在这个快照上发生的一起修改都要记录下来。垃圾回收开始之后，写屏障就生效，初始快照之后的修改都会被放到STAB队列里面等待最终标记来处理。

### 核心回答

1. GC 的核心目标是在停顿时间、吞吐量和内存占用之间做权衡。
2. 判断对象是否可回收要理解可达性分析、引用类型、分代假设和具体收集器机制。
3. G1/ZGC 等现代收集器更强调可预测停顿，但仍需要根据对象分配速率和堆布局调优。

- **面试表达**：用“现象 → 原理 → 工具验证 → 解决方案 → 风险边界”的链路回答。

- 不要只看平均停顿，要看 P99 和业务尖峰时段。
- 调优前先定位问题，别上来就改 Xmx、G1 参数。
