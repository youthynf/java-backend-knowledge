# JVM (Java 虚拟机)

## 核心概念

### JVM 内存模型
- **程序计数器**：线程私有，记录字节码执行位置
- **虚拟机栈**：线程私有，每个方法创建一个栈帧（局部变量表、操作数栈、动态链接、返回地址）
- **本地方法栈**：为 Native 方法服务
- **堆**：线程共享，存放对象实例，GC 主要区域
  - 新生代（Eden + Survivor0 + Survivor1）
  - 老年代
- **方法区**：存储类信息、常量、静态变量（JDK8 后用元空间实现）

### 垃圾回收算法
- **标记-清除**：效率低，产生内存碎片
- **复制算法**：新生代采用，Eden:Survivor = 8:1:1
- **标记-整理**：老年代采用，移动存活对象到一端
- **分代收集**：根据对象存活周期选择不同算法

### 垃圾收集器
- **Serial**：单线程，简单高效，适合客户端
- **Parallel**：多线程，关注吞吐量
- **CMS**：并发收集，低停顿，已废弃
- **G1**：区域化分代，可预测停顿时间，JDK9 默认
- **ZGC**：低延迟（<10ms），JDK15+，适合大内存

### 类加载机制
- **双亲委派模型**：
  - Bootstrap ClassLoader（核心类库）
  - Extension ClassLoader（扩展类库）
  - Application ClassLoader（应用类路径）
- **为什么要双亲委派**：避免类重复加载、保护核心类安全
- **如何打破**：自定义 ClassLoader 重写 loadClass()

## 面试高频问题

### 1. JVM 内存模型有哪些区域？各有什么作用？

**回答要点**：
- 线程私有：程序计数器、虚拟机栈、本地方法栈
- 线程共享：堆、方法区
- 各区域职责和特点

### 2. 垃圾回收算法有哪些？各有什么优缺点？

**回答要点**：
- 标记-清除：简单但有碎片
- 复制算法：无碎片但空间利用率低
- 标记-整理：无碎片但效率较低
- 分代收集：综合方案

### 3. G1 垃圾收集器的工作原理？

**回答要点**：
- Region 分区设计
- 可预测停顿时间
- 并发标记-整理
- 适合大内存场景

### 4. 类加载过程是怎样的？双亲委派模型有什么好处？

**回答要点**：
- 加载 → 验证 → 准备 → 解析 → 初始化
- 双亲委派保证核心类安全
- 避免类重复加载

## 实战场景

### JVM 调优

**常用参数**：
```
-Xms4g                    # 初始堆大小
-Xmx4g                    # 最大堆大小
-Xmn2g                    # 新生代大小
-XX:+UseG1GC              # 使用 G1 收集器
-XX:MaxGCPauseMillis=200  # 最大 GC 停顿时间
-XX:+HeapDumpOnOutOfMemoryError  # OOM 时生成堆转储
```

**故障排查工具**：
- `jstat`：查看 GC 统计
- `jmap`：生成堆转储
- `jstack`：查看线程栈
- `jvisualvm`：可视化监控
- `Arthas`：在线诊断工具

### OOM 排查流程
1. 分析 heap dump
2. 找出大对象
3. 定位代码位置
4. 优化或修复
