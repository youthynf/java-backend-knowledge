# 基础使用

基础使用
准备工作：创建独立的 Python 环境
以ZVT项目安装为例，由于 ZVT 依赖较多（如 Pandas, SQLAlchemy, PySide6 等），为了避免污染你系统的全局环境，必须使用虚拟环境。

# 1. 确保你有 Python 3.9 或 3.10 环境（ZVT 对这两版本兼容性最好）
python --version

# 2. 创建一个名为 zvt_env 的虚拟环境# Linux / macOS / Windows 均可
python -m venv zvt_env

# 3. 激活虚拟环境# Windows:
zvt_env\Scripts\activate
# Linux / macOS:source zvt_env/bin/activate
常见问题：.\zvt_env\Scripts\activate : 无法加载文件E:\Projects\python_project\zvt\zvt_env\Scripts\Activate.ps1
这是 Windows PowerShell 的一个典型安全限制。默认情况下，PowerShell 为了防止恶意脚本运行，禁止执行任何未签名的 .ps1 脚本（包括虚拟环境的激活脚本）。
方案一：在当前窗口临时解除限制（最推荐）
如果你不想改变全局系统设置，只需在当前的 PowerShell 窗口运行这一行命令，即可获得本次会话的执行权限：

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process
运行完后，再次执行：

.\zvt_env\Scripts\Activate.ps1
方案二：永久解除当前用户的限制
如果你希望以后创建其他项目也能顺畅激活，可以针对当前系统用户永久开启：
以管理员身份运行 PowerShell（右键点击 PowerShell 图标 -> 以管理员身份运行）。
输入以下命令并回车：
PowerShell

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
系统会询问是否确认，输入 Y 然后回车。
回到你的项目窗口，重新运行激活命令即可。
方案三：如果你使用的是 CMD（命令提示符）
如果你在用传统的 CMD 窗口（黑色背景），而不是 PowerShell（蓝色背景），激活命令是不一样的：

zvt_env\Scripts\activate.bat
注意：CMD 不需要设置 ExecutionPolicy，直接运行即可。

配置 pip 镜像加速 (国内环境必备)
ZVT 需要下载大量重型科学计算库，直接连接 PyPI 很容易超时。

pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

安装 ZVT 及核心组件
ZVT 支持模块化安装。既然你追求“功能完备”，建议直接安装 [all] 模式。

# 1. 升级 pip
pip install --upgrade pip

# 2. 安装 ZVT 核心库及所有扩展（包括可视化、数据抓取等）
pip install "zvt[all]"# 3. (可选) 安装 PostgreSQL 驱动（如果你之后按建议使用 PG 数据库）
pip install psycopg2-binary
常见错误：

ERROR: Could not install packages due to an OSError: [WinError 5] 拒绝访问。: 'e:\\projects\\python_project\\zvt\\zvt_env\\scripts\\pip.exe'
Check the permissions
因为 pip 正在运行，它尝试替换掉正在使用的 pip.exe，由于文件锁或权限问题，Windows 拒绝了这一操作，导致旧的 pip 卸载了，新的没装上。
作为后端开发，建议养成使用 python -m 前缀的习惯，这能有效避开很多 Windows 环境下的路径和权限坑。
请依次执行以下命令：强制修复并安装 pip，不要直接运行 pip install，使用 python 解释器来调度：

python -m pip install --upgrade pip -i https://pypi.tuna.tsinghua.edu.cn/simple
Python 自带了一个名为 ensurepip 的安全模块，专门用来恢复被误删的 pip。
在你的虚拟环境目录下运行：
PowerShell

python -m ensurepip --upgrade
