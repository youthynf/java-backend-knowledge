# MVC分层理解

MVC分层理解
MVC全名是Model View Controller，模型(Model)-视图(View)-控制器(Controller)的缩写，是一种软件设计典范，用一种业务逻辑、数据、界面显示分离的方式组织代码，达到分层解耦的目的。
·  模型(Model)：主要用来承载数据，也可以带有逻辑，对用户提交的请求进行计算的模块。模型也分为数据承载Bean和业务处理Bean两类，前者指实体类，如User类，后者指Service或Dao对象，专门用于处理用户提交的请求和数据获取处理；
·  视图(View)：为用户提供使用界面，与用户直接交互；
·  控制器(Controller)：用于将用户请求转发给响应的Model进行处理，并根据Model计算的结果向用户提供响应。它实现视图和模型的分离。
