# SpringMVC的工作流程

SpringMVC的工作流程
一、SpringMVC的工作机制中的核心类
·  DispatcherServlet：它是Spring MVC的前端控制器，是整个框架的核心入口点，所有的 Web 请求首先都会到达这个类进行统一处理。就如同一个交通枢纽，负责接收来自客户端（如浏览器）的各种 HTTP 请求，并根据配置好的规则将这些请求分发给相应的后端处理器进行处理，最后再把处理结果返回给客户端，起到了请求的调度和协调作用。
·  HandlerMapping：它是处理器映射器，主要负责将传入的请求与对应的控制器中的处理器方法进行映射匹配，也就是要找出在众多的控制器类和方法中，到底哪个方法应该处理当前接收到的这个请求，起到了请求到具体业务逻辑处理方法的 “导航” 作用
·  HandlerAdapter：它是处理器适配器，起到了一个桥梁的作用，负责将DispatcherServlet传来的请求适配到对应的处理器方法上，使得DispatcherServlet能够顺利调用具体的处理器方法进行业务逻辑处理，并且它还能处理方法执行后的一些善后工作，比如对返回结果进行相应的包装、转换等处理。
·  Handler：它是执行处理器Controller，也叫页面控制器；
·  ViewReslover：它是视图解析器，用于把处理器方法返回的视图名称（抽象的表示形式）转化为具体的视图对象（比如一个 JSP 页面、一个 HTML 模板等具体的可以用于渲染生成响应内容的形式），在整个请求处理链中，处在将业务逻辑处理结果转化为最终客户端可看到的响应内容的关键环节。

二、SpringMVC的工作流程
用户发送请求至前端控制器DispatcherServlet；
DispatcherServlet收到请求后调用处理器映射器HandlerMapping，查找Handler；
HandlerMapping根据请求的url找到具体的处理器Handler，并生成处理器执行链对象HandlerExecutionChain(包含Handler类型的处理器对象和HandlerInterceptor 类型的拦截器数组)一并返回给DispatcherServlet；
DispatcherServlet根据处理器Handler获取处理器适配器HandlerAdapter执行适配处理一系列操作，如参数封装、数据格式转换、数据验证等；
5. 执行处理器Handler逻辑（Controller，也叫页面控制器）；
Handler执行完成后返回ModelAndView类型对象给HandlerAdapter；
HandlerAdapter将Handler执行结果ModelAndView返回给DispatcherServlet；
DispatcherServlet将ModelAndView传给VidwReslover视图解析器；
ViewReslover解析后返回View给DispatcherServlet；
DispatcherServlet对View进行渲染视图，即将模型数据model填充至视图中；
最后DispatcherServlet响应用户。
助记：请求->DispatcherServlet->查找handler->找到对应的HandlerAdapter执行适配处理->执行handler->返回ModelAndView->传给ViewReslover视图解析器解析->dispatcherServlet渲染，并返回客户端。
