# Spring Bean的作用域

Spring Bean的作用域
Spring 中的 Bean 作用域（Scope）定义了 Bean 实例的创建方式以及在整个应用程序中的生命周期和可见性范围，Spring 框架提供了多种不同的 Bean 作用域，以下是对它们的详细介绍：

singleton（单例作用域）
Spring默认的 Bean 作用域。在整个Spring容器的生命周期内，对于某个被标记为singleton作用域的 Bean，只会创建一个实例。无论在应用程序的任何地方获取该Bean，得到的都是同一个实例对象。这个实例在容器启动时被创建（如果Bean配置了延迟加载lazy-init属性，则会在首次被请求时创建），并且一直存活直到容器关闭。
•  数据库连接池：数据库连接池通常配置为单例模式，因为只需要一个统一的连接池实例来管理数据库连接资源，供整个应用程序的不同部分共享使用，避免多次创建连接池导致资源浪费以及配置不一致等问题。例如，使用 Druid 连接池，在 Spring 配置中可以将对应的 DataSource Bean 配置为 singleton 作用域，这样整个应用通过注入该 DataSource 就能获取到唯一的连接池实例来获取数据库连接。
•  全局配置类：像应用程序中用于加载配置文件属性的配置类（例如读取 application.properties 中的配置信息并提供相应的访问方法），一般也作为单例存在，方便各个业务组件能统一获取配置内容，不会出现因配置类实例过多而导致配置不一致的情况。

prototype（原型作用域）：
与singleton相反，每次从Spring容器中获取一个标记为prototype作用域的Bean时，都会创建一个新的Bean实例。也就是说，容器不会对该作用域的Bean进行实例的复用，每一次请求获取该Bean都会触发创建操作。
•  用户登录凭证对象：考虑一个用户登录的场景，每次用户登录成功后，会生成一个登录凭证对象（比如包含用户登录的时间、IP地址、登录状态等信息的一个对象），不同用户登录所产生的登录凭证是相互独立的，具有不同的属性值，所以适合将其配置为prototype作用域，这样每次有用户登录时，从Spring容器中获取该登录凭证Bean，都会得到一个全新的、专属于此次登录的实例。
•  复杂业务操作中的临时数据对象：在一些复杂的业务流程中，可能会用到一些临时的数据对象来承载阶段性的计算结果等信息，这些对象在每次业务流程启动时都需要新的实例来进行独立的操作，使用prototype作用域就能保证每次获取到的都是符合需求的新对象。

request（基于请求的作用域，仅适用于 Web 应用）
在Web应用中，对于标记为request作用域的Bean，在同一个HTTP请求的处理过程中，会创建一个唯一的Bean实例，并且该实例仅在这个请求的生命周期内有效。不同的请求会创建各自对应的Bean实例，当请求结束后，相应的Bean实例就会被销毁。
•  请求相关的本地化信息对象：在一个国际化的Web应用中，需要根据不同的请求来获取对应的本地化语言信息，比如用户在浏览器端设置了不同的语言偏好（如中文、英文等），那么可以创建一个Locale对象作为request作用域的Bean，它会根据每个请求中携带的语言相关信息（如请求头中的Accept-Language参数）来获取对应的本地化语言配置，每个请求都有自己独立的Locale实例来处理本地化的业务逻辑，如页面文本的翻译等。
•  请求参数封装对象：将HTTP请求中的参数封装成一个对象（例如把请求中的表单数据封装到一个对应的Java对象中），这个对象只在当前请求中有用，使用request作用域能确保在整个请求处理流程中（比如经过多个Spring MVC的控制器、服务层处理等环节）都能使用同一个请求参数封装对象，方便业务逻辑的处理，并且不会与其他请求的同类对象混淆。

session（基于会话的作用域，仅适用于 Web 应用）
在Web应用中，对于标记为session作用域的Bean，在同一个用户会话（通常由浏览器端的会话标识来确定，如JSESSIONID等）期间，会创建一个唯一的Bean实例，该实例在整个会话生命周期内有效，直到会话结束（比如用户关闭浏览器或者会话超时等情况发生），对应的Bean实例才会被销毁。
•  用户购物车对象：在电商Web应用中，用户的购物车信息需要在整个会话期间进行维护，将购物车对象配置为session作用域，无论用户在同一个会话中如何在不同页面浏览商品、添加或移除商品等操作，始终使用同一个购物车实例来记录购物车中的商品列表、数量等信息，方便进行购物车相关的业务处理，如计算总价、生成订单等操作。
•  用户登录状态对象：用于记录用户登录后的状态信息（如是否已登录、登录的用户账号等），可以将其配置为 session 作用域，在整个会话期间通过这个对象来判断用户的登录状态，保障后续的业务操作能基于正确的登录情况进行，比如只有登录用户才能进行下单、查看订单历史等操作。

application（基于应用的作用域，仅适用于 Web 应用）
在Web应用中，标记为application作用域的 Bean，其创建的实例在整个Web应用的生命周期内都是有效的，整个应用中的所有用户、所有请求都共享这一个实例，类似于singleton作用域在 Web 应用中的扩展，但它强调的是整个Web应用层面的共享。
•  Web 应用的全局计数器：假设要统计整个Web应用的访问次数，创建一个计数器对象作为application作用域的Bean，每有一个用户访问页面或者发起请求，就可以通过这个共享的计数器对象进行计数操作，无论哪个用户、哪个请求都操作的是同一个计数器实例，方便实现对整个应用访问量的统计功能。
•  应用的共享配置对象：比如Web应用中有一些全局的样式配置、通用的功能开关（如某个功能是否启用）等信息，可以汇总到一个配置对象中，并将其配置为application作用域，供整个Web应用的各个部分使用，确保所有用户看到的应用样式、功能可用性等都是一致的。

websocket（基于 WebSocket 的作用域，仅适用于支持 WebSocket 的 Web 应用）
针对使用WebSocket 技术的Web应用，对于标记为websocket作用域的Bean，在同一个WebSocket连接的生命周期内，会创建一个唯一的Bean实例，该实例仅在这个WebSocket连接持续期间有效，当WebSocket连接断开后，相应的Bean实例就会被销毁。
•  WebSocket 通信中的消息处理对象：在一个支持WebSocket的实时聊天应用中，对于每一个WebSocket连接，都可以创建一个消息处理对象作为websocket作用域的Bean，用来处理这个连接上发送和接收的聊天消息，比如对消息进行格式转换、验证、转发等操作，不同的WebSocket连接都有各自独立的消息处理对象实例来保障通信的顺畅和准确性。
