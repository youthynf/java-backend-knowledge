# 如何区分动态代理和静态代理？

如何区分动态代理和静态代理？
静态代理
静态代理是由开发者显式编写的代理类，这个代理类需要实现与目标类相同的接口，并在代理类中调用目标对象的方法。
•  编译时确定代理类：代理类在编译期间就已经生成。
•  代码量较多：需要为每个目标类单独编写代理类，增加了代码冗余。
•  适用场景：适用于代理逻辑较为简单且代理对象较少的情况。

动态代理
动态代理是在运行时动态生成代理类，而不是在编译期显式定义代理类。
•  运行时生成代理类：代理类不需要事先定义。
•  减少代码冗余：多个目标对象可以共用一个动态代理处理逻辑。
•  灵活性更强：可以在运行时动态处理代理逻辑。

实现方式：
•  使用 JDK 自带的动态代理（java.lang.reflect.Proxy，要求目标对象实现接口）。
•  使用第三方库（如 CGLIB，适用于没有实现接口的类）。
