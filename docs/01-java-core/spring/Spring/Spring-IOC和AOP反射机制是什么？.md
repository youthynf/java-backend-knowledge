# Spring IOC和AOP反射机制是什么？

Spring IOC和AOP反射机制是什么？
Spring IOC：
Spring使用反射来实现其核心特性：依赖注入。在Spring中，开发者可以通过xml配置文件或者基于注解的方式声明组件间的依赖关系。当程序启动时，Spring容器会扫描这些配置和注解，通过反射来实例化Bean（即Java对象）,并根据配置自动装配它们。

Spring AOP:
在需要对现有类的方法进行拦截、记录日志、权限控制、或者事务管理的场景中，反射结合动态代理技术被广泛引用。Spring AOP允许开发定义切面（Aspect），这些切面可以横切关注点，并将其插入到业务逻辑中，而不需要修改业务逻辑代码。这一切都是在运行时通过反射来动态构建和执行的，无需硬编码到每个方法调用中。
