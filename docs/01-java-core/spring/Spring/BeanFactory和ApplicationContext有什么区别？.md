# BeanFactory和ApplicationContext有什么区别？

BeanFactory和ApplicationContext有什么区别？
一、概述
BeanFactory和ApplicationContext都是Spring框架中用于管理Bean的核心容器。
二、联系
•  继承关系：ApplicationContext接口是BeanFactory接口的子接口，这意味着ApplicationContext继承了BeanFactory的所有功能，都能够对Bean进行管理，包括创建、配置和获取Bean等操作。
•  核心功能相同：二者的主要目的都是为了解耦对象的创建和使用，通过依赖注入的方式管理对象之间的关系。例如，它们都可以根据配置信息将一个Service类所依赖的Dao类注入进去。

三、区别
•  加载机制：BeanFactory采用延迟加载，只有在第一次获取Bean时才会实例化。而ApplicationContext在启动时就会立即加载并实例化所有单例Bean，这样有助于提前发现配置问题。
•  功能丰富度：BeanFactory主要侧重于Bean的管理。ApplicationContext除了Bean管理外，还提供了许多额外的功能。像支持国际化，可以方便地获取不同语言的消息资源；能进行资源访问，例如读取配置文件等；还提供事件发布和监听机制，用于组件间的通信。
•  使用场景差异：如果只是简单的需要一个容器来管理Bean，对资源比较敏感，BeanFactory可能比较合适。但如果是开发一个完整的企业级应用，需要诸如国际化、事件处理等功能，ApplicationContext就是更好的选择。
