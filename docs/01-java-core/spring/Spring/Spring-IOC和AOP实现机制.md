# Spring IOC和AOP实现机制

Spring IOC和AOP实现机制
•  Spring IOC实现机制：
反射技术：SpringIOC容器利用Java反射机制动态的加载类、创建对象实例以及调用对象方法；
依赖注入：IOC的核心是依赖注入，即容器负责管理应用程序组件之间的依赖关系。Spring通过构造函数注入、属性注入、或者方法注入，将组件之间的依赖关系描述在配置文件中或使用注解；
工厂模式：采用工厂模式来管理对象的创建和生命周期。容器作为工厂负责实例化Bean并管理它们的生命周期；
容器实现：SpringIOC容器是实现IOC的核心，通常使用BeanFactory或ApplictionContext来管理Bean。BeanFactory是IOC容器的基本形式，提供基本的IOC功能；ApplicationContext是BeanFactory的扩展，并提供更多企业级功能；

•  Spring AOP实现机制：
Spring AOP的实现依赖于动态代理技术和反射技术。动态代理是运行时动态地生成代理对象，而不是编译时。允许开发者在运行时执行要代理的接口或行为，从而达到不修改源码的情况下实现方法增强。
基于JDK的动态代理：使用java.lang.reflect.Proxy类和java.lang.reflect.InvocationHandler接口实现，这种方法要求被代理的对象实现一个或多个接口；
基于CGLIB的动态代理：当被代理的类没有实现接口时，Spring会使用CGLIB库生成一个被代理类的子类作为代理。CGLIB（Code Generation Library）是一个第三方代码生成库，通过继承方式实现代理。
