# 传统EJB简介

传统EJB简介
一、概述
EJB（Enterprise JavaBeans）容器是一个用于管理 Java EE（现为 Jakarta EE）中的企业级 Bean 组件的运行环境。EJB 是一种服务器端的组件模型，旨在简化企业级应用程序的开发，提供了企业级应用程序所需的诸多服务，如事务管理、并发控制、安全性、持久化等。EJB 容器是 EJB 组件的执行环境，它负责管理 EJB 的生命周期、事务、并发、安全性等。EJB 容器是 Java EE 容器的一部分，是企业级应用的核心组件之一。
随着 Java EE 技术的演进，EJB 在许多场合已经被轻量级框架（如 Spring）所替代，但在大规模企业应用中，EJB 依然是一个强大的工具，尤其是在需要分布式事务管理和消息驱动的场景下。
二、EJB相对spring重量级体现
EJB 的重量级特点：
•  需要依赖于 Java EE 容器（应用服务器），配置复杂且需要大量的系统资源；
•  生命周期、事务管理、安全性等功能由容器控制，开发人员依赖容器的管理；
•  通常适用于大型企业级应用，尤其是在分布式系统中。
Spring 的轻量级特点：
•  可以独立于容器运行，灵活性高，适用于小到大型应用；
•  配置和部署相对简单，使用灵活的依赖注入和面向切面的编程；
•  提供了更高的模块化支持，支持微服务架构和集群架构。
三、总结
因此，EJB 被视为重量级框架，主要体现在其对容器和应用服务器的依赖、复杂的配置以及系统资源的占用，而 Spring 更加灵活和轻量，适合快速开发、微服务架构及小型到中型应用。
