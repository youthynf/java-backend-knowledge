# 线程状态如何转换？

线程状态如何转换？
线程状态转换
新建（New）：创建后尚未启动
可运行（Runnable）：可能正在运行，也可能正在等待CPU时间片，包含了操作系统线程状态中的Running和Ready；
阻塞（Blocking）：等待获取一个排它锁，如果其它线程释放了锁，就会释放此状态；
无限等待（Waiting）：等待其他线程显式地唤醒，否则不会被分配CPU时间片；
限时等待（Timed Waiting）：无需等待其他线程显式地唤醒，在一定时间之后会被系统自动唤醒。
死亡（Terminated）：可以是线程结束任务之后自己结束，或者产生了异常而结束；

区分阻塞/等待、睡眠/挂起
睡眠和挂起是用来描述行为，而阻塞和等待是用来描述状态；
阻塞时被动的，它是在等待获取一个排它锁，而等待是主动的，通过调用Thread.sleep()或Object.wait()等方法进入；
