# concurrency

## 面试复习重点

- 线程安全、锁、JMM、AQS、线程池和并发容器的关系。
- 可见性、原子性、有序性如何映射到 volatile、synchronized 和 Lock。
- 死锁、线程池耗尽、上下文切换和并发性能问题的排查。

## 建议掌握程度

- **能讲清概念**：用自己的话说明定义、背景和解决的问题。
- **能画出链路**：把关键组件、核心流程和状态变化串起来。
- **能回答追问**：准备优缺点、适用场景、常见坑和替代方案。
- **能落地排查**：结合日志、指标、工具和案例说明定位思路。

## 面试表达模板

1. 先给结论：说明它是什么、解决什么问题。
2. 再讲原理：说明核心组件、关键流程和数据结构。
3. 补充场景：结合项目或线上问题说明怎么用。
4. 说明边界：讲清楚缺点、风险、替代方案和排查手段。

## 面试复习建议

- 先用目录建立知识地图：概念 → 原理 → 参数/源码 → 排查或实战。
- 对每篇文章至少能回答三个问题：它解决什么问题、底层依赖什么机制、生产环境如何使用或排查。
- 复习时不要只背结论，建议结合 JDK 版本、业务场景和故障现象组织答案。

## 高频追问方向

1. 这个概念和相邻概念有什么区别？
2. 默认行为是什么？哪些参数或代码会改变默认行为？
3. 真实项目中怎么验证、监控、调优或避免风险？
