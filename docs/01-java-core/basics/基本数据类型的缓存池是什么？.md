# 基本数据类型的缓存池是什么？

## 核心概念

Java 的部分包装类型内部实现了缓存池，会缓存一段常用取值范围内的对象。通过自动装箱创建包装对象时，如果值在缓存范围内，会直接复用缓存对象。

缓存池的目的主要是减少频繁创建小整数等常用对象带来的内存和性能开销。

## 常见缓存范围

| 包装类型 | 默认缓存范围 |
|---|---|
| `Byte` | -128 ~ 127 |
| `Short` | -128 ~ 127 |
| `Integer` | -128 ~ 127 |
| `Long` | -128 ~ 127 |
| `Character` | 0 ~ 127 |
| `Boolean` | `true` / `false` |

其中 `Integer` 的上限可以通过 JVM 参数调整：

```bash
-XX:AutoBoxCacheMax=<size>
```

## 示例

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b); // true

Integer c = 128;
Integer d = 128;
System.out.println(c == d); // false
```

`127` 在默认缓存范围内，所以复用同一个对象；`128` 超出默认缓存范围，自动装箱时会创建不同对象。

## 注意事项

包装类型比较数值时，不要使用 `==`，应该使用 `equals()`。

```java
Integer a = 128;
Integer b = 128;
System.out.println(a.equals(b)); // true
```

## 总结

缓存池是包装类型的对象复用优化，不应该成为业务逻辑判断的依据。面试中常用它考察自动装箱、对象引用比较和 `equals()` 的区别。

## 面试复习版

### 核心概念
- Java 有 8 种基本类型，包装类型用于对象语义、集合和泛型。
- 装箱通常调用 valueOf，Integer 默认缓存 -128~127；拆箱调用 xxxValue。

### 面试官想考什么
- 基本类型大小、默认值和包装类型区别。
- 包装类型 ==、equals、缓存和 NPE 风险。

### 标准回答
基本类型直接保存值，包装类型是对象。自动装箱/拆箱提升编码便利性，但比较时要区分引用相等和值相等；包装对象可能为 null，拆箱会抛出 NullPointerException。

### 深挖追问
- 为什么集合不能直接存 int？
- Integer 127 和 128 的 == 为什么不同？
- char 能不能存中文？

### 实战场景/代码示例
```java
Integer a=127,b=127,c=128,d=128;
System.out.println(a==b); // 通常 true
System.out.println(c==d); // 通常 false
```

### 易错点/总结
- 业务比较不要依赖包装类型 ==。
- 金额不要用 float/double，优先 BigDecimal。
