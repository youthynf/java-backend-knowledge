# 如何区分重载和重写

## 核心概念

重载（Overload）和重写（Override）都是 Java 多态的体现，但二者发生的位置、判断依据和绑定时机完全不同。

- **重载**：发生在**同一个类**中，方法名相同但参数列表不同。编译期决定调用哪个方法，是"静态多态"。
- **重写**：发生在**父子类或接口实现**中，子类重新实现父类已有的方法。运行期根据真实对象类型决定调用哪个方法，是"动态多态"。

一句话区分：**重载看参数列表，重写看继承关系；重载是同名不同参，重写是子类改父类行为。**

```java
class Calculator {
    // 重载：同名方法不同参数
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}

class AdvancedCalculator extends Calculator {
    // 重写：子类重新实现父类方法
    @Override
    public int add(int a, int b) {
        System.out.println("advanced");
        return super.add(a, b);
    }
}
```

## 标准回答

重载和重写是多态的两种形式。重载在同一类中，方法名相同参数不同，编译期绑定；重写在父子类中，方法签名一致，运行期绑定。要点：

1. **重载判定**：方法名相同 + 参数列表不同（类型、个数、顺序）。返回值不参与判定。
2. **重写判定**：方法名 + 参数列表完全一致；访问权限不能更严格；返回类型协变；抛出异常不能更宽。
3. **绑定时机**：重载是编译期静态绑定；重写是运行期动态绑定。
4. **`@Override` 注解**：标注重写，让编译器帮助检查，强烈建议加。
5. **`private`/`static`/`final` 方法不能被重写**：`private` 不可见、`static` 只能隐藏、`final` 禁止重写。

## 实现原理

### 1. 重载的判定规则

重载的核心是**参数列表不同**，包括：

- 参数类型不同
- 参数个数不同
- 参数顺序不同（一般不推荐，可读性差）

```java
class Printer {
    public void print(int x) { System.out.println("int: " + x); }
    public void print(long x) { System.out.println("long: " + x); }
    public void print(String s) { System.out.println("String: " + s); }
    public void print(int x, int y) { System.out.println("two ints"); }

    // 仅有返回值不同，不是重载，编译错误
    // public int print(int x) { return x; }
}
```

返回值类型、访问修饰符、抛出异常**不参与重载判定**。如果两个方法仅返回值不同，编译器会报错"method is already defined"。

#### 重载方法的选择

编译器选择重载方法的优先级（JLS §15.12.2）：

1. **精确匹配**：参数类型完全一致。
2. ** widening primitive conversion**：基本类型小转大，如 `int` 转 `long`。
3. **autoboxing**：自动装箱，如 `int` 转 `Integer`。
4. **varargs**：可变参数。

```java
void call(int x) { System.out.println("int"); }
void call(long x) { System.out.println("long"); }
void call(Integer x) { System.out.println("Integer"); }
void call(int... xs) { System.out.println("varargs"); }

call(1);  // 输出 "int"，精确匹配优先
// 删除 call(int) 后输出 "long"，widening 优先于 boxing
// 删除 call(long) 后输出 "Integer"，boxing 优先于 varargs
```

这种优先级在面试中常考：`call(1)` 在有 `call(int)` 时调用 `int` 版本，没有时优先 `long`，再没有才 `Integer`，最后 varargs。

### 2. 重写的判定规则

重写要求子类方法与父类方法签名一致，并满足：

- 方法名相同
- 参数列表相同
- 返回类型相同或是协变（子类型）
- 访问权限不能更严格（`public` > `protected` > `default` > `private`）
- 抛出的受检异常不能更宽
- 不能重写 `final`、`static`、`private` 方法

#### 协变返回类型（Covariant Return Type）

Java 5+ 允许子类重写方法返回父类返回类型的子类型：

```java
class Parent {
    public Number getValue() { return 1; }
}

class Child extends Parent {
    @Override
    public Integer getValue() { return 2; }  // Integer 是 Number 的子类型
}
```

#### 访问权限规则

```java
class Parent {
    public void method() {}
    protected void method2() {}
}

class Child extends Parent {
    @Override
    public void method() {}      // 可以，public >= public

    @Override
    // void method2() {}         // 编译错误：default 比 protected 更严格
    public void method2() {}     // 可以，public >= protected
}
```

#### 异常规则

```java
class Parent {
    public void read() throws IOException {}
}

class Child extends Parent {
    @Override
    public void read() throws FileNotFoundException {}  // 可以，子类异常
    // public void read() throws Exception {}           // 编译错误：Exception 比 IOException 宽
}
```

子类方法可以**不抛出异常**或抛出**更窄的受检异常**，但不能抛出更宽的。非受检异常（`RuntimeException`）不受限制。

### 3. 绑定时机

#### 重载：静态绑定（编译期）

编译器根据**引用类型**和**参数类型**在编译期决定调用哪个重载方法。字节码层面直接生成 `invokevirtual` 或 `invokestatic` 指令指向具体方法。

```java
class Printer {
    public void print(String s) { System.out.println("String"); }
    public void print(Object o) { System.out.println("Object"); }
}

Object s = "hello";
new Printer().print(s);  // 输出 "Object"，因为编译期看引用类型是 Object
```

注意：重载看的是**编译期类型**，不是运行时类型。`s` 运行时是 `String`，但编译期是 `Object`，所以匹配 `print(Object)`。

#### 重写：动态绑定（运行期）

JVM 在运行时根据**对象真实类型**决定调用哪个方法。字节码用 `invokevirtual` 指令，运行时通过 vtable（虚方法表）查找。

```java
class Animal {
    public void speak() { System.out.println("animal"); }
}

class Dog extends Animal {
    @Override
    public void speak() { System.out.println("汪"); }
}

Animal a = new Dog();
a.speak();  // 输出 "汪"，运行期根据真实对象类型 Dog 找方法
```

### 4. `private`、`static`、`final` 方法

| 方法类型 | 能否重写 | 说明 |
|---------|---------|------|
| `private` | 否 | 子类不可见，定义同名方法不是重写 |
| `static` | 否（只能隐藏） | 子类同名静态方法是"隐藏"父类，调用看引用类型 |
| `final` | 否 | 编译器禁止重写 |
| 普通方法 | 能 | 通过 `@Override` 标注 |

```java
class Parent {
    public static void hello() { System.out.println("parent static"); }
    public final void bye() { System.out.println("parent final"); }
}

class Child extends Parent {
    public static void hello() { System.out.println("child static"); }  // 隐藏，不是重写

    // @Override  // 编译错误：final 方法不能重写
    // public void bye() {}
}

Parent p = new Child();
p.hello();  // parent static，静态方法看引用类型
```

### 5. 重写与 `@Override`

```java
class Parent {
    public void method(int x) {}
}

class Child extends Parent {
    @Override
    public void method(long x) {}  // 编译错误：method(long) 不重写 method(int)
}
```

`@Override` 让编译器帮你检查是否真正完成了重写。如果方法签名不匹配父类，编译错误。这是**强烈推荐**的实践——避免"以为重写了其实没有"的低级 bug。

### 6. 接口默认方法的重写

Java 8+ 接口可以有 `default` 方法，实现类可以重写：

```java
interface Greeter {
    default void greet() { System.out.println("hello"); }
}

class ChineseGreeter implements Greeter {
    @Override
    public void greet() { System.out.println("你好"); }
}
```

如果父类和接口都有同名方法，**类优先**：父类的方法会覆盖接口的 default 方法。

## 代码示例

### 重载综合

```java
public class OverloadDemo {
    public void show(int x) { System.out.println("int: " + x); }
    public void show(long x) { System.out.println("long: " + x); }
    public void show(Integer x) { System.out.println("Integer: " + x); }
    public void show(int... xs) { System.out.println("varargs: " + Arrays.toString(xs)); }

    public static void main(String[] args) {
        OverloadDemo d = new OverloadDemo();
        d.show(1);        // int: 1（精确匹配）
        d.show(1L);       // long: 1
        Integer i = 1;
        d.show(i);        // Integer: 1
        d.show(1, 2);     // varargs: [1, 2]
    }
}
```

### 重写 + 多态

```java
abstract class Shape {
    public abstract double area();
}

class Circle extends Shape {
    private final double r;
    public Circle(double r) { this.r = r; }
    @Override
    public double area() { return Math.PI * r * r; }
}

class Rectangle extends Shape {
    private final double w, h;
    public Rectangle(double w, double h) { this.w = w; this.h = h; }
    @Override
    public double area() { return w * h; }
}

public class ShapeDemo {
    public static void main(String[] args) {
        List<Shape> shapes = List.of(new Circle(2), new Rectangle(3, 4));
        shapes.forEach(s -> System.out.println(s.area()));  // 12.566 12.0
    }
}
```

`Shape` 是抽象类型，运行时根据真实对象（`Circle` 或 `Rectangle`）调用对应的 `area()`。这就是多态。

### 编译期类型 vs 运行时类型

```java
class Base {
    public void method(String s) { System.out.println("Base.string"); }
}

class Sub extends Base {
    @Override
    public void method(String s) { System.out.println("Sub.string"); }
    public void method(Object o) { System.out.println("Sub.object"); }
}

Base obj = new Sub();
obj.method("hello");  // 输出 "Sub.string"
```

分析：`obj` 编译期类型是 `Base`，所以重载只在 `Base` 的方法中选择，确定调用 `method(String)`。运行期根据真实类型 `Sub`，调用 `Sub.method(String)`。

如果删除 `Base` 中的 `method(String)`，编译会失败，因为编译期找不到 `Base.method(String)`。这就是"重载看编译期类型"。

## 实战场景

| 场景 | 用法 | 注意点 |
|------|------|--------|
| 工具类多版本方法 | `println(int)`、`println(String)` 等重载 | 参数类型清晰，避免歧义 |
| 构造器重载 | `User(String)`、`User(String, int)` | 通过 `this(...)` 链式调用 |
| 模板方法模式 | 抽象父类定义骨架，子类重写关键步骤 | 用 `final` 修饰骨架方法防止破坏流程 |
| 策略模式 | 接口定义策略，多个实现类重写 | 配合工厂或依赖注入 |
| 集合 `add` / `addAll` | 重载 `add(E)` 和 `addAll(Collection)` | 注意泛型擦除下的重载陷阱 |
| `equals` 重写 | 重写 `Object.equals(Object)` | 别误写成 `equals(MyType)`，那是重载 |

## 深挖追问

### 1. 为什么 `@Override` 强烈推荐

`@Override` 让编译器帮你检查"是否真正重写了父类方法"。常见低级 bug：

- 父类方法名拼错，子类以为重写了其实没有。
- 父类方法参数列表变了，子类方法签名不再匹配。
- 把 `equals(MyType)` 当成 `equals(Object)` 重写，实际是重载。

加了 `@Override`，这些 bug 在编译期暴露。

### 2. 重载方法的选择优先级

JLS §15.12.2 规定的优先级：

1. Phase 1：精确匹配（无装箱、无可变参数）。
2. Phase 2：允许装箱/拆箱，但无可变参数。
3. Phase 3：允许装箱/拆箱和可变参数。

`call(1)` 优先匹配 `call(int)`，没有则 `call(long)`，再没有则 `call(Integer)`，最后 `call(int...)`。这种优先级避免歧义。

### 3. 重写能否抛出更宽的异常

不能抛出更宽的**受检异常**，因为调用方按父类签名处理异常，子类抛更宽的异常会让调用方措手不及。可以抛更窄的受检异常或不抛。非受检异常（`RuntimeException`）不受限制。

### 4. 重写和隐藏的区别

- **重写**：实例方法，运行时根据对象真实类型决定调用，多态生效。
- **隐藏**：静态方法，编译期根据引用类型决定调用，多态不生效。

```java
class P { public static void m() {} public void n() {} }
class C extends P { public static void m() {} public void n() {} }

P p = new C();
p.m();  // P.m()，静态方法隐藏，看引用类型
p.n();  // C.n()，实例方法重写，看对象类型
```

### 5. `equals` 重写最常见的错误

```java
public boolean equals(MyClass o) { ... }  // 重载，不是重写！
```

正确写法是 `public boolean equals(Object o)`，参数必须是 `Object`。加了 `@Override` 编译器会报错。

### 6. 构造器能被重写吗

不能。构造器不是普通方法，没有"重写"概念。子类构造器通过 `super(...)` 调用父类构造器，但不叫重写。

## 易错点

- 把 `equals(MyType)` 当成 `equals(Object)` 重写，实际是重载。
- 忘记加 `@Override`，方法签名拼错时静默失败。
- 重载方法只改返回值类型，编译错误（不算重载）。
- 子类重写时把访问权限改严格（如 `public` 改 `protected`），编译错误。
- 子类重写时抛出更宽的受检异常，编译错误。
- 误以为静态方法能被重写，实际只能隐藏。
- 重载方法调用看运行时类型，实际看编译期类型。
- 在 Lambda 中误以为可以重写父类方法，Lambda 只能实现函数式接口。

## 总结

重载和重写是多态的两种形式，核心区别是：重载发生在同类中看参数列表，编译期绑定；重写发生在父子类中看方法签名一致，运行期绑定。生产中要给重写方法加 `@Override`、重载方法参数类型清晰避免歧义、注意 `private`/`static`/`final` 不能重写。理解"重载看编译期类型、重写看运行时类型"是 Java 多态的精髓，能解释大量"看似奇怪"的方法调用行为。
