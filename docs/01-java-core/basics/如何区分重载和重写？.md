# 如何区分重载和重写？

## 核心概念

重载（Overload）和重写（Override）都是 Java 多态的重要体现，但二者发生的位置、判断依据和绑定时机不同。

- **重载**：发生在同一个类中，方法名相同，参数列表不同。
- **重写**：发生在继承体系中，子类重新实现父类已有的方法。

## 重写：Override

重写指子类实现了一个与父类方法声明兼容的方法，用于改变或扩展父类方法的行为。

### 判断条件

- 必须存在继承或接口实现关系。
- 子类方法名、参数列表必须与父类方法一致。
- 子类方法的访问权限必须大于等于父类方法。
- 子类方法的返回类型必须是父类方法返回类型或其子类型。
- 子类方法不能抛出比父类方法更宽泛的受检异常。

建议使用 `@Override` 注解，让编译器帮助检查是否真正完成了重写。

```java
class Parent {
    public Number getValue() {
        return 1;
    }
}

class Child extends Parent {
    @Override
    public Integer getValue() {
        return 2;
    }
}
```

上面代码中，`Integer` 是 `Number` 的子类型，因此返回值满足协变返回类型要求。

## 重载：Overload

重载指在同一个类中定义多个同名方法，但它们的参数类型、参数个数或参数顺序至少有一个不同。

### 判断条件

- 方法名相同。
- 参数列表不同，包括参数类型、参数个数、参数顺序。
- 只有返回值不同，不构成重载。

```java
class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

## 对比总结

| 对比项 | 重载 | 重写 |
|---|---|---|
| 英文 | Overload | Override |
| 发生位置 | 同一个类中 | 父子类或接口实现类中 |
| 方法名 | 必须相同 | 必须相同 |
| 参数列表 | 必须不同 | 必须相同 |
| 返回值 | 不能仅靠返回值区分 | 必须兼容父类返回值 |
| 访问权限 | 无特殊限制 | 子类不能比父类更严格 |
| 绑定时机 | 编译期决定 | 运行期根据真实对象类型决定 |

## 总结

一句话区分：**重载看参数，重写看继承；重载是同名不同参，重写是子类改父类行为。**

## 面试复习版

### 核心概念
- 重载是同类同名不同参数，编译期决定。
- 重写是子类重新实现父类可继承方法，运行期动态绑定。

### 面试官想考什么
- 编译期多态与运行期多态。
- 返回值不同不能构成重载。

### 标准回答
重载看参数类型、个数、顺序；重写看继承关系和方法签名，访问权限不能更窄，受检异常不能更宽。

### 深挖追问
- private/static/final 方法能重写吗？
- 构造方法能重载或重写吗？
- @Override 有什么价值？

### 实战场景/代码示例
```java
class A { void f(Number n){} }
class B extends A { @Override void f(Number n){} void f(Integer i){} }
```

### 易错点/总结
- 静态方法隐藏不是重写。
- 重载解析发生在编译期。
