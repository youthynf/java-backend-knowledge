# 面向对象设计的SOLID原则是什么？

面向对象设计的SOLID原则是什么？
单一责任原则
让一个类只做一种类型责任，当这个类需要承当其他类型的责任的时候，就需要分解这个类。
开放封闭原则
对扩展是开放的，而对修改是封闭的。软件实体应该是可扩展，而不可修改的。也就是说。
里氏替换原则
当一个子类的实例应该能够替换任何其超类的实例时，它们之间才具有is-A关系。
依赖倒置原则
高层模块不应该依赖于低层模块，二者都应该依赖于抽象；而抽象不应该依赖于细节，应该细节依赖于抽象；
接口分离原则
使用多个专门的接口比使用单一的总接口总要好。不能强迫用户去依赖那些他们不使用的接口。

## 面试复习版

### 核心概念
- 三大特性：封装、继承、多态。
- SOLID：单一职责、开闭、里氏替换、接口隔离、依赖倒置。

### 面试官想考什么
- 能否把原则落到代码设计。
- 是否理解多态和抽象的价值。

### 标准回答
面向对象用封装隐藏复杂度，用继承表达共性，用多态提升扩展性；SOLID 是降低耦合、提升可维护性的设计原则，不是机械套公式。

### 深挖追问
- 开闭原则如何实现？
- 里氏替换违反会有什么后果？
- 依赖倒置和依赖注入关系？

### 实战场景/代码示例
```java
interface DiscountPolicy { BigDecimal discount(BigDecimal amount); }
class OrderService { private final DiscountPolicy policy; }
```

### 易错点/总结
- 原则服务于复杂度控制，不能过度设计。
- 继承不是复用代码的唯一方式，组合常更灵活。
