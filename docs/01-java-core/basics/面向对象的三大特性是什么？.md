# 面向对象的三大特性是什么

## 核心概念

面向对象的三大特性是 **封装、继承、多态**。它们分别解决代码组织、代码复用和行为扩展三个问题。

- **封装**：把数据和操作数据的方法组织到对象中，隐藏内部实现细节，对外只暴露必要接口。
- **继承**：子类复用父类的属性和方法，表达 `is-a` 关系。
- **多态**：同一父类引用在运行时可以指向不同子类对象，表现出不同行为。

```java
// 封装：状态 + 行为 + 访问控制
class Account {
    private long balance;   // 状态私有
    public void deposit(long amount) {  // 行为公开
        if (amount <= 0) throw new IllegalArgumentException();
        balance += amount;
    }
}

// 继承：子类复用父类
class SavingsAccount extends Account {
    public void addInterest() { /* ... */ }
}

// 多态：父类引用指向子类对象
Account a = new SavingsAccount();
a.deposit(100);
```

## 标准回答

面向对象三大特性是封装、继承、多态。要点：

1. **封装**：`private` 字段 + `public` 方法，隐藏实现、控制访问。
2. **继承**：`extends` 复用父类代码，表达 `is-a`，Java 单继承。
3. **多态**：编译时多态（重载）+ 运行时多态（重写 + 向上转型）。
4. **运行时多态三条件**：继承/实现、子类重写、父类引用指向子类对象。
5. **设计原则**：优先组合而非继承（Effective Java Item 18），多态比 `instanceof` 更优雅。

## 实现原理

### 1. 封装

封装是把数据（字段）和操作数据的方法（行为）放到一个对象里，并通过访问控制修饰符（`public`/`protected`/`private`/默认）控制外部访问。

```java
public class User {
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void changeName(String name) {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("name 不能为空");
        }
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("age 非法");
        }
        this.age = age;
    }
}
```

封装的核心价值：

- **数据保护**：字段私有，外部不能直接修改，必须通过方法。
- **校验集中**：修改规则集中在 setter 里，避免散落各处。
- **实现可变**：内部实现可以重构，外部接口不变，调用方无感。
- **降低耦合**：调用方只关心"做什么"，不关心"怎么做"。

#### 访问修饰符

| 修饰符 | 类内 | 同包 | 子类 | 其他 |
|--------|------|------|------|------|
| `public` | ✓ | ✓ | ✓ | ✓ |
| `protected` | ✓ | ✓ | ✓ | ✗ |
| 默认（package-private） | ✓ | ✓ | ✗ | ✗ |
| `private` | ✓ | ✗ | ✗ | ✗ |

### 2. 继承

继承表达 `is-a` 关系，子类自动获得父类非 `private` 的字段和方法，并可扩展或重写。

```java
class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating");
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 调用父类构造器
    }

    public void bark() {
        System.out.println(name + " is barking");
    }
}

Dog d = new Dog("Rex");
d.eat();    // 继承自 Animal
d.bark();   // Dog 自己的方法
```

#### 继承的注意事项

- **Java 单继承**：一个类只能 `extends` 一个父类，但可以 `implements` 多个接口。
- **构造器链**：子类构造器必须先调用父类构造器（`super(...)`），未显式调用则默认 `super()`。
- **`final` 类不能被继承**：`String`、`Integer` 等。
- **`private` 不被继承**：子类访问不到父类的 `private` 字段。
- **`super` 调用父类方法**：`super.method()` 在重写后调用父类版本。

#### 继承的代价

继承是强耦合：子类依赖父类的实现细节，父类变化会影响所有子类。Effective Java Item 18 建议**优先组合而非继承**：

```java
// 继承：强耦合
class HashSet<E> extends AbstractSet<E> { ... }

// 组合：松耦合
class CountingSet<E> {
    private final Set<E> delegate = new HashSet<>();
    private int addCount = 0;

    public boolean add(E e) {
        addCount++;
        return delegate.add(e);
    }
    // 转发其他方法到 delegate
}
```

组合更灵活：`CountingSet` 可以包装任意 `Set` 实现（`HashSet`、`TreeSet`），不绑定具体类。

### 3. 多态

多态表示"同一调用，不同表现"。Java 多态分两种：

| 类型 | 体现方式 | 决定时机 |
|------|---------|---------|
| 编译时多态 | 方法重载（Overload） | 编译期根据参数列表 |
| 运行时多态 | 方法重写（Override） + 向上转型 | 运行期根据对象真实类型 |

#### 运行时多态三条件

1. 存在继承或接口实现关系。
2. 子类重写父类方法。
3. 使用父类引用指向子类对象（向上转型）。

```java
class Animal {
    public void speak() { System.out.println("animal sound"); }
}

class Cat extends Animal {
    @Override
    public void speak() { System.out.println("喵"); }
}

class Dog extends Animal {
    @Override
    public void speak() { System.out.println("汪"); }
}

public class Zoo {
    public static void main(String[] args) {
        Animal a1 = new Cat();   // 向上转型
        Animal a2 = new Dog();
        a1.speak();              // 喵
        a2.speak();              // 汪
    }
}
```

引用类型是 `Animal`，但运行时根据对象真实类型调用对应方法。

#### 多态的字节码实现

运行时多态通过 `invokevirtual` 指令 + vtable（虚方法表）实现：

- 编译期生成 `invokevirtual` 指令，指向父类方法。
- 运行期 JVM 查对象真实类型的 vtable，找到子类重写后的方法。
- vtable 是方法指针数组，每个类一份，子类的 vtable 中重写方法的位置被子类方法覆盖。

`invokestatic`、`invokespecial`（构造器、`private` 方法）、`invokeinterface` 有不同的查找规则，但 `invokevirtual` 是多态的核心。

#### 多态的经典应用：策略模式

```java
interface DiscountStrategy {
    long apply(long original);
}

class FullDiscount implements DiscountStrategy {
    @Override
    public long apply(long original) { return original >= 100 ? original - 20 : original; }
}

class NoDiscount implements DiscountStrategy {
    @Override
    public long apply(long original) { return original; }
}

class OrderService {
    private final DiscountStrategy strategy;
    public OrderService(DiscountStrategy strategy) { this.strategy = strategy; }

    public long calcPrice(long original) {
        return strategy.apply(original);  // 多态：不同策略不同行为
    }
}
```

`OrderService` 不关心具体策略，运行时根据传入的策略对象调用对应实现。这是多态最典型的工程价值。

## 代码示例

### 封装：不可变对象

```java
public final class Money {
    private final long amount;       // 用分表示，避免浮点
    private final String currency;

    public Money(long amount, String currency) {
        if (amount < 0) throw new IllegalArgumentException();
        this.amount = amount;
        this.currency = currency;
    }

    public Money add(Money other) {
        if (!currency.equals(other.currency)) throw new IllegalArgumentException();
        return new Money(amount + other.amount, currency);
    }

    public long getAmount() { return amount; }
    public String getCurrency() { return currency; }
}
```

不可变对象是封装的极致：状态完全私有且不可变，所有"修改"都返回新对象，天然线程安全。

### 继承：模板方法模式

```java
public abstract class DataPipeline {
    public final void execute() {       // 模板方法
        String raw = extract();
        String data = transform(raw);
        load(data);
    }
    protected abstract String extract();
    protected abstract String transform(String raw);
    protected abstract void load(String data);
}

public class CsvToDbPipeline extends DataPipeline {
    @Override protected String extract() { return "csv data"; }
    @Override protected String transform(String raw) { return "transformed: " + raw; }
    @Override protected void load(String data) { System.out.println("save: " + data); }
}
```

### 多态：集合统一处理

```java
import java.util.*;

public class ShapeArea {
    abstract static class Shape {
        public abstract double area();
    }

    static class Circle extends Shape {
        private final double r;
        Circle(double r) { this.r = r; }
        @Override public double area() { return Math.PI * r * r; }
    }

    static class Rectangle extends Shape {
        private final double w, h;
        Rectangle(double w, double h) { this.w = w; this.h = h; }
        @Override public double area() { return w * h; }
    }

    public static void main(String[] args) {
        List<Shape> shapes = List.of(new Circle(2), new Rectangle(3, 4));
        double total = shapes.stream().mapToDouble(Shape::area).sum();
        System.out.println(total);  // 24.566...
    }
}
```

`shapes` 列表存的是 `Shape` 引用，但运行时分别调用 `Circle.area` 和 `Rectangle.area`。这就是多态。

## 实战场景

| 场景 | 用法 | 注意点 |
|------|------|--------|
| 实体类设计 | 私有字段 + getter/setter + 校验 | 字段别全 public |
| 不可变值对象 | `final` 类 + `final` 字段 + 无 setter | 优先不可变设计 |
| 模板方法模式 | 抽象类定义骨架，子类填充步骤 | 骨架方法用 `final` 防破坏 |
| 策略模式 | 接口 + 多实现 + 依赖注入 | 替代 `if-else` 分支 |
| 框架扩展点 | 继承框架基类或实现接口 | 注意框架的 `final` 限制 |
| 集合统一处理 | 父类引用 List 存不同子类 | 多态让代码可扩展 |
| 防御性拷贝 | getter 返回字段副本 | 防止外部修改破坏内部状态 |

## 深挖追问

### 1. 多态能不能"多继承"

Java 类只能单继承，但接口可以多实现。多态的本质是"父类引用指向子类对象"，可以通过接口实现"多能力多态"：

```java
interface A { void a(); }
interface B { void b(); }

class C implements A, B {
    public void a() {}
    public void b() {}
}

A a = new C();
B b = new C();
```

### 2. 继承和组合怎么选

- **继承**：表达 `is-a` 关系，且子类真正是父类的特化。例如 `Dog extends Animal`。
- **组合**：表达 `has-a` 关系，类 A 包含类 B 作为字段。例如 `Car has Engine`。

Effective Java Item 18 强烈建议"组合优于继承"。继承是强耦合，父类变化直接影响子类；组合是松耦合，可以包装、装饰、动态替换。

### 3. 运行时多态的字节码实现

`invokevirtual` 指令在运行时根据对象真实类型查 vtable。vtable 是方法指针数组，子类的 vtable 中重写方法的位置被子类方法覆盖。`invokeinterface` 类似但走 itable（接口方法表），查找稍慢。

`invokestatic`、`invokespecial`（构造器、`private` 方法）是静态绑定，编译期确定，不走 vtable。

### 4. 多态和重载的区别

- **重载**：编译时多态，根据参数列表选择方法，看编译期类型。
- **重写**：运行时多态，根据对象真实类型调用方法，看运行时类型。

两者常被混淆。详见 [如何区分重载和重写？](如何区分重载和重写？.md)。

### 5. 多态有什么缺点

- **性能开销**：`invokevirtual` 比 `invokespecial` 稍慢（多查一次 vtable），现代 JIT 会优化（CHA、内联）。
- **可读性**：追踪代码时需要看实际类型，IDE 帮助大。
- **设计复杂度**：滥用多态会导致继承层级过深、难以维护。

### 6. 封装的字段能被反射绕过吗

能。反射 `Field.setAccessible(true)` 后可以读写 `private` 字段，破坏封装。但这是"逃生舱"，不应该作为正常手段。Java 9+ 模块系统对反射做了进一步限制。

## 易错点

- 字段全 `public`，外部任意修改，封装形同虚设。
- setter 不做校验，传入非法值导致后续 bug。
- 子类重写时缩小访问权限（如 `public` 改 `protected`），编译错误。
- 父类构造器没无参版本，子类构造器没显式 `super(...)`，编译错误。
- 继承层级过深，父类变化导致所有子类连锁修改。
- 误把组合当继承：`Car extends Engine` 错误，应该 `Car has Engine`。
- 多态调用时引用类型决定能调用哪些方法，子类独有方法父类引用调不到。
- `static` 方法不能多态（只能隐藏），`final` 方法不能重写。
- 期望 `private` 方法能被重写，实际不会（子类不可见）。

## 总结

封装、继承、多态是面向对象三大特性。封装通过访问控制隐藏实现、保护数据；继承通过 `extends` 复用代码、表达 `is-a`；多态通过重写和向上转型实现"同一调用、不同表现"。生产实践中：封装要做到字段私有、方法校验、接口稳定；继承要克制使用，优先组合；多态要善用接口和重写替代 `if-else` 分支。理解三大特性不仅是面试要求，更是写出可维护、可扩展 Java 代码的基础。
