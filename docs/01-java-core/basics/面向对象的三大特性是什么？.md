# 面向对象的三大特性是什么？

## 核心概念

面向对象的三大特性是：**封装、继承、多态**。它们分别解决了代码组织、代码复用和行为扩展的问题。

- **封装**：把数据和操作数据的方法组织到一个对象中，并隐藏内部实现细节。
- **继承**：让子类复用父类的属性和方法，表达一种 `is-a` 关系。
- **多态**：同一个父类引用在运行时可以指向不同子类对象，从而表现出不同的行为。

## 封装

封装是指利用抽象数据类型，将数据和基于数据的操作封装在一起，形成一个相对独立的实体。

### 主要特点

- 数据被保护在对象内部，外部不能随意直接访问。
- 对外只暴露必要接口，隐藏内部实现细节。
- 调用方只需要关心对象提供了什么能力，不需要关心能力如何实现。

### 示例

```java
public class User {
    private String name;

    public String getName() {
        return name;
    }

    public void changeName(String name) {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("name cannot be blank");
        }
        this.name = name;
    }
}
```

上面代码通过 `private` 隐藏字段，通过方法控制修改规则，这就是封装的典型体现。

## 继承

继承用于表达类之间的 `is-a` 关系。子类可以获得父类非 `private` 的属性和方法，从而实现代码复用。

### 主要特点

- 子类继承父类，父类也称为超类或基类。
- 子类可以复用父类已有逻辑，也可以重写父类方法。
- 继承应该遵循里氏替换原则：子类对象必须能够替换父类对象，并保持程序语义正确。

### 示例

```java
class Animal {
    public void eat() {
        System.out.println("animal eat");
    }
}

class Dog extends Animal {
    public void bark() {
        System.out.println("dog bark");
    }
}
```

`Dog` 是一种 `Animal`，所以可以通过继承复用 `Animal` 的公共行为。

## 多态

多态表示同一个调用在不同对象上可以表现出不同结果。Java 中常见的多态包括编译时多态和运行时多态。

| 类型 | 体现方式 | 决定时机 |
|---|---|---|
| 编译时多态 | 方法重载 | 编译期根据参数列表决定 |
| 运行时多态 | 方法重写 + 向上转型 | 运行期根据真实对象类型决定 |

运行时多态通常需要满足三个条件：

1. 存在继承或接口实现关系。
2. 子类重写父类方法。
3. 使用父类引用指向子类对象，也就是向上转型。

### 示例

```java
class Animal {
    public void speak() {
        System.out.println("animal sound");
    }
}

class Cat extends Animal {
    @Override
    public void speak() {
        System.out.println("喵");
    }
}

class Dog extends Animal {
    @Override
    public void speak() {
        System.out.println("汪");
    }
}

public class Demo {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.speak(); // 汪
    }
}
```

虽然引用类型是 `Animal`，但运行时真实对象是 `Dog`，所以调用的是 `Dog#speak()`。

## 总结

- 封装关注“如何隐藏细节、暴露接口”。
- 继承关注“如何复用父类能力、表达 is-a 关系”。
- 多态关注“如何用统一接口调用不同实现”。

## 面试复习版

### 核心概念
- 三大特性：封装、继承、多态。
- SOLID：单一职责、开闭、里氏替换、接口隔离、依赖倒置。

### 面试官想考什么
- 能否把原则落到代码设计。
- 是否理解多态和抽象的价值。

### 标准回答
面向对象用封装隐藏复杂度，用继承表达共性，用多态提升扩展性；SOLID 是降低耦合、提升可维护性的设计原则，不是机械套公式。

### 深挖追问
- 开闭原则如何实现？
- 里氏替换违反会有什么后果？
- 依赖倒置和依赖注入关系？

### 实战场景/代码示例
```java
interface DiscountPolicy { BigDecimal discount(BigDecimal amount); }
class OrderService { private final DiscountPolicy policy; }
```

### 易错点/总结
- 原则服务于复杂度控制，不能过度设计。
- 继承不是复用代码的唯一方式，组合常更灵活。
