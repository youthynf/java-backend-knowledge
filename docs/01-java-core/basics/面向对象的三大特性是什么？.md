# 面向对象的三大特性是什么？

面向对象的三大特性是什么？
封装
•  利用抽象数据类型将数据和基于数据的操作封装在一起，使其构成一个不可分割的独立实体；
•  数据被保护在抽象数据类型的内部，尽可能地隐藏内部的细节，只保留一些对外接口使之与外部发生联系。用户无需知道对象内部的细节，但可以通过对象对外提供的接口来访问该对象。

继承
•  继承实现了 IS-A 关系，继承者是子类，被继承的称为父类，子类可以获得父类非private的属性与方法；
•  继承应该遵循里氏替换原则，子类对象必须能够替换掉所有父类对象。

多态
•  多态区分编译时多态（方法的重载）和运行时多态（程序中定义的对象引用所指向的具体类型在运行期间才确定）；
•  运行时多态有三个条件：继承、覆盖(重写)、向上转型（代码以父类定义）。

## 面试复习版

### 核心概念
- 三大特性：封装、继承、多态。
- SOLID：单一职责、开闭、里氏替换、接口隔离、依赖倒置。

### 面试官想考什么
- 能否把原则落到代码设计。
- 是否理解多态和抽象的价值。

### 标准回答
面向对象用封装隐藏复杂度，用继承表达共性，用多态提升扩展性；SOLID 是降低耦合、提升可维护性的设计原则，不是机械套公式。

### 深挖追问
- 开闭原则如何实现？
- 里氏替换违反会有什么后果？
- 依赖倒置和依赖注入关系？

### 实战场景/代码示例
```java
interface DiscountPolicy { BigDecimal discount(BigDecimal amount); }
class OrderService { private final DiscountPolicy policy; }
```

### 易错点/总结
- 原则服务于复杂度控制，不能过度设计。
- 继承不是复用代码的唯一方式，组合常更灵活。
