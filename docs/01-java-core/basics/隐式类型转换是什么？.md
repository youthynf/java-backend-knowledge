# 隐式类型转换是什么？

## 核心概念

隐式类型转换是指 Java 编译器在不需要显式强制类型转换的情况下，自动完成类型转换。

Java 只会自动执行相对安全的转换，典型场景是**小范围类型向大范围类型转换**，例如 `int` 转 `long`、`float` 转 `double`。

## 基本规则

### 自动向上转换

当目标类型的取值范围更大、精度损失风险较低时，Java 可以隐式转换。

```java
int a = 10;
long b = a;
double c = b;
```

### 不能自动向下转换

当目标类型范围更小或可能损失精度时，Java 不会隐式转换，需要显式强制类型转换。

```java
long a = 10L;
// int b = a;      // 编译错误
int b = (int) a;   // 显式强转
```

## `float` 与 `double`

Java 中小数字面量默认是 `double` 类型。

```java
// float f1 = 1.1; // 编译错误：1.1 默认是 double
float f2 = 1.1f;   // 正确：1.1f 是 float 字面量
```

`double` 转 `float` 属于向下转换，可能造成精度损失，所以不能隐式完成。

## `short` 与复合赋值运算符

整数字面量 `1` 默认是 `int` 类型，表达式计算时也容易发生类型提升。

```java
short s1 = 1;
// s1 = s1 + 1; // 编译错误：s1 + 1 的结果是 int
s1 += 1;        // 正确：等价于 s1 = (short) (s1 + 1)
```

`+=` 这类复合赋值运算符会隐式包含一次强制类型转换，因此可以编译通过。

## 常见类型提升

在表达式运算中，`byte`、`short`、`char` 通常会先提升为 `int` 再参与运算。

```java
byte a = 1;
byte b = 2;
// byte c = a + b; // 编译错误：a + b 的结果是 int
byte c = (byte) (a + b);
```

## 总结

- Java 可以隐式执行小范围到大范围的转换。
- 大范围到小范围、可能损失精度的转换必须显式强转。
- 小数字面量默认是 `double`，整数字面量默认是 `int`。
- `+=` 等复合赋值运算符会隐式执行一次类型转换。
