# 自动装箱和自动拆箱是什么？

自动装箱和自动拆箱是什么？
自动拆箱和装箱是 Java 语言中的一项特性，它简化了基本数据类型和对应的包装类之间的转换操作，由编译器自动处理。自动装箱实际调用valueOf()，拆箱调用xxxValue()方法，这比直接使用基本类型有额外的方法调用开销。

基本概念
•  自动装箱：指的是Java编译器会自动将基本数据类型转换为对应的包装类对象。例如，把int类型转换为Integer对象，double类型转换为Double对象等。
•  自动拆箱：指的是Java编译器会自动将包装类对象转换为对应的基本数据类型。例如，将Integer对象转换为int类型，Double对象转换为double类型。

原理
•  自动装箱：在进行自动装箱时，Java编译器会调用包装类的valueOf()方法。例如，对于Integer类型，编译器会将int类型的变量通过Integer.valueOf()方法转换为Integer对象。
•  自动拆箱：在进行自动拆箱时，编译器会调用包装类的xxxValue()方法，其中xxx是对应的基本数据类型。例如，对于Integer对象，编译器会调用intValue()方法将其转换为int类型。

## 面试复习版

### 核心概念
- Java 有 8 种基本类型，包装类型用于对象语义、集合和泛型。
- 装箱通常调用 valueOf，Integer 默认缓存 -128~127；拆箱调用 xxxValue。

### 面试官想考什么
- 基本类型大小、默认值和包装类型区别。
- 包装类型 ==、equals、缓存和 NPE 风险。

### 标准回答
基本类型直接保存值，包装类型是对象。自动装箱/拆箱提升编码便利性，但比较时要区分引用相等和值相等；包装对象可能为 null，拆箱会抛出 NullPointerException。

### 深挖追问
- 为什么集合不能直接存 int？
- Integer 127 和 128 的 == 为什么不同？
- char 能不能存中文？

### 实战场景/代码示例
```java
Integer a=127,b=127,c=128,d=128;
System.out.println(a==b); // 通常 true
System.out.println(c==d); // 通常 false
```

### 易错点/总结
- 业务比较不要依赖包装类型 ==。
- 金额不要用 float/double，优先 BigDecimal。
