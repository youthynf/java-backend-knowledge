# 自动装箱和自动拆箱是什么？

## 核心概念

自动装箱和自动拆箱是 Java 编译器提供的语法糖，用于简化基本数据类型和包装类型之间的转换。

- **自动装箱**：把基本数据类型自动转换成对应的包装类对象。
- **自动拆箱**：把包装类对象自动转换成对应的基本数据类型。

本质上，这些转换会在编译阶段被改写成对应方法调用。

## 自动装箱

自动装箱时，编译器通常会调用包装类的 `valueOf()` 方法。

```java
Integer a = 10;

// 编译器大致会改写成：
Integer b = Integer.valueOf(10);
```

常见包装类型都有类似的 `valueOf()` 方法，例如 `Long.valueOf()`、`Boolean.valueOf()`。

## 自动拆箱

自动拆箱时，编译器会调用包装类型对应的 `xxxValue()` 方法。

```java
Integer a = Integer.valueOf(10);
int b = a;

// 编译器大致会改写成：
int c = a.intValue();
```

## 常见场景

### 赋值转换

```java
Integer a = 1; // 自动装箱
int b = a;     // 自动拆箱
```

### 运算转换

```java
Integer a = 1;
Integer b = 2;
int c = a + b; // a 和 b 会先自动拆箱
```

### 集合场景

Java 泛型不支持基本类型，因此集合中存放基本类型值时会发生自动装箱。

```java
List<Integer> list = new ArrayList<>();
list.add(1);        // 自动装箱
int value = list.get(0); // 自动拆箱
```

## 注意事项

### 空指针风险

包装类型为 `null` 时，自动拆箱会触发 `NullPointerException`。

```java
Integer value = null;
// int result = value; // NullPointerException
```

### 性能开销

自动装箱会创建或复用包装对象，自动拆箱会发生方法调用。频繁装箱、拆箱可能带来额外开销。

### 对象比较风险

包装类型比较数值时，应该优先使用 `equals()`，不要依赖 `==`。

```java
Integer a = 128;
Integer b = 128;
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
```

## 总结

自动装箱和自动拆箱提升了代码可读性，但它们不是零成本特性。实际开发中要注意空指针、性能开销和包装对象比较问题。
