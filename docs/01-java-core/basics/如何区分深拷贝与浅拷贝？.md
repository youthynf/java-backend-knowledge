# 如何区分深拷贝与浅拷贝？

## 核心概念

深拷贝和浅拷贝的区别在于：**复制对象时，是否连同对象内部引用的子对象一起复制**。

- **浅拷贝**：只复制当前对象本身，引用类型字段仍然指向原来的子对象。
- **深拷贝**：不仅复制当前对象，还会递归复制其引用的子对象，形成完全独立的对象图。

判断标准很简单：修改拷贝对象中的引用子对象，是否会影响原对象。

## 浅拷贝

浅拷贝会创建一个新的对象，然后把当前对象的非静态字段复制到新对象中。

### 特点

- 如果字段是基本类型，直接复制值。
- 如果字段是引用类型，只复制引用地址，不复制引用指向的对象。
- 原对象和副本对象会共享同一个子对象。

### 常见实现方式

- 实现 `Cloneable` 接口。
- 重写 `clone()` 方法。
- 使用构造方法或工具类复制对象字段，但引用字段仍然复用原对象。

```java
class Address {
    String city;
}

class User implements Cloneable {
    String name;
    Address address;

    @Override
    protected User clone() throws CloneNotSupportedException {
        return (User) super.clone(); // 浅拷贝
    }
}
```

如果修改副本中的 `address.city`，原对象中的 `address.city` 也会受影响。

## 深拷贝

深拷贝会创建一个新对象，并递归复制当前对象引用的所有子对象。

### 特点

- 原对象和副本对象不共享引用子对象。
- 修改副本对象不会影响原对象。
- 适合对象内部存在可变引用字段，并且需要完全隔离副本的场景。

### 常见实现方式

- 手动递归拷贝所有引用字段。
- 通过序列化和反序列化生成副本。
- 使用 JSON、Bean Mapping、第三方工具类等方式完成对象图复制。

```java
class Address {
    String city;

    Address(String city) {
        this.city = city;
    }

    Address copy() {
        return new Address(this.city);
    }
}

class User {
    String name;
    Address address;

    User copy() {
        User user = new User();
        user.name = this.name;
        user.address = this.address == null ? null : this.address.copy();
        return user;
    }
}
```

## 对比总结

| 对比项 | 浅拷贝 | 深拷贝 |
|---|---|---|
| 是否创建新对象 | 是 | 是 |
| 是否复制基本类型字段 | 是 | 是 |
| 是否复制引用字段指向的对象 | 否 | 是 |
| 是否共享子对象 | 是 | 否 |
| 修改副本子对象是否影响原对象 | 可能影响 | 不影响 |
| 实现复杂度 | 较低 | 较高 |

## 总结

- 浅拷贝复制“对象壳”和字段值，但引用字段仍然共享。
- 深拷贝复制完整对象图，原对象和副本对象相互独立。
- 面试中可以用一句话回答：**看引用类型字段是否仍指向同一个对象，指向同一个就是浅拷贝，完全独立就是深拷贝。**
