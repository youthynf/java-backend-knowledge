# CIDR 无分类地址是什么

## 核心概念

CIDR（Classless Inter-Domain Routing，无类别域间路由，RFC 4632）是 1993 年提出的 IP 地址划分方式，用来替代 1981 年的 A/B/C/D/E 分类地址（RFC 791）。它解决的核心问题是：**分类地址过于死板，要么网络太大浪费、要么太小不够用**。CIDR 打破"网络号必须是 8/16/24 位"的限制，允许前缀长度取 0-32 任意值，让 IP 分配可以按需切分、按需聚合。

CIDR 的记法是 `IP/前缀长度`，例如 `192.168.1.0/24` 表示前 24 位是网络前缀、后 8 位是主机号。前缀长度越短，能容纳的主机越多（`/24` 容 254 台主机，`/16` 容 65534 台）。和它配套的还有 VLSM（可变长子网掩码，RFC 1812）和路由聚合（Route Aggregation / Supernetting）。

理解 CIDR 是看懂云上 VPC、容器网络（Flannel、Calico）、Kubernetes Service CIDR、安全组规则的基础。这一节也是面试中"给你一个 192.168.0.0/16 切出 N 个 /24 子网"这类题的解题钥匙。

## 标准回答

CIDR 取消了 A/B/C 类边界，IP 地址的网络前缀长度可以是任意值，记作 `a.b.c.d/N`，N 表示前 N 位是网络号。子网掩码就是把前 N 位置 1、其余置 0 得到的 32 位数，例如 `/24` 对应掩码 `255.255.255.0`。判断两个 IP 是否同网段，就是把它们各自与掩码按位与，结果相同即同网段。

要点：

- CIDR 让 IP 分配从"固定档位"变成"任意档位"，提升地址利用率，缓解路由表膨胀。
- 子网号（主机位全 0）和广播地址（主机位全 1）不可分配给主机，所以 `/N` 网段可用主机数 = `2^(32-N) - 2`。
- VLSM 允许同一组织内不同子网使用不同前缀长度，按部门大小切分。
- 路由聚合把多个连续小网段合并成一个大网段对外宣告，缩小全球 BGP 表。
- IPv6 也用 CIDR 记法，前缀长度范围 0-128，常见 `/64`、`/48`、`/128`。

## 实现原理

### 子网掩码与 CIDR 的等价关系

CIDR 前缀长度与子网掩码一一对应：

| CIDR | 子网掩码 | 主机位数 | 可用主机数 | 典型场景 |
|------|---------|---------|-----------|---------|
| /8   | 255.0.0.0 | 24 | 16,777,214 | A 类历史范围 |
| /16  | 255.255.0.0 | 16 | 65,534 | 大型企业、VPC |
| /24  | 255.255.255.0 | 8 | 254 | 传统 C 类、办公室网段 |
| /25  | 255.255.255.128 | 7 | 126 | 小型办公网段 |
| /26  | 255.255.255.192 | 6 | 62 | 点对点链路或小部门 |
| /30  | 255.255.255.252 | 2 | 2 | 路由器互联链路 |
| /31  | 255.255.255.254 | 1 | 2（RFC 3021） | 点对点链路（无广播） |
| /32  | 255.255.255.255 | 0 | 1 | 主机路由、VIP、安全组 |

### 同网段判定

`192.168.1.100/24` 与 `192.168.1.200/24`：

```
192.168.1.100  11000000.10101000.00000001.01100100
192.168.1.200  11000000.10101000.00000001.11001000
掩码 /24       11111111.11111111.11111111.00000000
两者与掩码相与都得 192.168.1.0，所以同网段
```

`192.168.1.100/24` 与 `192.168.2.100/24`：

```
192.168.1.0 != 192.168.2.0，不同网段，需要经过路由器
```

### VLSM 子网划分示例

把 `192.168.0.0/24` 切成满足以下需求的子网：A 部门 100 台、B 部门 50 台、C 部门 25 台、3 条路由器互联链路：

1. A 部门需 100 台 → 至少 7 位主机位（2^7-2=126）→ `192.168.0.0/25`，范围 `.0`-`.127`，可用 `.1`-`.126`。
2. B 部门需 50 台 → 至少 6 位（2^6-2=62）→ `192.168.0.128/26`，范围 `.128`-`.191`，可用 `.129`-`.190`。
3. C 部门需 25 台 → 至少 5 位（2^5-2=30）→ `192.168.0.192/27`，范围 `.192`-`.223`，可用 `.193`-`.222`。
4. 路由器互联 → `/30` 即可 → `192.168.0.224/30`、`192.168.0.228/30`、`192.168.0.232/30`。

这种"先大切小、按需切分"的能力是分类地址做不到的——A 类历史范围 `10.0.0.0/8` 容纳 1600 万主机，按 B 类 `/16` 切就是 256 个等长子网，根本无法精细分配。

### 路由聚合（Supernetting）

如果某 ISP 拿到了 `200.16.0.0/24`、`200.16.1.0/24`、`200.16.2.0/24`、`200.16.3.0/24` 四个连续 /24，可以聚合成 `200.16.0.0/22` 对外宣告，全球 BGP 表只增长 1 条而不是 4 条。聚合的前提是地址连续、前缀公共部分一致：

```
200.16.0.0  11001000.00010000.00000000.00000000
200.16.1.0  11001000.00010000.00000001.00000000
200.16.2.0  11001000.00010000.00000010.00000000
200.16.3.0  11001000.00010000.00000011.00000000
公共前 22 位相同 → 200.16.0.0/22
```

### 私有 IP 范围（RFC 1918）

无论分类还是 CIDR，私有 IP 范围都是这三段：

| 网段 | 范围 | 默认前缀 |
|------|------|---------|
| 10.0.0.0 - 10.255.255.255 | A 类私有 | /8 |
| 172.16.0.0 - 172.31.255.255 | B 类私有 | /12 |
| 192.168.0.0 - 192.168.255.255 | C 类私有 | /16 |

其他特殊地址：

- `0.0.0.0/8`：本网本机，`0.0.0.0` 作为源地址表示"未配置 IP"。
- `127.0.0.0/8`：环回地址（loopback），仅本机使用。
- `169.254.0.0/16`：链路本地地址，DHCP 失败时自动配置（APIPA）。
- `224.0.0.0/4`：组播地址（D 类）。
- `240.0.0.0/4`：保留（E 类）。
- `255.255.255.255`：受限广播地址。

## 代码示例

Linux 下用 `ipcalc` 计算子网信息：

```bash
# 计算一个 CIDR 网段的网络号、广播、可用主机范围
ipcalc 192.168.1.100/24

# 把一个 /24 切成 4 个 /26
ipcalc 192.168.1.0/24 26 4

# 用 ip 命令查看本机路由与所属网段
ip route show
ip addr show eth0
```

Java 中计算 CIDR 是否包含某个 IP（常用于 IP 白名单、限流规则）：

```java
import java.net.InetAddress;
import java.net.UnknownHostException;

public class CidrMatcher {
    private final int network;   // 网络号（无符号 int）
    private final int prefix;    // 前缀长度

    public CidrMatcher(String cidr) throws UnknownHostException {
        String[] parts = cidr.split("/");
        byte[] addr = InetAddress.getByName(parts[0]).getAddress();
        this.prefix = Integer.parseInt(parts[1]);
        this.network = toInt(addr) & mask(prefix);
    }

    public boolean contains(String ip) throws UnknownHostException {
        int target = toInt(InetAddress.getByName(ip).getAddress());
        return (target & mask(prefix)) == network;
    }

    private static int toInt(byte[] b) {
        return ((b[0] & 0xFF) << 24) | ((b[1] & 0xFF) << 16)
             | ((b[2] & 0xFF) << 8)  | (b[3] & 0xFF);
    }

    private static int mask(int prefix) {
        return prefix == 0 ? 0 : (int) (0xFFFFFFFFL << (32 - prefix));
    }

    public static void main(String[] args) throws Exception {
        CidrMatcher m = new CidrMatcher("192.168.1.0/24");
        System.out.println(m.contains("192.168.1.100"));  // true
        System.out.println(m.contains("192.168.2.100"));  // false
    }
}
```

## 实战场景

| 场景 | 用法 | 注意点 |
|------|------|--------|
| VPC 规划 | 主 CIDR `10.0.0.0/16`，按可用区切 `10.0.0.0/24`、`10.0.1.0/24` | 预留扩容空间，避免后续容器网段冲突 |
| Kubernetes Service CIDR | 与 Pod CIDR 隔离，常见 `10.96.0.0/12` | 不要与节点网段重叠，否则路由冲突 |
| 安全组规则 | 用 CIDR 限制源 IP，如只允许 `10.10.0.0/16` 访问 3306 | 不要写 `0.0.0.0/0` 开放内网端口 |
| 路由器互联 | `/30` 或 `/31` 节省地址 | `/31` 需两端设备支持 RFC 3021 |
| IP 白名单 / 限流 | 用 CIDr matcher 判断客户端是否在内网 | 注意 IPv4 与 IPv6 双栈，分别匹配 |

## 深挖追问

- **CIDR 出现前路由表有多糟糕？**
  90 年代初互联网爆发，B 类地址快耗尽，C 类地址空间碎裂导致全球 BGP 路由表暴涨（每年翻倍），路由器内存与 CPU 跟不上。CIDR 通过路由聚合把数千条 C 类路由压缩成一条 `/16` 或 `/15`，挽救了当时的互联网。

- **/31 是怎么做到没有广播地址的？**
  RFC 3021 规定点对点链路不需要广播，两个地址都给两端主机用，节省一半地址。但 `/31` 不能用于多接入网段（以太网广播域），只能用于路由器互联。

- **VLSM 与 CIDR 是一回事吗？**
  严格不同。VLSM 是"一个组织内部"用不同掩码切分子网；CIDR 是"互联网层面"取消类别的地址与路由方式。但两者依赖同一套机制（任意前缀长度），所以常一起讲。

- **为什么 Kubernetes 默认 Service CIDR 用 /12？**
  `/12` 容纳约 100 万个 Service IP，覆盖中大型集群需求。Pod CIDR 通常用 `/16`（约 6.5 万 IP），按节点切 `/24` 给每个节点。

- **IPv6 还需要 CIDR 吗？**
  需要。IPv6 地址 128 位，依然按前缀长度切分，常用 `/64` 作为终端网段、`/48` 作为站点分配、`/32` 作为 ISP 分配。CIDR 机制对 IPv6 完全适用。

## 易错点

- **忘记扣掉网络号和广播地址**：`/24` 可用主机是 254，不是 256；`/30` 是 2 不是 4。
- **以为 `192.168.1.0/24` 的 0 和 255 可以给主机用**：0 是网络号、255 是广播，不可分配。但 `/31`、`/32` 例外。
- **混淆 CIDR 前缀与子网掩码位数**：`/24` 对应 24 个 1，不是 24 个 0；掩码写法 `255.255.255.0`。
- **以为 VLSM 可以任意切分**：必须按 2 的幂对齐，否则不连续无法聚合。
- **以为私有 IP 跨公网能直接路由**：RFC 1918 私有 IP 在公网不可路由，必须通过 NAT。

## 总结

CIDR 是互联网从"分类地址"走向"灵活寻址"的关键一步，让 IP 分配和路由聚合都按需而行。掌握 CIDR 的关键在于把"前缀长度 + 主机位数"这套位运算玩熟：`/N` 对应掩码、可用主机数、网络号计算。云上 VPC、容器网络、安全组规则、IP 白名单都建立在 CIDR 之上。和 ARP、NAT、DHCP 这些"具体机制"不同，CIDR 更像一套"地址数学"，是理解现代网络的基础工具。
