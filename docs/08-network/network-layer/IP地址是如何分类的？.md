# IP地址是如何分类的？

IP地址是如何分类的？
在TCP/IP网络通信时，为了保证正常通信，每个设备都需要配置正确的IP地址，否则无法实现正常的通信。IP地址（IPV4地址）由32位正整数来表示，IP地址在计算机是以二进制的方式进行处理的。人类为了方便记忆，采用点分十进制等方式进行标记，将32位IP地址分为4组，每组8位并使用十进制数展示，组与组之间通过"."隔开。

IP地址可以分为五类，分别是A、B、C、D、E类：
ABC类IP地址，由网络号和主机号两部分组成，网络号位数依次为8位，16位，24位；
A类首位网络号为0，B类第二位网络号为0，C类第三位网络号为0，D类IP地址第四位为0，E类前四位IP地址为1；
ABC类主机地址全为0，表示网络号，全为1表示广播地址；
D类地址为多播地址，E类地址为预留地址；

区分广播地址和多播地址：
广播地址有区分本地广播地址（本网络内的广播，不允许跨路由器）和直接广播地址（指定网络内的广播，允许跨路由器，但需要配置）；
多播地址是指发到到特定组内的设备，允许跨路由器，并且需要配置加入组内广播。

## 面试总结
### 核心概念

网络层负责主机到主机的寻址和路由。IP 解决跨网络寻址，ARP 解决局域网 IP 到 MAC 的映射，ICMP 用于差错和诊断，DHCP 分配地址，NAT 做地址转换，CIDR 提高地址划分灵活性。

### 面试官想考什么

面试官想看你能否把打开网页、跨网段通信、内网访问公网、排查 ping/traceroute 串起来。

### 标准回答

同网段通信先通过 ARP 找目标 MAC；跨网段则发给默认网关，由路由器逐跳转发。DHCP 帮主机获取 IP、网关、DNS；NAT 让内网地址复用公网地址；ICMP 支持 ping 和网络差错反馈。IPv6 提供更大地址空间并简化部分机制。

### 实战场景/示例

容器或云主机无法访问外网时，要检查 IP/掩码、路由、网关、DNS、安全组和 NAT 出口。

### 易错点/总结

IP 地址负责逻辑寻址，MAC 地址负责链路层转发；不要把二者混为一谈。
