# 中断是什么？

中断是什么？
中断是指CPU停下当前的工作任务，去处理其他事情，处理完后回来继续执行刚才的任务。
中断分为外部中断和内部中断，中断事件来自CPU外部的称为外部中断，来自CPU内部的则为内部中断：
外部中断：外部中断分为可屏蔽中断和不可屏蔽中断。
可屏蔽中断是指通过INTR线向CPU请求的中断，主要来自于外部设备，如硬盘、打印机、网卡等，此类中断并不会影响系统运行，可随时处理，甚至不处理，所以名为可屏蔽；
不可屏蔽中断是指通过NMI线向CPU请求的中断，如电源掉电、硬件线路故障等，这里不可屏蔽的意思不是不可以屏蔽或不建议屏蔽，而是问题太大无法屏蔽的意思；

内部中断：内部中断分为陷阱、故障、终止。
陷阱是指一种有意的，预先安排的异常事件，一般是在编写程序时故意设下的陷阱指令，当执行到陷阱指令后，CPU将会调用特定程序进行相应处理，处理结束后回到陷阱指令的下一条指令继续执行，如系统调用、程序调试功能等，如print函数，最底层的实现会中会有一条int 0x80指令，这就是一条陷阱指令，使用0x80号中断进行系统调用；
故障是在引起故障的指令被执行，但还没有执行结束时，CPU检测到的一类意外事件。出错时交由故障处理程序处理，如果能处理修正这个错误，CPU重新执行这条指令，如果不能处理就报错。常见的故障是缺页，当CPU引用的虚拟地址不存在时会发生故障，缺页异常时能够修正的，有专门的缺页处理程序，它将缺失的物理页从磁盘中重新调进主存，而后再次执行引起故障的指令时便能够顺利执行了；
终止是指执行指令的过程中发生了致命错误，不可修复，程序无法继续执行，只能终止，通常是一些硬件错误，终止处理程序不会将控制返回给源程序，而是直接终止原程序。
