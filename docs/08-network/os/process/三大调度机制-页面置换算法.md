# 三大调度机制-页面置换算法

三大调度机制-页面置换算法
前置知识：缺页异常（缺页中断）
当 CPU 访问的页面不在物理内存时，便会产生一个缺页中断，请求操作系统将所缺页调入到物理内存。那它与一般中断的主要区别在于：
缺页中断在指令执行「期间」产生和处理中断信号，而一般中断在一条指令执行「完成」后检查和处理中断信号。
缺页中断返回到该指令的开始重新执行「该指令」，而一般中断返回回到该指令的「下一个指令」执行。

缺页中断处理流程：
在 CPU 里访问一条 Load M 指令，然后 CPU 会去找 M 所对应的页表项。
如果该页表项的状态位是「有效的」，那 CPU 就可以直接去访问物理内存了，如果状态位是「无效的」，则 CPU 则会发送缺页中断请求。
操作系统收到了缺页中断，则会执行缺页中断处理函数，先会查找该页面在磁盘中的页面的位置。
找到磁盘中对应的页面后，需要把该页面换入到物理内存中，但是在换入前，需要在物理内存中找空闲页
如果找到空闲页，就把页面换入到物理内存中。
页面从磁盘换入到物理内存完成后，则把页表项中的状态位修改为「有效的」。
最后，CPU 重新执行导致缺页异常的指令。
如果第5步发现在物理内存中找不到空闲页时，就需要页面置换算法选择一个物理页，如果该物理页有被修改过（脏页），则把它换出到磁盘，然后把该被置换出去的页表项的状态改成「无效的」，最后把正在访问的页面装入到这个物理页中。
页表项通常包含：页号、物理页号、状态位、访问字段、修改位、硬盘地址
其中：
状态位：用于表示该页是否有效，也就是说是否在物理内存中，供程序访问时参考。
访问字段：用于记录该页在一段时间被访问的次数，供页面置换算法选择出页面时参考。
修改位：表示该页在调入内存后是否有被修改过，由于内存中的每一页都在磁盘上保留一份副本，因此，如果没有修改，在置换该页时就不需要将该页写回到磁盘上，以减少系统的开销；如果已经被修改，则将该页重写到磁盘上，以保证磁盘中所保留的始终是最新的副本。
硬盘地址：用于指出该页在硬盘上的地址，通常是物理块号，供调入该页时使用。

页面置换算法的功能是，当出现缺页异常，需调入新页面而内存已满时，选择被置换的物理页面，也就是说选择一个物理页面换出到磁盘，然后把需要访问的页面换入到物理页。那其算法目标则是，尽可能减少页面的换入换出的次数。
常见的页面置换算法：
最佳页面（OPT）：置换未来最长时间不访问的页面。但是因为无法预知每个页面在下一次访问前的等待时间，所以现实无法实现；
先进先出（FIFO）：选择内存驻留时间很长的页面进行置换，有可能会将访问频繁但是驻留时间长的页面误淘汰，所以性能相对较差；
最近最久未使用（LRU）：选择最长时间没有被访问的页面进行置换，但是成本较高，需要维护所有页面的链表，并涉及链表的删除和移动，开销大，耗时大；
时钟页面（Lock）：可以看做是LRU和FIFO的综合，把所有页面保存在一个类似钟面的环形链表，一个表针指向最老的页面。当发生缺页中断时，算法首先检查表针指向的页面，如果访问位为0就淘汰该页，并把新页插入这个位置，然后把表针前移一个位置；如果访问位是1就清除该访问位，并把表针前移一位，重复这个过程知道找到一个访问位位0的页面位置；
最不常用（LFU）：选择访问次数最少的页面，将其淘汰。需要对每个页面维护一个访问计数器，每当一个页面被访问，该页面的访问计数器加1，发生缺页中断时，淘汰计数器值最小的那个页面。硬件成本较高，页面需要通过链表维护，长度很大时耗时高，只考虑了频率问题，没考虑时间问题，有些页面过去时间访问频率很高，但是现在已经没有访问了，而当前频繁访问的页面没有这些页面次数高会被误伤；LFU优化方案是定期减少访问次数，如当发生时间中断，则把过去时间访问的页面的访问次数除以2，随着时间流逝，以前高访问次数的页面会慢慢减少。相当于综合考量频次和频率。
