# 进程上下文切换指的是什么？

进程上下文切换指的是什么？
进程上下文切换
各个进程之间是共享 CPU 资源的，在不同的时候进程之间需要切换，让不同的进程可以在 CPU 执行，那么这个一个进程切换到另一个进程运行，称为进程的上下文切换。

CPU上下文切换
任务是交给 CPU 运行的，那么在每个任务运行前，操作系统需要事先帮 CPU 设置好 CPU 寄存器和程序计数器。CPU 寄存器和程序计数是 CPU 在运行任何任务前，所必须依赖的环境，这些环境就叫做 CPU 上下文。CPU 上下文切换就是先把前一个任务的 CPU 上下文（CPU 寄存器和程序计数器）保存起来，然后加载新任务的上下文到这些寄存器和程序计数器，最后再跳转到程序计数器所指的新位置，运行新任务。系统内核会存储保持下来的上下文信息，当此任务再次被分配给 CPU 运行时，CPU 会重新加载这些上下文，这样就能保证任务原来的状态不受影响，让任务看起来还是连续运行。这里说的所谓的「任务」，主要包含进程、线程和中断。所以，可以根据任务的不同，把 CPU 上下文切换分成：进程上下文切换、线程上下文切换和中断上下文切换。

进程上下文切换的内容
进程是由内核管理和调度的，所以进程的切换只能发生在内核态。所以，进程的上下文切换不仅包含了虚拟内存、栈、全局变量等用户空间的资源，还包括了内核堆栈、寄存器等内核空间的资源。通常，会把交换的信息保存在进程的 PCB，当要运行另外一个进程的时候，我们需要从这个进程的 PCB 取出上下文，然后恢复到 CPU 中，这使得这个进程可以继续执行。上下文切换中需要保存和恢复以下内容：
CPU 寄存器：包括程序计数器（Program Counter）、栈指针（Stack Pointer）和通用寄存器。
程序状态字：包括中断状态、控制寄存器等。
内存映射：包括页表信息等，用于管理进程的虚拟内存。
内核栈：每个进程在内核中都有独立的栈，用于存储内核执行过程中的中间变量。
调度信息：如进程控制块（PCB）中的调度状态。

进程上下文切换的场景
当某个进程的时间片耗尽了，进程就从运行状态变为就绪状态，系统从就绪队列选择另外一个进程运行；
进程在系统资源不足（比如内存不足）时，要等到资源满足后才可以运行，这个时候进程也会被挂起，并由系统调度其他进程运行；
当进程通过睡眠函数 sleep 这样的方法将自己主动挂起时，自然也会重新调度；
当有优先级更高的进程运行时，为了保证高优先级进程的运行，当前进程会被挂起，由高优先级进程来运行；
发生硬件中断时，CPU 上的进程会被中断挂起，转而执行内核中的中断服务程序；

进程上下文切换的过程
上下文切换通常由操作系统内核完成，具体步骤如下：
保存当前进程的状态：操作系统暂停当前正在执行的进程，将该进程的寄存器、程序计数器等状态信息保存在进程控制块（PCB）中，其中PCB 是操作系统用于管理进程的重要数据结构，保存了进程的所有关键状态；
切换到新的进程：操作系统根据调度算法（如时间片轮转、优先级调度）选择下一个要执行的进程，加载新进程的 PCB，将其状态信息恢复到 CPU 寄存器和内存中；
恢复新进程的执行：操作系统将 CPU 控制权交给新进程，恢复其上下文，开始执行该进程。

进程上下文切换的开销
时间开销：保存和恢复上下文需要操作 CPU 寄存器、内核栈、页表等，增加了系统开销；
缓存失效：切换进程时，CPU 缓存中存储的上一个进程的数据可能被新进程覆盖，导致缓存命中率下降；
内存管理：切换进程时，内核需要加载新的页表，可能会导致 TLB（Translation Lookaside Buffer）失效。
