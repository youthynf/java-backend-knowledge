# 线程间通信方式有哪些？

线程间通信方式有哪些？
Linux系统提供了五种用于线程通信的方式：互斥锁、读写锁、条件变量、自旋锁和信号量。

1.互斥锁（Mutex）:
定义：互斥量（mutex）从本质上说是一把锁；
通信过程：当访问共享资源前对互斥量进行加锁，在访问完释放互斥量的锁；对互斥量加锁后，任何其他线程试图再次对互斥量加锁都会被阻塞直到加锁成功的线程释放该互斥量。如果释放互斥量时有多个线程阻塞，这些阻塞线程都会变成可运行状态，第一个变为运行状态的线程可以对互斥量加锁，其他线程将会看到互斥量依然被锁住，只能再次等待它重新变成可用。

2.条件变量（Condition Variables）：
定义：条件变量是在多线程程序中用来实现“等待→唤醒”逻辑常用的方法；
通信过程：条件变量利用线程间共享的全局变量进行同步的一种机制，主要包括两个动作，一个是线程等待条件变量成立而挂起，另一个线程使条件成立。为了防止竞争，条件变量的使用总是和一个互斥锁结合在一起。

3.自旋锁（Spinlock）：
定义：自旋锁通过CPU提供的CAS函数（compare and swap），在用户态完成加锁和解锁操作，不会主动产生线程上下文切换；
通信过程：首先查看锁的状态，如果锁是空闲则执行第二步，第二步将锁设置为当前线程持有。使用自旋锁的时候，当发生多线程进程锁的情况，加锁失败的线程会忙等待，直到拿到锁。CAS将这两个步骤合并成一条硬件级指令，形成原子指令，保证两个操作不可分割。这里的忙等待可以使用while循环等待来实现，不过最好使用CPU提供的PAUSE指令来实现忙等待。

4.信号量：
定义：信号量可以是命名的（有名信号量）或无名的（仅限当前进程内的线程），用于控制对资源的访问次数。通常信号量表示资源的数量，对应的变量是一个整型变量。
通信过程：除了信号量外，另外还有两个原子操作的系统调用函数来控制信号量，分别是P操作（将整型变量减1，如果结果小于0则进程/线程进入阻塞等待，否则继续执行）和V操作（将整型变量加1，如果结果小于等于0，唤醒一个等待中的进程/线程）；

5.读写锁：
定义：读写锁有读锁和写锁两部分构成，如果只读取共享资源用读锁，如果修改共享资源则用写锁。读写锁适用于能明确区分读操作和写操作的场景。写锁是独占锁，读锁是共享锁。读写锁在读多写少场景能发挥出优势。
