# 进程、线程、协程、管程有什么区别？

进程、线程、协程、管程有什么区别？
进程
我们编写的代码只是一个存储在硬盘的静态文件，通过编译后就会生成二进制可执行文件，当我们运行这个可执行文件后，它会被装载到内存中，接着 CPU 会执行程序中的每一条指令，那么这个运行中的程序，就被称为「进程」（Process）。进程是计算机中正在运行的程序的实例，是操作系统进行资源分配和调度的基本单位，拥有自己独立的地址空间（包括代码段、数据段、堆栈段等），可以看做是一个独立的运行环境，拥有独立的系统资源，如内存、文件句柄等。进程间通信需要特定的机制，如管道、消息队列、共享内存、信号量、信号、Socket等。由于进程拥有独立的内存空间，因此其稳定性和安全性较高，但是同时上下文切换的开销也大，因为需要保存和恢复整个进程的状态。

线程
线程是进程内的一个执行单元，是进程内部的可执行路径，是CPU调度和分派的基本单位。一个进程可以包含多个线程，这些线程共享进程的地址空间以及其他资源（如已打开的文件、全局变量等），但每个线程都有自己独立的栈空间和程序计数器（用于记录当前执行指令的位置）。线程之间通信高效，因为它们可以直接读写共享内存。线程的上下文开销较小，只需要保存和恢复线程的上下文，而不是整个进程的状态。但是由于多个线程共享内存，因此存在数据竞争和线程安全的问题，需要同步和互斥机制来解决。

协程
协程是一种用户态的轻量级线程，其调度完全由用户程序控制，而不需要内核的参与。协程拥有自己的寄存器上下文和栈，而无需进行内核级的上下文切换。协程在执行时可以主动让出执行权，暂停自身的执行，等合适的时机再恢复执行，并且多个协程之间可以按照设定的顺序或者逻辑进行切换，这使得协程在处理大量并发任务时具有非常高的效率。然而协程需要程序员显示进行调度和管理，编程模型更为复杂。

管程
管程是一种高级的并发编程结构，主要是对共享资源进行管理和控制访问的一种机制，用于管理共享资源以及协调多个并发进程和线程对这些资源的访问。它把共享资源以及对这些资源的操作封装在一个抽象的数据结构中，同时提供了一套特定的机制（如条件变量等）来保证在同一时刻只有一个进程或线程能够访问共享资源，冰面并发访问导致的冲突和数据不一致问题。
