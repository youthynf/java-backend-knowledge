# 银行家算法是什么？

银行家算法是什么？
银行家算法是最优代表性的避免死锁的算法。银行家算法的业务逻辑是：
不负荷运行：一个进程的最大需求量不超过系统拥有的总资源数才被采纳执行（需求在银行能力范围内采接纳）；
可分期：一个进程可分期请求资源，但总请求数不可超过最大需求量（接收多笔资源申请，但总数是真实需求的数量）；
推迟分配：当系统现有资源小于进程需求时，对进程的需求可以延迟分配，但总是让进程在有限时间内获取资源（需要接受延迟资源分配，但能确保时效范围内处理完毕）。

银行家算法核心思想，就是在分配给进程资源前，首先判断这个进程的安全性，也就是预知性，判断分配后是否产生死锁。如果系统当前资源满足其执行，则尝试分配，如果不满足则让该进程等待。（核心过程）通过不断检查剩余可用资源是否满足进程的最大需求，如果可以则将该进程加入安全队列，并把该进程当前持有的资源回收，不断重复这个过程，看最后能否实现让所有进程都加入安全队列，安全队列一定不会发生死锁。
