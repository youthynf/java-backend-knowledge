# 写时复制的过程是怎么样的？

写时复制的过程是怎么样的？
1.页表复制
主进程在执行fork的时候，操作系统会把主进程的页表复制一份给子进程，这个页表记录着虚拟地址和物理地址的映射关系，而不会复制物理内存。也就是说，父子进程的虚拟空间不同，但是对应的物理空间是同一个，从而是子进程共享了父进程的物理内存数据，能够节省物理内存资源，页表对应的页表项的属性会标记物理内存的权限为只读。

2.修改内存
当父进程或子进程在向这个内存发起写操作时，CPU就会触发写保护中断，这个写保护中断是由于违反了权限导致的，然后操作系统会在写保护中断处理函数里进行物理内存的复制，并重新设置其内存映射关系，将父子进程的内存读写权限设置为可读写，最后才会对内存进行写操作，这个过程被称为写时复制。

总结：
写时复制在发生写操作的时候，操作系统才会去复制物理内存，这样是为了防止fork创建子进程时，由于物理内存数据的复制时间过长导致父进程长时间阻塞的问题。

注意：一旦触发写时复制，就会在新分配的物理内存中进行写操作，此时另一个进程的页表仍然指向原来的物理内存页面，访问的还是原来的数据，从而保证父子进程间数据的独立性，它们在对共享内存进行写操作后，各自操作自己独立的副本，不会互相影响。
