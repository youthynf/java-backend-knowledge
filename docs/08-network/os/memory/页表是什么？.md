# 页表是什么？

页表是什么？
概述
页表是操作系统中用于实现虚拟内存管理的一种重要数据结构，存储在内存，主要作用是将进程的虚拟地址映射到物理地址。页表由多个页表项组成，每个页表项对应一个虚拟页。页表项包含：
物理页帧号：存储了该虚拟页映射到的物理页帧的编号，通过该编号可以找到对应的物理内存位置；
有效位：表示该虚拟页是否已经映射到物理内存中，如果为 1 表示已映射，为 0 表示未映射；
保护位：表示该页的访问权限，如只读、读写、可执行等；
存在位：表示该页是否在物理内存中，如果为 1 表示在物理内存中，为 0 表示不在，可能在磁盘上（触发缺页异常）；
修改位：表示该页是否被修改过，如果被修改过，在换出时需要将其写回磁盘；
引用位：表示该页最近是否被访问过，可用于页面置换算法，如 LRU（Least Recently Used）算法，帮助决定哪些页面可以被换出；

内存分页
分页是把整个虚拟和物理内存空间切成一段段固定尺寸的大小。这样一个连续并且尺寸固定的内存空间就是页（page）。Linux中每个页的大小是4KB。内存分页由于内存空间时预先划分好的，也就不会像内存分段一样，在段与段之间产生间隙非常小的内存碎片。但是内存分页机制分配内存的最小单位是一页，即时程序不足一页大小，我们最少只能分配一个页，所以页内会出现内存浪费，存在内部内存碎片现象。

地址转换
内存管理单元（MMU）负责将虚拟地址转换成物理地址的工作，转换步骤：
当进程访问一个虚拟地址时，将该虚拟地址分成三个部分：页目录索引、页表索引和页内偏移；
首先，根据页目录索引在页目录中找到相应的页表地址；
然后，根据页表索引在页表中找到相应的页表项；
最后，根据页表项中的物理页帧号和页内偏移得到物理地址。
