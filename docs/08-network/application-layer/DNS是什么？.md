# DNS是什么？

DNS是什么？
DNS的全称是Domain Name System（域名系统），它是互联网中用于将域名转换为对应IP地址的分布式数据库系统。DNS扮演着重要的角色，使得人们可以通过易记的域名访问互联网资源，而无需记住复杂的IP地址。

DNS 中的域名都是用句点来分隔的，比如 www.server.com，这里的句点代表了不同层次之间的界限。在域名中，越靠右的位置表示其层级越高。实际上域名最后还有一个点，比如 www,server.com.，这个最后的一个点代表根域名，也就是，.根域是在最顶层，它的下一层就是.com 顶级域，再下面是 server.com。所以域名的层级关系类似一个树状结构：
根 DNS 服务器（.）
顶级域 DNS 服务器（.com）
权威 DNS 服务器（server.com）

根域的 DNS 服务器信息保存在互联网中所有的 DNS 服务器中，因此任何 DNS 服务器就都可以找到并访问根域 DNS 服务器了。因此，客户端只要能够找到任意一台 DNS 服务器，就可以通过它找到根域 DNS 服务器，然后再一路顺藤摸瓜找到位于下层的某台目标 DNS 服务器。

## 面试总结
### 核心概念

DNS 负责把域名解析为 IP 地址。解析链路可能经过浏览器/系统缓存、本地 DNS、根域、顶级域、权威 DNS；传输通常优先 UDP，响应过大、区域传送等场景会使用 TCP。

### 面试官想考什么

面试官常考 DNS 解析流程、缓存、UDP/TCP 使用场景、DNS 劫持以及它如何影响接口耗时。

### 标准回答

访问域名前先查本地缓存，未命中则向本地 DNS 递归查询；本地 DNS 可能迭代访问根、TLD、权威 DNS，最终返回记录并缓存。DNS 失败会导致连接还没建立就超时，Java 服务中要关注域名缓存、TTL 和连接池复用。

### 深挖追问

- 这个行为发生在浏览器、客户端库、代理网关还是后端服务？
- 如果接口偶发超时/失败，如何用 curl、DevTools、网关日志和 tcpdump 分层验证？
- 连接池、缓存、CDN、TLS 或反向代理配置会怎样改变现象？

### 实战场景/示例

服务调用突然大量超时但目标服务正常，可能是 DNS 解析慢、域名被劫持或 JVM DNS 缓存导致旧 IP 未刷新。

### 易错点/总结

不要把 DNS 当成只发生一次的动作。缓存、TTL、负载均衡和故障切换都会影响线上行为。
