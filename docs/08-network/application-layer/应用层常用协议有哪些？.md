# 应用层常用协议有哪些？

应用层常用协议有哪些？
HTTP、HTTPS、CDN、DNS、FTP都是应用层协议。

<!-- 面试复习补充 -->

## 面试复习补充

### 核心概念

应用层协议定义应用进程之间如何交换数据，常见有 HTTP/HTTPS、DNS、FTP、SMTP/POP3/IMAP、WebSocket、RPC 协议等。

### 面试官想考什么

面试官想看你能否说明每种协议解决什么问题，以及它们通常依赖 TCP 还是 UDP。

### 标准回答

HTTP/HTTPS 用于 Web 访问和 API；DNS 用于域名解析；SMTP/POP3/IMAP 用于邮件；WebSocket 用于双向实时通信；RPC 用于服务间调用。理解时要结合端口、传输层协议和典型场景。

### 深挖追问

- 如果线上出现超时/失败，如何验证是不是这个环节？
- 它和相邻层协议的职责边界是什么？
- 有哪些参数或默认行为会影响生产表现？

### 实战场景/示例

一次浏览器访问可能先用 DNS 解析域名，再建立 TCP/TLS 连接，最后通过 HTTP 获取资源。

### 易错点/总结

不要把应用层协议和传输层协议混淆，HTTP/RPC 不是 TCP 的替代品。
