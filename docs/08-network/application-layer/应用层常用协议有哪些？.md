# 应用层常用协议有哪些？

## 核心概念

应用层协议定义应用进程之间如何交换数据。常见协议包括 HTTP/HTTPS、DNS、FTP/SFTP、SMTP/POP3/IMAP、WebSocket、MQTT、RPC 协议（如 gRPC/HTTP2）、SNMP、NTP 等。

应用层协议通常建立在传输层之上：多数 Web 协议基于 TCP，DNS 主要使用 UDP 也可使用 TCP，HTTP/3 基于 QUIC/UDP。协议选择取决于实时性、可靠性、数据格式、连接模型和生态。

## 面试官想考什么

- 是否能说出常见应用层协议及用途；
- 是否理解 HTTP、DNS、WebSocket、gRPC 的差异；
- 是否知道应用层和传输层的关系；
- 是否能结合业务场景选择协议。

## 标准回答

> 常见应用层协议有 HTTP/HTTPS 用于 Web API 和网页访问，DNS 用于域名解析，SMTP/POP3/IMAP 用于邮件，FTP/SFTP 用于文件传输，WebSocket 用于全双工实时通信，gRPC 常用于服务间高性能 RPC，MQTT 常用于物联网消息。应用层协议一般运行在 TCP 或 UDP 之上，选择时要看可靠性、实时性、兼容性和开发生态。

## 深挖追问

### HTTP 和 WebSocket 怎么选？

普通请求响应、资源获取、REST API 用 HTTP 更简单；需要服务端主动推送、低延迟双向通信，如 IM、实时行情、在线协作，可用 WebSocket。WebSocket 建立连接时通过 HTTP Upgrade 升级，之后在同一 TCP 连接上双向传输。

### DNS 为什么既用 UDP 又用 TCP？

普通查询通常用 UDP，开销小；当响应较大、区域传输或 UDP 被截断时会使用 TCP。现代 DNS 还可能使用 DoH/DoT 增强隐私和安全。

## 实战场景 / 示例

电商系统可能同时使用多种应用层协议：

```text
浏览器/APP -> HTTPS -> API 网关
服务间调用 -> gRPC/HTTP2 或 HTTP JSON
实时库存提醒 -> WebSocket
域名解析 -> DNS
邮件通知 -> SMTP
```

## 易错点 / 总结

- HTTPS 不是新应用协议，本质是 HTTP + TLS；
- WebSocket 适合长连接双向通信，不适合所有接口；
- gRPC 性能好但浏览器和调试生态不如 HTTP JSON 普适；
- DNS 不是只用 UDP；
- 协议选型要考虑网关、负载均衡、监控和排障成本。
