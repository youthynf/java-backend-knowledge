TCP处于TIME_WAIT状态收到SYN报文会怎么样？
需要根据这个SYN报文是否合法进行分情况讨论：
如果SYN报文合法（序列号、时间戳都比当前连接保存的上一个报文的序列号和时间戳大）
重用该四元组，跳过2MSL而进入SYN_RECV状态，接着就能进行建立连接的过程。
如果SYN报文不合法（序列号、时间戳比当前连接保存的上一个报文的序列号和时间戳小）
回复第一个第四次挥手的ACK报文，对端收到后，发现并不是自己期望收到的确认好，就会回复一个RST报文，断开连接。
在TIME_WAIT状态，收到RST报文会断开连接吗？
如果net.ipv4.tcp_rfc1337=0，则提前结束TIME_WAIT状态，释放连接
如果net.ipv4.tcp_rfc1337=1，则会丢掉该RST报文
