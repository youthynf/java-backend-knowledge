TCP协议有什么缺陷？
主要缺陷有四个方面：
升级工作很困难：TCP内核实现，升级TCP只能升级内核，应用程序只能使用不能升级；
建立连接耗时：基于TCP实现的应用协议，都需要先建立三次握手才能进行数据传输，如HTTP，HTTPS，其中HTTPS还需要经过TLS四次握手后才能进行HTTP数据的传输，这在一定程度上增加了数据传输的延迟；
存在队头阻塞问题：TCP是字节流协议，TCP层必须保证收到的字节数据是完整且有序的，如果序号较低的TCP段在网络传输中丢失了，即时序号较高的TCP段已经被接受了，应用层也无法从内核中读取到这部分的数据。
网络迁移需要重新建立TCP连接：当移动设备的网络从4G切换到WIFI时，意味着IP地址变化了，那么就必须断开连接，然后重新建立连接。而建立连接的过程包含TCP三次握手和TLS四次挥手的实验，以及TCP慢启动的减速过程，给用户的感觉就是网络突然卡顿了一下，因此连接迁移的成本是很高的。
