# TCP三次握手和accept是什么关系？

TCP三次握手和accept是什么关系？
TCP 在三次握手过程中涉及两个连接队列：半连接队列和全连接队列。
半连接队列：存储已接收到客户端的 SYN 报文（第一步完成），但尚未完成三次握手的连接请求；
全连接队列：存储完成三次握手的连接，等待服务器应用程序接受的连接请求。

总结：三次握手与连接队列的交互
第一步：客户端发送 SYN，服务器将请求放入半连接队列。
第二步：服务器发送 SYN + ACK，连接仍在半连接队列中。
第三步：客户端发送 ACK，连接从半连接队列移动到全连接队列。
应用层调用：服务器调用 accept()，从全连接队列取出连接并处理。
