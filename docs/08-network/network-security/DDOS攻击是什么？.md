# DDOS攻击是什么？

DDOS攻击是什么？
DDoS攻击（分布式拒绝服务攻击，Distributed Denial of Service）是一种通过海量恶意流量淹没目标系统（如服务器、网络或应用程序），使其无法正常提供服务的网络攻击手段。

原理机制：
攻击目标：耗尽目标的带宽、计算资源或连接容量，导致合法用户无法访问。
分布式特性：攻击流量来自被控制的僵尸网络（Botnet，如感染病毒的IoT设备、服务器等），而非单一源头。
攻击链：控制端（C&C服务器） → 僵尸设备 → 目标受害者

常见的攻击分类：
流量型攻击（带宽耗尽）
UDP洪水：发送大量UDP包（如随机端口）占用目标带宽。
ICMP洪水：利用Ping请求（如死亡之Ping）淹没网络。
反射/放大攻击（如NTP/DNS反射）：通过小请求触发大响应（放大倍数可达数千倍），伪造受害者IP发送请求。
协议行攻击（资源耗尽）
SYN洪水：发送大量TCP连接请求但不完成握手，耗尽连接池。
HTTP慢速攻击（如Slowloris）：保持大量低速HTTP连接，占用服务器线程。
应用层攻击（精准打击）
HTTP洪水：模拟高并发合法请求（如刷API接口），消耗CPU/内存。
DNS查询洪水：向目标DNS服务器发送海量解析请求。

为了防范DDoS攻击，可以采取以下措施：
增强网络基础设施：提升网络带宽、增加服务器的处理能力和承载能力，通过增强基础设施的能力来抵御攻击；
使用防火墙和入侵检测系统：配置防火墙规则，限制不必要的网络流量，阻止来自可疑IP地址的流量。入侵检测系统可以帮助及时发现并响应DDoS攻击；
流量清洗和负载均衡：使用专业的DDoS防护服务提供商，通过流量清洗技术过滤掉恶意流量，将合法流量转发给目标服务器；负载均衡可以将流量均匀地分发到多台服务器上，减轻单一服务器的压力；
配置访问控制策略：限制特定IP地址或IP段的访问，设置访问频率限制，防止过多请求集中在单个IP上。

## 面试总结
### 核心概念

网络安全关注机密性、完整性、可用性和身份认证。常见攻击包括 XSS、CSRF、SQL 注入、DDoS、DNS 劫持和中间人攻击。

### 面试官想考什么

面试官想看你是否能说明攻击原理、危害和工程防护，而不是只背缩写。

### 标准回答

XSS 是注入脚本在用户浏览器执行，防护靠输出编码、CSP、HttpOnly；CSRF 是借用户登录态伪造请求，防护靠 SameSite、CSRF Token、校验 Origin/Referer；SQL 注入靠拼接 SQL 注入恶意语句，防护靠预编译和参数绑定；DDoS 需要限流、清洗和弹性防护；HTTPS 可降低中间人窃听篡改风险。

### 深挖追问

- 攻击入口、利用条件和影响范围分别是什么？
- 只靠 HTTPS、前端校验或 WAF 能不能完全解决？为什么？
- 如果线上已经发生攻击，如何止血、排查、修复和补审计？

### 实战场景/示例

评论区未转义用户输入导致脚本窃取 Cookie，这是典型存储型 XSS；转账接口只靠 Cookie 鉴权且无 CSRF Token，则可能被跨站表单触发。

### 易错点/总结

前端校验不能替代服务端防护；HTTPS 也不能防止应用层越权和注入漏洞。
## 核心概念
DDOS攻击是什么？ 可以放在“网络协议能力”这条主线里理解。复习时不要只背结论，要先说明它解决的核心问题，再解释关键机制、适用边界和代价。围绕这个知识点，重点关注：连接建立、报文结构、状态码、长连接、拥塞控制、TLS、代理和超时重试。如果面试官继续追问，通常会从“为什么这样设计、在什么场景会失效、线上如何排查”三个方向展开。

## 面试回答与追问
- **标准回答**：先给出 DDOS攻击是什么？ 的定位，再说明它依赖的核心原理，最后结合业务场景说明如何使用。回答时要把“能解决什么问题”和“会带来什么成本”一起讲清楚。
- **常见追问**：如果数据量、并发量或调用链路继续放大，DDOS攻击是什么？ 的瓶颈会出现在哪里？如何观测、如何优化、如何回滚？
- **易错点**：不要把概念和具体实现混在一起，也不要只说 API 名称。面试中更重要的是说清楚边界条件、失败场景和取舍依据。

## 实战场景与排查
典型落地场景包括：接口超时、连接耗尽、网关转发、上传下载、移动端弱网和跨域访问。实际处理线上问题时，可以按“现象确认 → 指标采集 → 假设验证 → 小步修复 → 复盘沉淀”的路径推进。先看日志、监控、链路追踪和核心指标，再判断是容量问题、配置问题、代码路径问题，还是外部依赖抖动。

## 总结
复习 DDOS攻击是什么？ 时，建议把它和相邻知识点放在一起比较：相同点是什么、区别在哪里、为什么当前场景选择它而不是替代方案。能讲清楚这些内容，才算真正掌握。
