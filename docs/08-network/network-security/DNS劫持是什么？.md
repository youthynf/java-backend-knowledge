# DNS 劫持是什么？

## 核心概念

DNS 劫持是攻击者或不可信网络篡改域名解析结果，让用户访问错误 IP，可能跳转到钓鱼网站、广告页面或恶意服务器。它发生在“域名到 IP”的解析链路上，常见位置包括本机 hosts、路由器、运营商递归 DNS、恶意 Wi-Fi、被污染的缓存等。

DNS 劫持和 HTTP 劫持不同：DNS 劫持改的是解析结果；HTTP 劫持是在请求或响应内容传输过程中插入、篡改内容。HTTPS 能防止内容被篡改，但如果 DNS 被劫持到攻击者 IP，浏览器仍会通过证书校验发现域名不匹配，除非用户忽略警告或证书体系被破坏。

## 面试官想考什么

- 是否理解 DNS 解析链路和劫持位置；
- 是否能区分 DNS 劫持、DNS 污染、HTTP 劫持；
- 是否知道 HTTPS、证书校验、HSTS 的防护作用；
- 是否能给出客户端和服务端治理方案。

## 标准回答

> DNS 劫持是域名解析结果被篡改，导致用户访问到错误 IP。它可能发生在本机 hosts、路由器、递归 DNS 或运营商链路中。防护上可以使用可信 DNS、DoH/DoT、HTTPS 证书校验、HSTS、证书固定（谨慎使用）、监控多地解析结果；服务端要配置正确证书和告警，客户端不要忽略证书错误。

## 深挖追问

### HTTPS 能完全解决 DNS 劫持吗？

不能完全解决解析被改，但能阻止攻击者伪装成合法站点，因为攻击者通常没有目标域名的有效证书。用户如果看到证书错误仍继续访问，或者设备被安装恶意根证书，就仍有风险。

### DNS 劫持怎么排查？

比较不同网络、不同 DNS 服务器的解析结果；检查本机 hosts、路由器 DNS、浏览器代理；使用 `dig`/`nslookup` 指定公共 DNS 查询；观察证书域名和服务端 IP 是否符合预期。

## 实战场景 / 命令示例

```bash
nslookup example.com 8.8.8.8
nslookup example.com 223.5.5.5
dig example.com @1.1.1.1
```

如果同一域名在异常网络解析到陌生 IP，而 HTTPS 证书也不匹配，基本可判断存在劫持或污染风险。

## 易错点 / 总结

- DNS 劫持改解析结果，HTTP 劫持改传输内容；
- HTTPS 依赖证书校验，不能忽略浏览器安全警告；
- 公共 DNS、DoH/DoT 能降低风险但不等于绝对安全；
- 企业内网 DNS 可能有合法劫持/分流策略，要先确认场景；
- 多地解析监控能及早发现异常。
