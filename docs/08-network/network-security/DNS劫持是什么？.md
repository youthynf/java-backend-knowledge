# DNS 劫持是什么？

## 核心概念

DNS 劫持是攻击者或不可信网络篡改域名解析结果，让用户访问错误 IP，可能跳转到钓鱼网站、广告页面或恶意服务器。它发生在“域名到 IP”的解析链路上，常见位置包括本机 hosts、路由器、运营商递归 DNS、恶意 Wi-Fi、被污染的缓存等。

DNS 劫持和 HTTP 劫持不同：DNS 劫持改的是解析结果；HTTP 劫持是在请求或响应内容传输过程中插入、篡改内容。HTTPS 能防止内容被篡改，但如果 DNS 被劫持到攻击者 IP，浏览器仍会通过证书校验发现域名不匹配，除非用户忽略警告或证书体系被破坏。

## 面试官想考什么

- 是否理解 DNS 解析链路和劫持位置；
- 是否能区分 DNS 劫持、DNS 污染、HTTP 劫持；
- 是否知道 HTTPS、证书校验、HSTS 的防护作用；
- 是否能给出客户端和服务端治理方案。

## 标准回答

> DNS 劫持是域名解析结果被篡改，导致用户访问到错误 IP。它可能发生在本机 hosts、路由器、递归 DNS 或运营商链路中。防护上可以使用可信 DNS、DoH/DoT、HTTPS 证书校验、HSTS、证书固定（谨慎使用）、监控多地解析结果；服务端要配置正确证书和告警，客户端不要忽略证书错误。

## 深挖追问

### HTTPS 能完全解决 DNS 劫持吗？

不能完全解决解析被改，但能阻止攻击者伪装成合法站点，因为攻击者通常没有目标域名的有效证书。用户如果看到证书错误仍继续访问，或者设备被安装恶意根证书，就仍有风险。

### DNS 劫持怎么排查？

比较不同网络、不同 DNS 服务器的解析结果；检查本机 hosts、路由器 DNS、浏览器代理；使用 `dig`/`nslookup` 指定公共 DNS 查询；观察证书域名和服务端 IP 是否符合预期。

## 实战场景 / 命令示例

```bash
nslookup example.com 8.8.8.8
nslookup example.com 223.5.5.5
dig example.com @1.1.1.1
```

如果同一域名在异常网络解析到陌生 IP，而 HTTPS 证书也不匹配，基本可判断存在劫持或污染风险。

## 易错点 / 总结

- DNS 劫持改解析结果，HTTP 劫持改传输内容；
- HTTPS 依赖证书校验，不能忽略浏览器安全警告；
- 公共 DNS、DoH/DoT 能降低风险但不等于绝对安全；
- 企业内网 DNS 可能有合法劫持/分流策略，要先确认场景；
- 多地解析监控能及早发现异常。

## 面试复习强化

### 核心概念

从面试角度看，**DNS 劫持是什么？** 可以放在“网络安全”这一类知识中理解。复习时不要只背定义，要能说清：它解决什么问题、依赖哪些前提、正常流程是什么、异常情况下系统会怎样退化或恢复。

### 面试官想考什么

- 是否理解概念背后的设计目标，而不是只记住名词；
- 是否能把机制和真实工程场景联系起来；
- 是否能分析边界条件、失败场景、性能与安全取舍；
- 是否能给出可落地的排查、实现或优化步骤。

### 标准回答

> 先建立威胁模型：攻击者控制什么输入、越过什么信任边界、造成什么影响。 追问会考漏洞成因、攻击链路、检测方式、防御手段和防御局限。 对于“DNS 劫持是什么？”，回答时建议先给一句话定义，再按“工作流程/关键机制 → 典型场景 → 风险与优化”展开，最后补充一两个线上实践点。

### 深挖追问

- 如果该机制失效，会出现什么现象？如何定位是配置、代码、资源还是外部依赖导致？
- 它和相邻概念有什么区别？例如语义、适用场景、性能成本、可靠性保证分别是什么？
- 在高并发、网络抖动、服务重启、数据不一致或权限受限时，需要补充哪些保护措施？
- 有哪些指标可以证明方案有效？例如延迟、吞吐、错误率、资源使用率、重试次数或业务成功率。

### 示例 / 实战场景

- 设计方案时：先明确业务目标和约束，再选择对应机制，不要为了使用某个技术而引入复杂度。
- 排查问题时：先确认现象和影响面，再查看日志、监控、配置、版本变更和上下游依赖，最后小步验证修复。
- 复盘沉淀时：补充自动化测试、容量评估、告警阈值、降级预案和文档，避免同类问题再次发生。

### 本题高频补充

- DNS 回答要包含递归/迭代查询、缓存 TTL、UDP 53 为主，以及大响应、区域传送或截断重试时可能使用 TCP。

### 易错点 / 总结

- 只背结论、不讲原因，是面试扣分点；要主动解释“为什么这样设计”。
- 只讲正常路径、不讲异常路径，会显得缺少生产经验；至少补充超时、重试、降级、回滚或兜底。
- 不要把理论保证无限放大，工程实现通常还受网络、资源、配置、版本和业务语义约束。
- 总结一句：默认输入不可信、权限最小化、失败默认拒绝，并配合日志审计和告警。
