# MQ 基础

## 面试复习重点

- 消息模型、发布订阅、消费组、确认机制和重试机制。
- 顺序、重复、丢失、积压和幂等这些高频问题的治理方案。
- 容量评估、监控告警、故障恢复和降级预案。

## 建议掌握程度

- **能讲清概念**：用自己的话说明定义、背景和解决的问题。
- **能画出链路**：把关键组件、核心流程和状态变化串起来。
- **能回答追问**：准备优缺点、适用场景、常见坑和替代方案。
- **能落地排查**：结合日志、指标、工具和案例说明定位思路。

## 面试表达模板

1. 先给结论：说明它是什么、解决什么问题。
2. 再讲原理：说明核心组件、关键流程和数据结构。
3. 补充场景：结合项目或线上问题说明怎么用。
4. 说明边界：讲清楚缺点、风险、替代方案和排查手段。


MQ 基础模块关注所有消息系统共性：消息模型、投递语义、重复消费、顺序性、延迟、死信、监控和补偿。

## 复习摘要

- **核心价值**：异步化、系统解耦、削峰填谷、广播/订阅。
- **核心风险**：消息丢失、重复消费、乱序、积压、延迟、数据一致性。
- **通用方案**：本地消息表/Outbox、唯一业务键、消费幂等、重试退避、死信队列、监控告警、人工补偿。

## 高频问题

1. MQ 会不会丢消息？分别从生产者、Broker、消费者讲。
2. 如何防止重复消费？核心是业务幂等，不是幻想 MQ 绝不重复。
3. 如何保证顺序？同一业务键进入同一分区/队列，消费端单线程或有序处理。
4. 消息积压怎么办？定位瓶颈、扩容消费、限流下游、临时旁路和复盘治理。
