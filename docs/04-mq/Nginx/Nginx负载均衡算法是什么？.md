# Nginx负载均衡算法是什么？

Nginx负载均衡算法是什么？
正向代理与反向代理定义：
正向代理：正向代理是位于用户设备和互联网之间的服务器。它代理的是客户端，是站在用户一方的。其真实客户端对于服务器不可见。主要作用是保护客户端，隐藏客户端真实IP，避免攻击；解决访问限制；提高访问速度；阻止访问指定内容等作用；
反向代理：反向代理是一种服务器，它接受客户端的请求，将请求转发给网络服务器，然后将结果返回给客户端，就像代理服务器处理了请求一样。主要保护服务端隐私，隐藏服务器真实IP；负载均衡；缓存静态内容以及部分短时间的大量动态请求；作为应用防火墙提供防护；加密/解密 SSL 通信；

负载均衡算法：
负载均衡将网络流量或一组任务以某种算法合理分配给各个处理节点，使得节点得到平等的使用，并及时可靠地返回结果给用户。
循环：客户端请求按顺序发送到不同的服务实例。通常要求服务是无状态的。
粘性循环：一个用户的请求都发往同一个服务节点，适合客户端操作有连续性的情况。
加权循环：指定每个服务的权重。权重高的服务会比其他服务处理更多请求。
散列：对传入请求的 IP 或 URL 应用哈希函数。根据哈希函数的结果将请求路由到相关服务。
最少连接数：新请求会发送到并发连接最少的服务节点。
最短响应时间：新请求会发送到响应时间最快的服务节点。
助记：正向代理代理的是客户端，反向代理的是服务端。负载均衡分为静态负载均衡和动态负载均衡。
