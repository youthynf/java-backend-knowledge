## 复习摘要

- **核心概念**：先理解组件模型，再理解数据/请求在组件间如何流转。
- **面试官想考什么**：不仅问“是什么”，更看你是否知道为什么这么设计、有哪些故障边界。
- **标准回答**：用“场景 → 原理 → 配置/参数 → 实战坑点”的结构回答。
- **易错点**：不要孤立背概念，要结合生产场景说明如何排查、监控、扩容和降级。
