# Zookeeper

ZooKeeper 是 Apache 开源的分布式协调服务，基于 ZAB（ZooKeeper Atomic Broadcast）协议实现数据一致性。核心定位是解决分布式系统中"一致性、可靠性、协调性"问题，而非传统存储系统。广泛用于 Hadoop、Kafka（旧版）、Dubbo 等分布式框架，承担配置管理、服务注册、分布式锁、集群选主等协调任务。

## 核心要点

- **数据模型**：树形 ZNode 结构，类似文件系统，单节点数据上限 1MB，仅存元数据。
- **节点类型**：持久节点、持久顺序、临时节点、临时顺序；3.5+ 新增容器节点和 TTL 节点。
- **ZAB 协议**：崩溃恢复（Leader 选举，过半机制）+ 消息广播（两阶段提交，过半 ACK）。
- **Watcher**：一次性触发的事件通知机制，回调中需重新注册。
- **集群角色**：Leader（处理写）、Follower（处理读 + 投票）、Observer（仅处理读，不投票）。
- **节点数**：奇数（3、5、7），过半机制下 3 台容忍 1 台故障，4 台也只容忍 1 台。
- **替代方案**：配置中心用 Nacos，分布式锁用 Redis，注册发现用 Etcd。
