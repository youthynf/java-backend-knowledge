# ClickHouse

ClickHouse 是 Yandex 开源的列式 OLAP 数据库，2016 年开源。底层围绕"高效处理海量数据分析"设计，核心是"存储层极致优化 + 计算层高效执行 + 分布式协同"三大支柱。适合写多读多的分析型场景（报表、行为分析、日志分析），不适合 OLTP 事务和单行频繁更新。

## 目录

- [ClickHouse 是什么](ClickHouse是什么？.md) — 列式存储、MergeTree 引擎、分区/排序键、物化视图、分布式架构

## 核心要点

- **列式存储**：同一列连续存储，查询时列裁剪减少 IO，同列同类型压缩率高，适配向量化计算。
- **MergeTree 引擎**：核心表引擎，按 `PARTITION BY` 分区，按 `ORDER BY` 排序，后台合并小 Part。
- **稀疏主键索引**：默认每 8192 行一个索引值（`index_granularity`），定位数据块而非行。
- **向量化执行**：以 Block（默认 65536 行）为单位，利用 CPU SIMD 指令批量处理。
- **分布式**：分片（Shard）+ 副本（Replica），通过 ZooKeeper 同步副本，分布式表并行查询。
- **物化视图**：预聚合热点指标，提升查询性能。
- **不适用**：单行频繁更新、强事务、点查（OLTP 场景）。
