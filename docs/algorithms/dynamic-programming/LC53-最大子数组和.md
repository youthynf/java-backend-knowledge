# LC53 · 最大子数组和 (Maximum Subarray)

## 题目

给定一个整数数组 `nums`，找到一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

**示例**：
```
输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6
```

**约束**：
- `1 <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

---

## 解法一：Kadane 算法（动态规划）✅ 最优

### 思路

遍历数组，维护两个变量：
- `currentMax`：以当前元素结尾的最大子数组和
- `globalMax`：全局最大子数组和

对于每个元素 `nums[i]`：
- 要么加入前面的子数组：`currentMax + nums[i]`
- 要么从当前元素重新开始：`nums[i]`
- 取两者较大值

### 代码

```java
class Solution {
    public int maxSubArray(int[] nums) {
        int globalMax = nums[0];
        int currentMax = nums[0];
        
        for (int i = 1; i < nums.length; i++) {
            // 核心转移方程：
            // dp[i] = max(dp[i-1] + nums[i], nums[i])
            currentMax = Math.max(currentMax + nums[i], nums[i]);
            globalMax = Math.max(globalMax, currentMax);
        }
        
        return globalMax;
    }
}
```

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n) |
| 空间 | O(1) |

### DP 转移方程推导

```
定义：dp[i] = 以 nums[i] 结尾的最大子数组和

dp[i] = max(dp[i-1] + nums[i], nums[i])

解释：
- dp[i-1] + nums[i]：将 nums[i] 接在以 nums[i-1] 结尾的最大子数组后面
- nums[i]：从 nums[i] 开始新起一个子数组

如果 dp[i-1] < 0，说明前面的累积是负收益，不如从当前重新开始
```

## 解法二：前缀和

### 思路

最大子数组和 = 最大前缀和差 = max(前缀和[j] - min(前缀和[i]))，其中 j > i

### 代码

```java
class Solution {
    public int maxSubArray(int[] nums) {
        int maxSum = nums[0];
        int prefixSum = 0;
        int minPrefix = 0; // 前缀和最小值
        
        for (int num : nums) {
            prefixSum += num;
            // 当前前缀和 - 之前最小前缀和 = 最大子数组和
            maxSum = Math.max(maxSum, prefixSum - minPrefix);
            minPrefix = Math.min(minPrefix, prefixSum);
        }
        
        return maxSum;
    }
}
```

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n) |
| 空间 | O(1) |

## 解法三：分治法

### 思路

将数组从中间分成两半，最大子数组要么：
1. 完全在左半部分
2. 完全在右半部分
3. 跨越中点（左半的最大后缀 + 右半的最大前缀）

### 代码

```java
class Solution {
    public int maxSubArray(int[] nums) {
        return divide(nums, 0, nums.length - 1);
    }
    
    private int divide(int[] nums, int left, int right) {
        if (left == right) return nums[left];
        
        int mid = left + (right - left) / 2;
        
        // 左半最大
        int leftMax = divide(nums, left, mid);
        // 右半最大
        int rightMax = divide(nums, mid + 1, right);
        // 跨越中点
        int crossMax = crossSum(nums, left, right, mid);
        
        return Math.max(Math.max(leftMax, rightMax), crossMax);
    }
    
    private int crossSum(int[] nums, int left, int right, int mid) {
        // 左半最大后缀
        int leftSum = Integer.MIN_VALUE;
        int sum = 0;
        for (int i = mid; i >= left; i--) {
            sum += nums[i];
            leftSum = Math.max(leftSum, sum);
        }
        
        // 右半最大前缀
        int rightSum = Integer.MIN_VALUE;
        sum = 0;
        for (int i = mid + 1; i <= right; i++) {
            sum += nums[i];
            rightSum = Math.max(rightSum, sum);
        }
        
        return leftSum + rightSum;
    }
}
```

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n log n) |
| 空间 | O(log n) 递归栈 |

> 分治法虽不是最优解，但它是线段树解法的基础，可以支持区间查询和动态更新。

## 变体题

### 变体 1：返回子数组的起止下标

```java
public int[] maxSubArrayWithIndex(int[] nums) {
    int globalMax = nums[0], currentMax = nums[0];
    int start = 0, end = 0, tempStart = 0;
    
    for (int i = 1; i < nums.length; i++) {
        if (currentMax + nums[i] < nums[i]) {
            currentMax = nums[i];
            tempStart = i;  // 重新开始
        } else {
            currentMax += nums[i];
        }
        
        if (currentMax > globalMax) {
            globalMax = currentMax;
            start = tempStart;
            end = i;
        }
    }
    
    return new int[]{globalMax, start, end};
}
```

### 变体 2：环形子数组的最大和（LC918）

```
两种情况：
1. 最大子数组不跨过边界 → 普通 Kadane
2. 最大子数组跨过边界 → 总和 - 最小子数组（用 Kadane 求最小子数组）
特殊情况：全负数时，最大子数组就是最大元素（不是 总和-最小=0）
```

### 变体 3：乘积最大子数组（LC152）

```
需要同时维护 max 和 min（因为负负得正）
dpMax[i] = max(dpMax[i-1] * nums[i], dpMin[i-1] * nums[i], nums[i])
dpMin[i] = min(dpMax[i-1] * nums[i], dpMin[i-1] * nums[i], nums[i])
```

## 面试要点

1. **为什么 Kadane 有效？** 本质是贪心 + DP，每步选择局部最优（接 or 不接），局部最优推导全局最优
2. **为什么不用暴力？** O(n²) 或 O(n³)，数据量 10^5 必然超时
3. **Kadane 的前提条件？** 无特殊前提，对全负数同样有效
4. **和前缀和的关系？** Kadane 可以看作"边算前缀和边维护最小前缀和"的优化版
5. **分治法的意义？** 虽然时间复杂度更高，但为线段树解法铺路，支持 O(log n) 区间查询
