# LC53 最大子数组和

## 核心概念

最大子数组和（Maximum Subarray）要求在一个整数数组中找到一个连续子数组，使其元素和最大，并返回这个最大和。关键限制是“连续”，所以不能像子序列那样随意跳过元素。

这题是动态规划和贪心思想的经典题，也可以用分治解决。最常用解法是 Kadane 算法：维护“以当前位置结尾的最大子数组和”。

## 面试官想考什么

- 是否能定义正确的 DP 状态；
- 是否理解为什么状态只依赖前一个位置，可以空间优化到 O(1)；
- 是否能处理全负数数组；
- 是否能说清连续子数组与子序列的区别；
- 是否了解扩展问题：返回区间下标、环形最大子数组和、最大子矩阵。

## 标准回答

> 定义 `dp[i]` 为“以 `nums[i]` 结尾的连续子数组的最大和”。对于每个位置，要么把当前元素接到前一个最大子数组后面，要么从当前元素重新开始，因此转移方程是 `dp[i] = max(nums[i], dp[i-1] + nums[i])`。答案是所有 `dp[i]` 的最大值。由于 `dp[i]` 只依赖 `dp[i-1]`，可以用一个变量 `current` 滚动保存，时间复杂度 O(n)，空间复杂度 O(1)。

## 深挖追问

### 为什么答案不是最后一个 `dp[n-1]`？

因为 `dp[i]` 表示必须以第 i 个元素结尾的最大和，而全局最大子数组可能在任意位置结束，所以需要维护 `maxSum = max(maxSum, dp[i])`。

### 全负数数组怎么处理？

不能把初始最大值设为 0，否则 `[-3, -2, -5]` 会错误返回 0。应初始化为 `nums[0]`，这样答案会是最大的那个负数。

### 这题是动态规划还是贪心？

两种视角都可以。DP 视角是状态转移；贪心视角是：如果前缀和 `current` 已经小于 0，那么它只会拖累后续子数组，应从当前元素重新开始。面试中用 DP 定义更稳。

### 如何返回最大子数组的起止下标？

在决定“从当前元素重新开始”时记录候选起点，在更新全局最大值时记录最终起止位置。

## 实战场景/代码示例

### Java：只返回最大和

```java
public int maxSubArray(int[] nums) {
    int current = nums[0];
    int maxSum = nums[0];

    for (int i = 1; i < nums.length; i++) {
        current = Math.max(nums[i], current + nums[i]);
        maxSum = Math.max(maxSum, current);
    }
    return maxSum;
}
```

### Java：同时返回区间

```java
public int[] maxSubArrayWithRange(int[] nums) {
    int current = nums[0];
    int maxSum = nums[0];
    int start = 0;
    int end = 0;
    int candidateStart = 0;

    for (int i = 1; i < nums.length; i++) {
        if (current + nums[i] < nums[i]) {
            current = nums[i];
            candidateStart = i;
        } else {
            current += nums[i];
        }

        if (current > maxSum) {
            maxSum = current;
            start = candidateStart;
            end = i;
        }
    }
    return new int[]{maxSum, start, end};
}
```

### 示例推演

数组：`[-2,1,-3,4,-1,2,1,-5,4]`

| i | nums[i] | current = max(nums[i], current+nums[i]) | maxSum |
| --- | --- | --- | --- |
| 0 | -2 | -2 | -2 |
| 1 | 1 | 1 | 1 |
| 2 | -3 | -2 | 1 |
| 3 | 4 | 4 | 4 |
| 4 | -1 | 3 | 4 |
| 5 | 2 | 5 | 5 |
| 6 | 1 | 6 | 6 |
| 7 | -5 | 1 | 6 |
| 8 | 4 | 5 | 6 |

最大子数组是 `[4, -1, 2, 1]`，最大和为 6。

## 易错点/总结

- 子数组必须连续，不能跳着选；
- 全负数时答案是最大负数，不是 0；
- `dp[i]` 是以 i 结尾的局部最优，答案要取全局最大；
- 空间可优化，因为只依赖前一个状态；
- 如果数组元素和可能超过 int 范围，业务代码应使用 long；
- 返回区间时要区分候选起点和最终起点。
