# Redis如何运用IO多路复用的？

Redis如何运用IO多路复用的？
Redis通过I/O多路复用技术实现了高效的网络I/O处理，在Linux中主要依赖于epoll来实现。

单线程模式实现机制：（主要两大步骤）
Redis初始化过程：
首先，调用epoll_create()创建一个epoll对象和调用socket()创建一个服务端socket；
然后，调用bind()绑定端口和调用listen()监听该socket；
然后，将调用epoll_ctl()将listen socket加入到epoll进行管理，同时注册【连接事件】处理函数；

初始化完成之后，主进程进入到一个事件循环函数，主要执行以下操作：
首先，调用处理发送队列函数，查看发送队列中是否存在任务，如果有则通过write()函数将客户端发送缓存区中的数据发送出去，如果这一轮数据没有发送完，就会注册【写事件】处理函数，等待epoll_wait()函数等待事件的到来；
接着，调用epoll_wait()函数等待事件的到来：
1）如果是连接事件到来，则调用连接事件处理函数，该函数会调用accept()函数获取已连接的socket，然后调用epoll_ctl()将已连接的socket加入到epoll中进行管理，最后注册【读事件】处理函数；
2）如果是读事件到来，则调用读事件处理函数，该函数会调用read()函数读取客户端发送的数据，然后解析命令，处理命令，最后将客户端对象添加到发送队列，然后将执行结果写到发送缓冲区等待发送；
3）如果是写事件到来，则调用些时间处理函数，该函数会调用write()函数将客户端发送缓冲区里的数据发送出去，如果这一轮数据没有发送完（内核socket发送缓冲区过小），就会继续注册写事件处理函数，等待epoll_wait()发现可写后再处理。
在epoll的系列函数里，epoll_create用于创建一个epoll对象，epoll_ctl用于给epoll对象添加或者删除一个socket套接字，epoll_wait就是查看epoll当前管理的这些socket上有没有可读可写事件发送。
网络事件发生过程：
当网卡上收到数据包后，Linux内核进行一些列处理后把数据放到socket的接受队列。然后会检查是否有epoll在管它，如果有，则在epoll队列的就绪队列中插入一个元素。epoll_wait的操作就简单了，就是到epoll的就绪队列上查询有没有事件发生就行了。Javaer习惯把基于epoll的网络开发模型叫做NIO。

Reactor模型：
在基于epoll的编程中，其与传统的函数调用思路不同的是，我们并不能主动调用某个API来处理，因为无法知道我们想要处理的事件什么时候发生，所以只好提前把想要处理的事件的处理函数注册到一个事件分发器上，当事件发生的时候，由这个事件分发器调用回调函数进行处理。这类基于实现注册事件分发器的开发模式，就叫做Reactor模型。

优点
高效性：通过epoll，Redis能够同时监控大量的套接字连接，而不需要为每个连接创建一个线程，减少了线程创建和管理的开销。
可扩展性：能够处理高并发的网络请求，适用于大规模的分布式系统。

Redis 6.0的改进
多线程I/O：从Redis 6.0开始，Redis引入了多线程模型来处理网络I/O操作。主线程负责接收连接请求，而多个I/O线程负责读取和写入数据，进一步提高了I/O性能。
命令执行仍为单线程：尽管引入了多线程I/O模型，命令的执行仍然由主线程负责，以保持实现的简单性和一致性。
