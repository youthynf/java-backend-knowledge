# Redis 持久化

本目录覆盖 Redis 数据持久化的核心机制：RDB 快照、AOF 日志、AOF 重写、混合持久化，以及大 Key 对持久化的影响。

## 目录

- [RDB 快照是如何实现的](RDB快照是如何实现的？.md) — fork 子进程 + 写时复制，二进制全量快照
- [AOF 日志是如何实现的](AOF日志是如何实现的？.md) — 命令后写、appendfsync 三种策略
- [AOF 重写机制是什么](AOF重写机制是什么？.md) — 子进程扫描内存压缩历史命令
- [为什么会有 Redis 混合持久化](为什么会有Redis混合持久化？.md) — Redis 4.0+ 的 RDB 头 + AOF 尾组合方案
- [Redis 如何实现数据不丢失](Redis如何实现数据不丢失？.md) — 三种持久化对比与生产配置
- [Redis 大 Key 对持久化有什么影响](Redis大Key对持久化有什么影响？.md) — fork 阻塞、COW 内存放大、fsync 延迟

## 阅读建议

1. 先读 [Redis 如何实现数据不丢失](Redis如何实现数据不丢失？.md) 建立整体认知
2. 再分别深入 RDB、AOF、AOF 重写的实现细节
3. 最后看混合持久化和大 Key 影响，理解生产实践
