# 底层数据结构-整数集合

底层数据结构-整数集合
整数集合是Set对象的底层数据结构实现之一。当一个Set对象只包含整数数值元素，并且元素数量不大时，就会使用整数集这个数据结构作为底层实现。

整数集合结构设计
整数集合本质上是一块连续的空间，intset结构定义如下：
encoding：集合元素的编码方式；
length：集合包含的元素数量；
contents[]：保存元素的数组，存放的元素类型取决于encoding成员变量；

encoding取值：
如果encoding属性值为INTSET_ENC_INT16，那么contents就是一个int16_t类型的数组，数组中每个元素类型都是int16_t；
如果encoding属性值为INTSET_ENC_INT32，那么contents就是一个int32_t类型的数组，数组中每个元素类型都是int32_t；
如果encoding属性值为INTSET_ENC_INT64，那么contents就是一个int64_t类型的数组，数组中每个元素类型都是int64_t；

整数集合的升级操作：
整数集合会有一个升级规则，当新加入的元素类型比现有集合所有类型都要长时，整数集合需要先进行升级：
计算出需要升级后所需要的空间大小减去当前空间大小，就是需要扩容的空间大小，完成扩容升级后，
再将新元素键入到整数集合里。
整数集合升级的过程不是新分配一个新类型的数组，而是在原本数组上扩展空间，然后再将每个元素按照间隔类型大小分隔，如果encoding是INTSET_ENC_INT16，则每个元素的间隔空间是16位。整数集合升级的好处是节省内存资源。同时整数集合只支持升级，不支持降级操作。
