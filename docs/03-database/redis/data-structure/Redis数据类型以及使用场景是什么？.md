# Redis数据类型以及使用场景是什么？

Redis数据类型以及使用场景是什么？
Redis提供了丰富的数据类型，常见的有五种数据类型：String（字符串）、Hash（哈希）、List（列表）、Set（集合）、ZSet（有序集合）。随着Redis版本的更新，后面又新增支持了四种数据类型：BitMap（2.2版新增）、HyperLogLog（2.8版新增）、GEO（3.2版新增）、Stream（5.0版新增）。

Redis五种数据类型使用场景：
String：缓存对象、常规计数、分布式锁、共享session信息等；
List：消息队列（但有两个问题：生产者需要自行实现全局唯一ID；不能以消费组形式消费数据）等；
Hash：缓存对象、购物车等；
Set：聚合计算（交/并/差集）场景，比如点赞、共同关注、抽奖活动等；
ZSet：排序场景，比如排行榜、电话和姓名排序等；

Redis后续版本新支持的四种数据类型使用场景：
BitMap：二值状态统计的场景，比如签到、判断用户登录、连续登录用户总数等；
HyperLogLog：海量数据技术统计的场景，比如百万级网页UV计数等；
GEO：存储地理位置信息的场景，比如滴滴叫车；
Stream：消息队列，相比与基于List类型实现的消息队列，有两个特有特性：自动生成全局唯一消息ID，支持消费组形式消费数据。
