# Redis与Memcache区别是什么？

## 核心概念

Redis与Memcache区别是什么？
Redis与Memcached共同点：
都是基于内存的数据库，一般都是用来当做缓存使用；
都有过期策略；
两者的性能都非常高；

Redis与Memcached区别：
Redis 支持的数据类型更丰富（String、Hash、List、Set、ZSet），而 Memcached 只支持最简单的 key-value 数据类型；
Redis 支持数据的持久化，可以将内存中的数据保持在磁盘中，重启的时候可以再次加载进行使用，而 Memcached 没有持久化功能，数据全部存在内存之中，Memcached 重启或者挂掉后，数据就没了；
Redis 原生支持集群模式，Memcached 没有原生的集群模式，需要依靠客户端来实现往集群中分片写入数据；
Redis 支持发布订阅模型、Lua 脚本、事务等功能，而 Memcached 不支持；

## 面试官想考什么

- 缓存收益、失效机制和一致性边界。
- 穿透、击穿、雪崩、热 Key、大 Key 的治理。
- 过期删除、内存淘汰、缓存更新策略的区别。

## 标准回答

Redis 缓存的核心是用内存访问降低后端压力，但必须接受缓存与数据库之间存在一致性窗口。工程上通常结合 TTL、随机过期、空值缓存、布隆过滤器、互斥重建、逻辑过期、限流降级、消息重试等方案治理风险。

## 深挖追问

1. 一致性怎么保证？通常做最终一致，更新 DB 后删缓存并补偿重试。
2. 击穿/穿透/雪崩区别？热点失效/不存在数据/大量同时失效或 Redis 故障。
3. 大 Key 危害？阻塞、占内存、放大网络和迁移成本。

## 实战场景 / SQL 示例

```text
GET cache:user:1
-- miss 后查 DB，SETEX cache:user:1 300 <json>；热点重建用互斥锁或逻辑过期。
```

## 易错点 / 总结

- 不要给所有 Key 设置相同过期时间。
- 不要缓存无限增长的大对象或大集合。
- 缓存一致性通常是最终一致，需要补偿和监控。
