## 标准回答

MySQL 锁问题要结合隔离级别、索引访问路径和 SQL 条件分析。InnoDB 行锁锁的是索引记录；范围当前读在可重复读下可能使用间隙锁/临键锁。线上遇到锁等待应先定位阻塞事务，再缩短事务、补齐索引、固定更新顺序并加入重试。

## 深挖追问

1. 为什么不走索引会扩大锁冲突？行锁依附索引，扫描越多冲突越大。
2. 死锁一定是数据库 bug 吗？通常是业务加锁顺序、事务长度或索引问题。
3. 如何降低锁冲突？短事务、精准索引、固定顺序、拆批和重试。

## 易错点 / 总结

- 不要只看 SQL 文本，要看实际执行计划和索引。
- 死锁不能完全避免，业务要能幂等重试。
- 大批量更新建议拆批，降低锁持有时间。
