# InnoDB和MyISAM数据文件组织方式

InnoDB和MyISAM数据文件组织方式
MySQL 5.7及以前版本
InnoDB存储引擎
表空间管理模式：
· 系统表空间模式：所有表的数据和索引存储在共享的ibdata1文件中
· 独立表空间模式（5.6后默认）：每个表独立存储于.ibd文件

独立表空间下数据库目录文件：
.opt文件：数据库默认字符集和校验规则
.frm文件：表结构定义信息（每个表一个）
.ibd文件：表数据和索引（聚簇索引+所有二级索引）

MyISAM存储引擎
每个表对应三个独立文件：
.frm文件：表结构定义信息
.MYD文件（MyISAM Data）：实际数据记录
.MYI文件（MyISAM Index）：所有索引信息（含指针指向.MYD对应记录）

MySQL 8.0重要变化
关键变更
移除.frm文件：表结构定义改为存储在系统数据字典中（事务性）
移除.opt文件：数据库属性整合到数据字典
保留的核心文件：
· InnoDB：仍使用.ibd文件存储表数据和索引
· MyISAM：仍使用.MYD和.MYI文件（但不需.frm文件）

架构优势
· 提升元数据管理的一致性和可靠性
· 支持原子DDL操作
· 统一了不同存储引擎的元数据存储方式
