# MySQL为什么选择InnoDB作为默认存储引擎？

MySQL为什么选择InnoDB作为默认存储引擎？
InnoDB存储引擎在事务支持、并发性能、崩溃恢复等方面具有优势，因此5.5之后被选择作为默认的存储引擎。
事务支持：InnoDB存储引擎提供了对事务支持，可以进行ACID属性操作，MyISAM不支持事务；
并发性能：InnoDB存储引擎支持行级锁机制，可以提供更好的并发性能，而MyISAM存储引擎只支持表级锁，锁粒度较大；
崩溃恢复：InnoDB存储引擎通过redolog日志实现了崩溃恢复，可以在数据库发生异常情况时，通过日志文件进行恢复，保证数据的持久性和一致性，而MyISAM则不支持崩溃恢复。
