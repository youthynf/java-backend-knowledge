# CHAR和VARCHAR区别是什么？

## 核心概念

CHAR和VARCHAR区别是什么？
CHAR是固定长度的字符串类型，定义时需要指定固定长度，实际存储长度未达到定义时会在末尾补足空格。适合存储长度固定的数据，最大是255字符；
VARCHAR是可变长的字符串类型，定义时需要指定最大长度，实际存储是根据实际长度占用存储空间。适合存储长度可变的数据，最大是65535字符（受行大小限制）。

超出定义的长度时：
在严格模式下，插入超出长度的数据会直接报错，禁止插入操作；
在非严格模式下，varchar对超出长度的数据进行截断处理，只保留符合定义最大长度的数据部分进行插入；char同样会截断，但如果截断后不足5个字符（如多字节字符被截断为不完整字符），则会填充空格。

在 MySQL 中，CHAR和VARCHAR类型定义的长度表示的含义会因字符集的不同而有所区别：
非二进制字符集（如 utf8、utf8mb4 等）
对于使用非二进制字符集的 CHAR 和 VARCHAR 类型，其定义的长度指的是字符数。
二进制字符集（如 binary、varbinary 等）
当使用二进制字符集时，CHAR 和 VARCHAR 的类似类型（即 binary 和 varbinary）定义的长度指的是字节数。
