# 分表和分库有什么区别？

分表和分库有什么区别？
分库：一种水平扩展的数据库技术，将数据根据一定规则分为多个独立的数据库，每个数据库只负责存储部分数据，实现数据的拆分和分布式存储。主要解决并发连接过多，单机MySQL扛不住问题；
分表：将单个数据库中的表拆分成多个表，每个表只负责存储一部分数据。这种数据的垂直拆分能够提高查询效率，减轻单个表的压力。主要解决单表数据量太大，导致查询性能下降的问题；

分库与分表可以从垂直和水平两种维度进行拆分：
垂直分库：一般按照业务和功能维度进行拆分，将不同业务数据放到不同的数据库中，核心理念是专库专用。
垂直分表：针对业务上字段较多的大表进行，一般把业务宽表中比较独立的字段，或者不常用的字段拆分到独立的数据表中，是一种大表拆小表的模式。数据库是以行为单位将数据加载到内存中，这样拆分以后核心表大多是访问频率较高的字段，而且字段长度也都较短，因此可以加载更多数据到内存中，减少磁盘I/O，增加索引命中率；
水平分库：指的是将同一个表按照一定规则拆分到不同的数据库中，每个库可以位于不同的服务器上，以此实现水平扩展，是一种常见的提升数据库性能的方式。这种方案往往能解决单裤存储量及性能瓶颈问题，但是由于同一个表被分配到不同的数据库中，数据的访问需要额外的路由工作，因此系统的复杂度也被提升；
水平分表：在同一个数据库内，将一张大数据量的表按一定规则，切分成多个结构相同的表，而每个表只存原表的一部分数据。水平分表尽管分了表，但子表还是在同一个数据库实例中，只是解决了单一表数据量过大问题，并没有将拆分后的表分散在不同的机器上，还在竞争同一个物理机的CPU、内存、网络IO等，进一步提升需要分散到不同的数据库，达到分布式效果。
