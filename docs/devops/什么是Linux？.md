# 什么是 Linux？

## 核心概念

Linux 是一种开源类 Unix 操作系统内核，通常和 GNU 工具、发行版软件包一起组成完整操作系统。后端开发接触最多的是服务器上的 Linux 发行版，例如 CentOS、Ubuntu、Debian 等。

## 面试官想考什么

- 是否具备基本服务器操作能力；
- 是否理解进程、文件、权限、网络、服务等基础概念；
- 是否能使用常见命令定位线上问题；
- 是否知道 Linux 和应用部署、容器化之间的关系。

## 标准回答

> Linux 是服务器领域常用的操作系统，具备稳定、开源、可定制、生态成熟等特点。Java 后端通常需要在 Linux 上部署应用、查看日志、管理进程、排查端口和资源问题。常见操作包括 `ps` 看进程、`top` 看资源、`tail` 看日志、`grep` 搜索、`netstat/ss` 看端口、`df/du` 看磁盘。

## 深挖追问

### 为什么后端要懂 Linux？

因为线上服务大多运行在 Linux 上。出现 CPU 高、内存泄漏、磁盘满、端口不通、日志异常时，后端需要能第一时间定位问题，而不是完全依赖运维。

### Linux 权限怎么看？

常见权限包括读 `r`、写 `w`、执行 `x`，分别作用于 owner、group、others。部署脚本或日志目录权限错误，经常会导致应用启动失败或无法写日志。

## 实战场景

应用启动失败时，可以先看启动日志，再用 `ps -ef | grep java` 确认进程，用 `ss -lntp` 确认端口是否被占用，用 `df -h` 确认磁盘是否满。

## 易错点/总结

- 不要在线上随意执行危险命令；
- 修改配置前保留备份，发布前准备回滚；
- Linux 排障要按时间线和证据推进，不要靠猜。
