# Docker 与虚拟机有什么区别？

## 核心概念

Docker 是容器化技术，虚拟机是硬件虚拟化技术。二者都能提供隔离环境，但隔离层级和资源开销不同。

- **虚拟机**：通过 Hypervisor 虚拟硬件，每个虚拟机运行完整操作系统；
- **Docker 容器**：共享宿主机内核，通过 namespace、cgroups、镜像分层等机制隔离进程和资源。

## 面试官想考什么

- 是否理解容器不是完整虚拟机；
- 是否知道镜像、容器、仓库的关系；
- 是否能说出 Docker 的优势和限制；
- 是否理解容器化对部署一致性的价值。

## 标准回答

> Docker 相比虚拟机更轻量，因为容器共享宿主机内核，不需要为每个实例启动完整操作系统，所以启动更快、资源占用更低、镜像分发更方便。虚拟机隔离更彻底，适合需要完整 OS 隔离的场景；Docker 更适合应用交付、环境一致性和弹性部署。生产中通常结合 Kubernetes 做容器编排。

## 深挖追问

### Docker 为什么能保证环境一致？

应用依赖、运行时、配置模板可以打包进镜像，开发、测试、生产使用同一镜像版本，减少“我本地能跑、线上不能跑”的问题。

### 容器安全吗？

容器提供进程级隔离，但共享宿主机内核，隔离强度通常弱于虚拟机。生产环境需要控制权限、镜像来源、挂载目录和网络策略。

## 实战场景

Java 应用可以打成镜像，镜像中包含 JDK、应用 jar、启动脚本和基础配置。发布时通过镜像版本回滚，比手工复制 jar 更可控。

## 易错点/总结

- Docker 容器不是完整操作系统；
- 不要把大量可变数据写在容器内部，重要数据应挂载卷或使用外部存储；
- 镜像要尽量小，避免包含无用工具和敏感信息；
- 容器化解决交付一致性，不自动解决架构高可用。
