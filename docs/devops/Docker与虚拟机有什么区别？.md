# Docker与虚拟机有什么区别？

Docker与虚拟机有什么区别？
Docker与虚拟机的区别是什么？
虚拟机是一种软件，能在物理计算机上模拟出一台或多台虚拟的计算机。每台虚拟机都有自己的操作系统、内存、硬盘等，它们之间相互独立，就像在不同的物理机上运行。而Docker是一种开源的应用容器引擎，它可以让开发者打包他们的应用及其依赖包到一个轻量级、可移植的容器中。与虚拟机相比，Docker容器不需要运行一个完整的操作系统，而是共享宿主机的内核。Docker使用操作系统级虚拟化，而非硬件级虚拟化，每个容器共享宿主操作系统的内核，但各自拥有独立的用户空间。核心区别如下：
性能：Docker容器非常轻量级，启动迅速，消耗资源极少，容器共享宿主机的操作系统内核，因此相对虚拟机，容器的性能更加接近宿主机操作系统；而虚拟机是通过虚拟化技术模拟一台完整的计算机，所以虚拟机的操作更重；
速度：Docker容器由于直接运行在宿主机器上，所以启动无需启动操作系统，速度更快；而虚拟机需要启动完整的操作系统，因此启动耗时会更高；
隔离：Docker容器间相互隔离，但是共享宿主机的内核，而虚拟机则是完全隔离的，拥有自己的操作系统、内核和硬件虚拟化层，因此虚拟机的隔离性会更好。
迁移：Docker容器可以轻松地在不同的环境之间迁移，只要操作系统支持Docker即可，而虚拟机通常与特定的操作系统和硬件紧密绑定的，因此迁移和跨平台兼容性较差；
应用：Docker容器通过Dockfile记录构建过程，实现快速分发和部署，适合持续集成和微服务架构，而虚拟机通过镜像实现环境交付，灵活性较差；

云服务器中的vps与ecs是什么？
vps实际上就是服务器上分隔出来的虚拟机，也就是虚拟专用服务器，每个虚拟机都拥有独立的操作系统、CPU、内存、存储空间和公网IP，但是他不支持动态伸缩配置。如果加入了支持动态升降配，就是ecs了，也就是弹性计算服务。有了ecs服务器，我们可以手动部署我们的应用，但是一旦服务器数量激增，手动部署变得麻烦，而且不同的esc的操作系统可能存在差异，最简单的解决办法是将应用和操作系统打包成一个虚拟机，直接将其部署到esc上，但是这样的操作太重。更好的解决办法就是利用Docker容器技术，只将应用和系统依赖库打包，然后将这部分文件挂载esc服务器上，并利用namespace能力让它看起来上一个独立的操作系统，并通过cgroup的能力限制其使用资源的能力。这样就省去了打包操作系统。总的来说，服务器上跑多个esc，esc跑多个Docker容器，多个Docker容器共享esc实例的操作系统内核。
