# Docker与K8s之前的关系是什么？

Docker与K8s之前的关系是什么？
Docker是什么？
Docker是将应用程序与环境进行打包并运行的工具软件。应用程序跑在操作系统上，而操作系统又安装了各种不同的依赖库和配置，而这些被应用程序所依赖的信息，就是我们所讲的“环境”。

基础镜像是什么？
我们知道环境不同，会导致程序运行结果不同，那么我们首先要做的最重要的事情就是统一环境。首先最终要的就是操作系统，我们需要让所有的程序跑在同一个操作系统上。而操作系统分为用户空间和内核空间，应用程序运行在用户控件，因此我们只需要利用操作系统用户空间部分，就能构建应用所需的环境。其次是统一程序语言依赖，安装开发语言所以来的解析器或JVM等。选中一个基础操作系统和语言之后，我们将它们对应的文件系统、依赖库、配置等放一起打包成一个类似压缩包的文件，这就是所谓的基础镜像。

Dockerfile是什么？
通过命令清单的形式，指定完成从指定操作系统，到依赖环境安装，再到应用服务启动等一系列所需执行的操作，就是所谓的Dockerfile。
容器镜像是什么？
Dockerfile只是描述了要做什么事情，但并没有真正开始做。当我们命令执行docker build的时候，Docker软件就会按着Dockerfile的说明，一行行构建环境+应用程序，最终将这个环境+程序，打包成一个类似压缩包的东西，也就是“容器镜像”。只要将容器镜像传到任意一台服务器上，对这个压缩包进行解压缩，我们就能同时运行环境和程序。
Registry是什么？
为了解决容器镜像的传输问题，参考github代码仓库的做法，Docker也一样，弄一个镜像仓库，通过docker push将镜像推到仓库，有需要的时候再通过docker pull将镜像拉到机器上。这个负责管理镜像残酷推拉能力的服务，就���Docker Registry。基于Docker Registry的能力，我们可以搭建各种官方或私人镜像仓库，比如官方的叫DockerHub等。
容器是什么？
我们通过docker pull将容器镜像拉取到服务器本地，然后执行docker run命令，将这个类似压缩包的容器镜像给解压缩，获得一个独立的环境和应用程序并运行起来。这样一个独立的环境和应用程序，就是所谓的容器。我们可以在一个操作系统上同时跑多个容器，且这些容器之间都是相互独立，相互隔离的。容器本质上只是个自带独立运行环境的特殊进程，底层用的其实是宿主机的操作系统内核。
Docker的架构
Docker是经典的Client/Server架构，Client对应的是Docker-cli，Server对应Docker daemon。我们在命令行里敲Docker命令，使用的就是Docker-cli。Docker-cli会解析我们输入的cmd命令，然后调用Docker daemon守护进程提供的RESTful API，守护进程收到命令后，会根据指令创建和管理各个容器。Docker daemon底层内部分为Docker Server、Engine两层：
Docker Server本质就是个HTTP服务，负责堆外提供操作容器和镜像的api接口，接收到API请求后，会分发任务给Engine层；
Engine层负责创建Job，由Job实际执行各种工作。
Docker 本质上就是一个将程序和环境打包并运行的工具软件。具体点来说就是，它通过 Dockerfile 描述环境和应用程序的依赖关系， docker build 构建镜像， docker pull/push 跟 Docker Registry 交互实现存储和分发镜像，docker run 命令基于镜像启动容器，基于容器技术运行程序和它对应的环境，从而解决环境依赖导致的各种问题。
Docker Composer是什么？
如果我们需要部署多个容器，且对这些容器的顺序有要求，那么我们可以通过YAML文件编写好都有哪些容器，这些容器的顺序是怎么样，以及可以指定容器用到的内存和CPU等。然后通过docker composer up命令，就开始解析命令，然后按照指定的顺序对容器进行一键部署，从而完整一整套服务的部署，这其实就是docker composer实现的。
Docker Swarm是什么？
Docker解决一个容器的部署，Docker composer解决多个容器的部署，而Docker Swarm则是解决更高层次的容器集群部署问题的，比如A服务器坏了，则在B服务器上重新部署一套服务，完成迁移，同时还根据需求提供扩容。
Docker与K8s的关系是什么？
K8s的工作机制是在多态Node服务器上调度Pod，进行部署和扩容。每个Pod上部署则多个Container。这里的Container本质上就是一个进程，Docker容器实际上就是对应Container，它们都叫容器，只是K8s不只是支持Docker容器，还支持其他容器。K8s实现的功能和Docker Swarm有点类似，实际上它们之间是竞品关系，本质上都是在调度Pod。K8s官方定义，叫容器编排引擎，将它理解为，以 API 编程的方式管理安排各个容器的引擎，是不是就特别精辟。
