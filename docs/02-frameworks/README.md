# 框架篇

框架篇重点关注 **Spring 体系的核心机制**：IOC、AOP、Bean 生命周期、事务、MVC 调用链，以及框架如何帮助我们解耦、扩展和治理复杂业务。

## 复习路线

1. [Spring](spring/README.md)：先理解 Spring 容器的定位。
2. [Spring IOC 与 AOP 核心原理](spring/ioc-aop.md)：高频基础题，建议重点复习。

## 面试官想考什么

- 是否理解框架背后的设计思想，而不是只会使用注解；
- 是否知道 Bean 从扫描、实例化、依赖注入到初始化的完整流程；
- 是否能解释 AOP 的代理机制、失效场景和事务传播问题；
- 是否能结合项目说清楚为什么用 Spring，而不是只背概念。

## 标准回答思路

> Spring 的核心价值是通过 IOC 管理对象生命周期，通过 AOP 抽取横切逻辑，从而降低业务代码耦合。面试时需要把“用了什么注解”上升到“容器如何工作、代理如何生成、哪些场景会失效”。

## 易错点

- 把 IOC 简单理解成“new 对象交给 Spring”，忽略生命周期与扩展点；
- 不区分 JDK 动态代理和 CGLIB；
- 不理解同类内部方法调用导致事务/AOP 失效；
- 忽略循环依赖只在特定条件下可解决。
