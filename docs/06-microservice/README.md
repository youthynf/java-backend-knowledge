# 微服务篇

微服务面试重点不只是 Spring Cloud 组件，而是服务拆分后带来的治理问题：注册发现、配置、网关、负载均衡、熔断限流、链路追踪、分布式事务、灰度发布和可观测性。

## 复习路线

1. **架构取舍**：单体、分布式、微服务的边界与成本。
2. **服务治理**：注册发现、配置中心、网关、负载均衡、限流熔断降级。
3. **数据一致性**：本地事务、TCC、Saga、可靠消息最终一致、Seata。
4. **稳定性工程**：超时、重试、隔离、降级、监控、日志、链路追踪、压测和容量规划。
5. **发布运维**：蓝绿、金丝雀、灰度、滚动发布、回滚和兼容性。

## 面试官想考什么

- 你是否知道微服务解决了团队协作和独立部署问题，也引入了网络、数据一致性、运维复杂度。
- 你是否能把治理组件背后的通用原理讲清楚，而不是只背 Nacos、Gateway、Sentinel 名字。
- 你是否有线上稳定性思维：超时、重试、限流、熔断、隔离和观测必须成体系。

## 标准回答模板

回答微服务问题建议按 **业务背景 → 架构拆分 → 治理能力 → 故障边界 → 落地经验** 展开。不要一上来堆组件名，要先解释为什么需要这个能力。
