# 微服务

本目录覆盖微服务架构设计与治理：拆分原则、服务注册发现、配置中心、API 网关、熔断限流、服务通信、链路追踪等核心知识点。

- [服务注册发现如何实现](服务注册发现如何实现？.md) — Nacos/Eureka/Consul 注册中心原理与选型
- [微服务配置中心如何工作](微服务配置中心如何工作？.md) — Nacos Config/Apollo 动态刷新与灰度发布

- [服务熔断限流如何实现](服务熔断限流如何实现？.md) — Sentinel/Hystrix/Resilience4j 雪崩防护
- [微服务之间如何通信](微服务之间如何通信？.md) — Feign/Dubbo/gRPC/MQ 同步异步通信选型
- [微服务链路追踪如何实现](微服务链路追踪如何实现？.md) — SkyWalking/Zipkin/Jaeger 跨服务调用链追踪

## 复习路线

1. **架构取舍**：单体、模块化单体、微服务的边界与拆分原则。
2. **治理基石**：服务注册发现、配置中心。
3. **流量入口**：API 网关的路由、鉴权、限流、灰度。
4. **稳定性**：熔断、限流、降级、隔离。
5. **通信选型**：同步 HTTP/RPC、异步 MQ、超时重试幂等。
6. **可观测性**：链路追踪与日志、指标联动。
