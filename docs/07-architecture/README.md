# 架构设计

本目录覆盖架构层面的设计模式与系统设计实践。设计模式部分聚焦 Spring/AOP/RPC 等架构视角的落地，与 [01-java-core/design-pattern/](../01-java-core/design-pattern/) 偏 Java 实现的视角互补；系统设计部分覆盖高并发、高可用、限流、幂等、线上排障等工程主题。

## 目录

- [架构设计模式](patterns/README.md) — 代理、单例、策略、模板方法、工厂在架构层的落地
- [系统设计与线上排障](system-design/README.md) — CPU 排查、Redis 排查、秒杀、幂等、项目优化

## 推荐阅读顺序

1. 设计模式：先看策略、模板方法（理解 Spring 容器和模板类的本质），再看代理、工厂、单例（理解 AOP 和 Bean 管理背后的模式）。
2. 系统设计：先看 CPU 排查（建立排障方法论），再看秒杀和幂等（典型业务场景），最后看项目优化（综合应用）。
