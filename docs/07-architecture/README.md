# 架构设计

架构设计模块分为 **设计模式** 和 **系统设计** 两部分：前者关注代码层面的可扩展性，后者关注业务系统在高并发、高可用和可维护性上的取舍。

## 文档导航

- [设计模式](patterns/README.md)：单例、代理、策略、模板方法等常见模式。
- [设计模式补充](design-patterns/README.md)：工厂、单例等模式整理。
- [系统设计](system-design/README.md)：秒杀、幂等、CPU 排查、项目优化等场景题。

## 面试官想考什么

- 你是否能从业务目标出发设计系统，而不是堆技术名词；
- 你是否理解容量、延迟、一致性、可用性和成本之间的取舍；
- 你是否能识别瓶颈，并给出可验证的优化方案；
- 你是否有生产排障和灰度上线意识。

## 标准回答框架

> 先明确需求和约束，再拆分模块和核心流程，然后说明数据模型、缓存、异步、限流、降级、监控等设计，最后讲清楚风险与演进路线。

## 易错点

- 上来就选技术，不问业务目标和约束；
- 只画架构图，不讲数据一致性和异常流程；
- 忽略容量估算、压测和灰度；
- 把设计模式机械套用，导致过度设计。
