## 面试官想考什么

- 是否能拆清楚读写链路和核心瓶颈；
- 是否能设计幂等、限流、降级、异步削峰等机制；
- 是否能处理异常场景，例如重复请求、消息重复、服务超时；
- 是否能通过监控、日志和压测验证方案有效性。

## 标准回答模板

1. 明确需求：用户量、QPS、数据量、延迟、可用性目标。
2. 画核心链路：入口、服务、缓存、数据库、消息队列。
3. 找瓶颈：CPU、IO、锁竞争、热点 key、慢 SQL、外部依赖。
4. 设计方案：缓存、异步、限流、分片、读写分离、降级。
5. 兜底验证：压测、监控、告警、灰度、回滚。

## 易错点

- 不做需求澄清，直接套模板；
- 只说“加缓存/加 MQ”，不说一致性和失败补偿；
- 没有容量估算和监控指标；
- 忽视线上变更风险。
