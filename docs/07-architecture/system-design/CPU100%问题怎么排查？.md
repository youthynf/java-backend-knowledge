# CPU100%问题怎么排查？

CPU100%问题怎么排查？
首先通过 top 命令确认是不是 Java 进程导致的问题。如果确认是 Java 进程导致，可以通过两种方法进行问题定位，分别是：
使用系统工具和 JDK 自带的 jstack 工具；
使用 Arthas 探测工具；

使用 jstack 工具
使用 top 找到占用 CPU 最高的 Java 进程，确定进程 PID=13731；
找到占用 CPU 最高的线程，此时展示记录列表中的 PID 即为线程ID，假设 PID=13756：

top -Hp 13731
保存线程堆栈信息：通过 JDK 默认提供的一个工具 jstack，用于生成 Java 进程的线程快照（thread dump）。线程快照是一个关于 Java 进程中所有线程当前状态的快照，包括每个线程的堆栈信息。通过分析线程快照，可以了解 Java 进程中各个线程的运行状态、锁信息等。

jstack 13731 > thread_stack.log
在线程栈中找到罪魁祸首的线程：第二步已经拿到线程 ID（13756），然后我们需要将线程 ID 转换为十六进制，然后在上一步导出的 thread_stack.log 中查找这个16进制的线程 ID，进而我们可以看到对应的线程名称、线程状态、哪个方法的哪一行代码消耗了最多的 CPU 等信息。
使用 Arthas
Arthas 是阿里开源的一款线上监控诊断产品，通过全局视角实时查看应用 load、内存、gc、线程的状态信息，并能在不修改应用代码的情况下，对业务问题进行诊断，包括查看方法调用的出入参、异常，监测方法执行耗时，类加载信息等，大大提升线上问题排查效率。用 Arthas 查找占用 CPU 最高的方法只是一个开胃小菜，除此之外，它最大的用途是在不改代码、不重启服务的情况下对程序进行动态监控。如果你碰到了线上诡异问题，一定要用 Arthas 尝试找一找问题，开阔一下眼界。
下载 jar 包并安装到目标的服务器上，并启动服务：

curl -O https://arthas.aliyun.com/arthas-boot.jar
启动 Arthas 服务：

java -jar arthas-boot.jar
启动后，会列出当前这台服务器上的所有 Java 进程，然后选择需要排查的那个服务即可。
出现 arthas@ 后表示已经启动，并成功 attach 到目标进程上，通过输入 dashboard 命令可以看到实时面板信息，默认每5秒刷新一次，面板信息包括线程、内存堆栈、GC 和 Runtime 的基本信息，类似 VisualVM。
执行 thread 命令，这个命令会显示所有线程的信息，并且把 CPU 使用率最高的线程排在前面，帮助快速找到高占用的线程；
查看堆栈信息：使用 thread ID 获取堆栈信息，其中 ID 为线程 ID，可以直接将线程的堆栈信息打印出来。
