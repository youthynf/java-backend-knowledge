# 如何保证接口幂等

如何保证接口幂等
一、什么是接口幂等
所谓“接口幂等”是指任意多次对接口进行访问，对于资源的影响和返回结果都是相同的，不会对结果产生破坏或不可预料的结果。特别注意，这里的接口幂等不等同于禁止重复访问。
二、常见的保证幂等的方式
唯一标识
客户端生成唯一的UUID标识，服务端记录是否已经执行处理。前端请求时通过请求头或入参方式传递，服务端判断如果已经存在则不处理，不存在则处理并记录。
唯一索引约束
通过数据库唯一键约束，为业务字段增加唯一键索引，防止数据重复插入。如果重复数据插入捕获异常，并返回已存在的结果。
乐观锁
通过使用版本号或时间戳判断的方式，防止并发更新问题。
状态机
通过状态机记录当前业务所处的阶段，定义状态流转规则。当收到重复请求时，检查当前状态，如果已经是终态则直接返回。
去重表
单独维护一张已请求的记录表，业务操作前，先插入去重表，表中通过设计唯一键为请求ID或业务ID，如果插入失败，则说明已经处理。
Token机制
服务端生成Token，客户端提交时携带Token，服务端校验Token的有效性，无效则直接返回，有效则处理请求，并作废Token或标记已使用，防止客户端重复提交。
消息队列幂等
通过由消费者记录消息ID和业务ID是否已经处理，避免重复消费消息。
HTTP幂等操作
遵循HTTP的幂等设计，PUT请求和DELETE请求天然幂等。
