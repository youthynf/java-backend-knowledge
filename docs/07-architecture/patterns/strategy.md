# 策略模式

## 核心概念

策略模式把一组可替换的算法或业务规则封装成独立策略类，调用方依赖统一接口，在运行时选择具体策略。

## 面试官想考什么

- 是否理解策略模式解决的是变化的业务规则；
- 是否能用策略模式替代复杂 `if-else`；
- 是否知道策略模式和工厂、模板方法的区别；
- 是否能结合 Spring 自动注入策略集合。

## 标准回答

> 策略模式适用于同一业务流程中某个算法或规则经常变化的场景。它通过定义策略接口和多个实现类，把变化点从主流程中抽离出来，新增策略时尽量不修改主流程，符合开闭原则。实际项目中常配合工厂或 Spring Map 注入，根据类型选择策略。

## 示例

```java
public interface DiscountStrategy {
    BigDecimal discount(BigDecimal amount);
}
```

满减、折扣、会员价分别实现该接口，订单结算时根据活动类型选择对应策略。

## 深挖追问

### 策略模式和模板方法有什么区别？

策略模式通过组合切换不同算法；模板方法通过继承固定流程，把部分步骤交给子类实现。策略更灵活，模板方法更适合流程骨架稳定的场景。

## 易错点/总结

- 策略太少且不变化时，没必要强行抽象；
- 策略选择逻辑也要管理好，避免只是把 if-else 挪到工厂；
- 策略实现应尽量无状态，减少并发问题；
- 面试中最好结合支付、优惠、路由等业务场景回答。
